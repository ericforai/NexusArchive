## 🔴 发现的问题

### 1. 并发归档去重机制可能缺失 ⚠️ **潜在 Bug**

**问题描述**:
- 测试用例：5 个并发请求上传相同 `uniqueBizId` 的档案
- **发现**: 所有 5 个请求都成功返回 200/201
- **预期**: 应该只有一个成功，其他应该返回 409 Conflict 或类似错误

**测试输出**:
```
警告：5 个并发请求都成功了，可能没有实现去重机制
```

**影响**:
- 🔴 **严重程度**: 中-高
- 📊 **影响范围**: 数据一致性、存储空间
- ⚠️ **风险**: 
  - 可能导致重复档案
  - 浪费存储空间
  - 数据不一致

**建议**:
1. 检查后端是否实现了 `uniqueBizId` 的唯一性约束
2. 实现数据库唯一索引或应用层去重逻辑
3. 并发请求应该使用分布式锁或乐观锁机制
4. 失败的请求应该返回 409 Conflict 状态码

**复现步骤**:
```bash
BASE_URL=http://localhost:8080 npx playwright test tests/playwright/api/archive_concurrency.spec.ts
```
