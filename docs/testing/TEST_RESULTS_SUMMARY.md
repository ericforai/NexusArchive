## ❌ 失败的测试用例

### 主要失败原因分析

#### 1. 登录超时问题 (7个失败)
**影响范围**:
- `legacy_import.spec.ts` - 3个测试用例
- `destruction_workflow.spec.ts` - 1个测试用例
- `user_lifecycle.spec.ts` - 3个测试用例

**可能原因**:
- 登录流程响应慢
- 登录后跳转逻辑有问题
- 需要更长的等待时间

**建议修复**:
```typescript
// 增加超时时间
await page.waitForURL(url => url.pathname !== '/system/login', { timeout: 30000 });

// 或使用更灵活的等待策略
await page.waitForLoadState('networkidle');
```

#### 2. 页面标题选择器不匹配 (4个失败)
**影响范围**:
- `legacy_import.spec.ts` - 历史数据导入页面
- `auth_ticket.spec.ts` - 授权票据申请页面
- `user_lifecycle.spec.ts` - 权限复核页面

**可能原因**:
- 实际页面的标题结构与测试用例预期不一致
- 标题可能使用了不同的HTML标签或类名

**建议修复**:
- 检查实际页面的DOM结构
- 使用更通用的选择器（如 `h1, h2, h3` 或包含特定文本的元素）
- 或者添加 `data-testid` 属性到页面标题

#### 3. 路由路径未配置 (P1功能，5个失败)
**影响范围**:
- `mfa.spec.ts` - MFA设置页面
- `user_lifecycle.spec.ts` - 用户生命周期管理页面
- `freeze_hold.spec.ts` - 冻结/保全管理页面

**可能原因**:
- 这些页面的路由尚未添加到路由配置中
- 或者路由路径与测试用例中使用的不一致

**建议修复**:
- 确认所有P1功能页面的路由已正确配置
- 更新测试用例中的路由路径

#### 4. 元素不可见 (3个失败)
**影响范围**:
- `auth_ticket.spec.ts` - 授权票据列表
- `freeze_hold.spec.ts` - 冻结/保全列表和按钮

**可能原因**:
- 元素存在但被CSS隐藏
- 需要等待元素变为可见状态
- 或者元素在特定的交互后才显示

**建议修复**:
- 使用 `waitFor()` 等待元素可见
- 检查元素的CSS样式
- 确认是否需要先执行某些操作
