## 🛠️ 测试工具和环境

### 测试框架
- **UI E2E测试**: Playwright
- **单元测试**: Vitest (前端) + JUnit (后端)
- **API测试**: Playwright API Test Context

### 测试环境
- **开发环境**: `http://localhost:15175` (前端) + `http://localhost:19090/api` (后端)
- **测试数据库**: 独立的测试数据库实例
- **测试数据**: 使用fixtures提供测试数据

### 测试数据管理
- 使用测试fixtures提供基础数据
- 每个测试用例独立数据，避免相互影响
- 测试后清理测试数据

## 📝 测试报告模板

每个测试执行后应生成测试报告，包含：

1. **测试概览**
   - 测试执行时间
   - 测试用例总数
   - 通过/失败/跳过统计

2. **详细结果**
   - 每个测试用例的执行结果
   - 失败的测试用例详细错误信息
   - 截图或视频（Playwright自动生成）

3. **问题汇总**
   - 发现的问题列表
   - 问题严重程度
   - 问题复现步骤

4. **改进建议**
   - 测试覆盖率分析
   - 测试用例优化建议
