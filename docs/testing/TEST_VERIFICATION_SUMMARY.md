## 🔍 失败原因分析

### 1. 登录超时 (4个失败)
**影响测试**:
- `destruction_workflow.spec.ts` - 销毁执行页面
- `legacy_import.spec.ts` - 历史数据导入（2个测试）
- `mfa.spec.ts` - MFA设置相关（2个测试）
- `user_lifecycle.spec.ts` - 权限复核相关（2个测试）
- `freeze_hold.spec.ts` - 冻结/保全相关（2个测试）

**可能原因**:
- 登录流程在某些情况下响应较慢
- 需要进一步优化等待策略
- 可能需要增加重试机制

### 2. 标签页选择器超时 (2个失败)
**影响测试**:
- `legacy_import.spec.ts` - 导入和历史标签页

**可能原因**:
- 标签页的role属性可能不是"tab"
- 需要使用更灵活的选择器策略

### 3. 元素查找超时 (10个失败)
**影响测试**:
- 多个测试用例中的元素查找超时

**可能原因**:
- 元素加载较慢
- 需要更长的等待时间
- 选择器策略需要优化
