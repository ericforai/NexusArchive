# NexusArchive 技术栈文档

## 项目概述

digivoucher 是一个现代化的电子会计档案管理系统，采用前后端分离架构，后端基于 Spring Boot，前端基于 React。系统支持 AI 驱动的 OCR、智能关联和四性检测功能，并符合信创环境要求（达梦/人大金仓数据库适配）。

## 数据库架构

### 核心表结构
- **sys_user**: 系统用户
- **sys_role**: 角色 (三员管理)
- **sys_department**: 组织机构
- **acc_archive**: 电子会计档案核心表
- **acc_archive_relation**: 档案关联关系
- **sys_audit_log**: 安全审计日志
- **audit_inspection_log**: 四性检测日志

### 初始化脚本
- `src/main/resources/sql/schema-postgresql.sql`: PostgreSQL 初始化脚本
- `src/main/resources/db/migration/`: 数据库迁移脚本 (Flyway 格式)

## 开发环境配置

### 前端配置 (.env)
```env
VITE_API_BASE_URL=http://localhost:8080/api
```

### 后端配置 (application.yml)
```yaml
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/nexusarchive
    username: postgres
    password: postgres
  profiles:
    active: dev
```

## 部署方案

### 前端
- **构建**: `npm run build` -> 生成 `dist` 目录
- **部署**: Nginx 静态托管或 Vercel

### 后端
- **构建**: `mvn clean package` -> 生成 JAR 包
- **运行**: `java -jar target/nexusarchive-backend-2.0.0.jar`
- **环境**: 需要 JRE 17+
