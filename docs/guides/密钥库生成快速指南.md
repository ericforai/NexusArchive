# 密钥库生成快速指南

## 🚀 三种生成方法（按推荐顺序）

### 方法一：交互式脚本（最简单）⭐ 推荐

```bash
cd nexusarchive-java
bash scripts/generate_keystore_simple.sh
```

**优点**：
- 最简单，按提示输入即可
- 适合不熟悉命令的用户

**输入示例**：
- 密钥库密码：`changeit`
- 再次输入：`changeit`
- 私钥密码：直接回车
- 姓名：`NexusArchive Signing Certificate`
- 组织单位：`IT Department`
- 组织：`NexusArchive`
- 城市：`Beijing`
- 省份：`Beijing`
- 国家代码：`CN`
- 确认：`y`

---

### 方法二：使用 Docker（无需本地 Java）⭐ 推荐（如果没有 Java）

```bash
cd nexusarchive-java
bash scripts/generate_keystore_docker.sh
```

**优点**：
- 不需要本地安装 Java
- 只需要 Docker
- 自动下载 OpenJDK 镜像

**前提条件**：
- 已安装 Docker
- Docker 服务正在运行

---

### 方法三：手动命令（最灵活）

```bash
cd nexusarchive-java
mkdir -p keystore

keytool -genkeypair \
  -alias signing-cert \
  -keyalg RSA \
  -keysize 2048 \
  -sigalg SHA256withRSA \
  -validity 3650 \
  -keystore keystore/signature.p12 \
  -storetype PKCS12 \
  -storepass changeit \
  -keypass changeit \
  -dname "CN=NexusArchive Signing Certificate, OU=IT Department, O=NexusArchive, L=Beijing, ST=Beijing, C=CN"
```

**优点**：
- 完全控制所有参数
- 适合自动化脚本

**前提条件**：
- 已安装 Java JDK
- `keytool` 命令可用

---

## ⚙️ 安装 Java（如果需要）

### macOS

```bash
# 使用 Homebrew
brew install openjdk@17

# 设置环境变量
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
export PATH=$JAVA_HOME/bin:$PATH
```

### Linux

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install openjdk-17-jdk

# CentOS/RHEL
sudo yum install java-17-openjdk-devel
```

### 验证安装

```bash
java -version
keytool -help
```

## ✅ 验证生成结果

```bash
# 查看密钥库内容
keytool -list -v \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storepass changeit

# 查看证书有效期
keytool -list -v \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storepass changeit \
  -alias signing-cert | grep "Valid"
```

**预期输出**：
- 应该看到证书信息
- 有效期应该是 10 年（3650 天）

## 🔧 配置应用

生成密钥库后，确保 `application.yml` 配置正确：

```yaml
signature:
  keystore:
    path: ./keystore/signature.p12  # 相对路径
    password: changeit  # 您设置的密码
```

**使用绝对路径**（如果相对路径有问题）：
```yaml
signature:
  keystore:
    path: /Users/user/nexusarchive/nexusarchive-java/keystore/signature.p12
    password: changeit
```

## 🐛 常见问题

### Q1: 找不到 keytool

**解决方法**：
1. 安装 Java JDK（不是 JRE）
2. 设置 `JAVA_HOME` 环境变量
3. 或使用 Docker 方法

### Q2: 权限被拒绝

```bash
chmod +x scripts/generate_keystore_simple.sh
chmod +x scripts/generate_keystore_docker.sh
```

### Q3: Docker 镜像下载慢

```bash
# 使用国内镜像源
docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/openjdk:17-jdk-slim
```

### Q4: 密钥库已存在

删除旧文件或使用不同的文件名：
```bash
rm nexusarchive-java/keystore/signature.p12
```

## 📝 快速命令参考

### 生成密钥库（一行命令）

```bash
cd nexusarchive-java && \
mkdir -p keystore && \
keytool -genkeypair -alias signing-cert -keyalg RSA -keysize 2048 \
  -validity 3650 -keystore keystore/signature.p12 -storetype PKCS12 \
  -storepass changeit -keypass changeit \
  -dname "CN=NexusArchive Signing Certificate, OU=IT Department, O=NexusArchive, L=Beijing, ST=Beijing, C=CN"
```

### 使用 Docker（一行命令）

```bash
cd nexusarchive-java && \
mkdir -p keystore && \
docker run --rm -v $(pwd)/keystore:/keystore openjdk:17-jdk-slim \
  keytool -genkeypair -alias signing-cert -keyalg RSA -keysize 2048 \
  -validity 3650 -keystore /keystore/signature.p12 -storetype PKCS12 \
  -storepass changeit -keypass changeit \
  -dname "CN=NexusArchive Signing Certificate, OU=IT Department, O=NexusArchive, L=Beijing, ST=Beijing, C=CN"
```
