# 密钥库用途说明

## 📋 什么是密钥库？

**密钥库（Keystore）** 是一个安全存储容器，用于保存：
- **数字证书**（Certificate）：用于身份验证和加密
- **私钥**（Private Key）：用于数字签名和解密
- **公钥**（Public Key）：用于验证签名和加密

在 **NexusArchive 电子会计档案系统** 中，密钥库主要用于 **电子签章功能**。

---

## 🎯 密钥库在本系统中的作用

### 1. **电子签章（Digital Signature）**

密钥库存储的私钥用于对电子档案进行数字签名，确保：

- ✅ **真实性**：证明文件确实由指定机构/人员签署
- ✅ **完整性**：文件内容未被篡改
- ✅ **不可否认性**：签署者无法否认已签署的事实

**使用场景**：
- 归档凭证自动签章
- PDF/OFD 文件签章
- 档案完整性验证

### 2. **签名验证（Signature Verification）**

密钥库中的证书用于验证签名的有效性：

- 验证签名是否由正确的私钥生成
- 验证证书是否有效（未过期、未被吊销）
- 验证签名算法是否符合要求（SM2/SM3 国密算法）

### 3. **合规要求**

根据 **DA/T 94-2022《电子会计档案管理规范》**：

- 电子会计档案必须具有电子签名
- 签名必须符合《电子签名法》要求
- 信创环境必须支持国密算法（SM2/SM3/SM4）

## 🔐 密钥库的技术细节

### 密钥库格式

系统使用 **PKCS12** 格式（`.p12` 文件）：
- 标准格式，兼容性好
- 可以同时存储私钥和证书
- 支持密码保护

### 密钥库内容

```
密钥库 (signature.p12)
├── 证书别名: signing-cert
│   ├── 私钥 (Private Key)
│   │   └── 用于生成数字签名
│   ├── 证书 (Certificate)
│   │   ├── 公钥 (Public Key)
│   │   ├── 证书信息 (CN, OU, O, L, ST, C)
│   │   └── 有效期 (Valid from/to)
│   └── 证书链 (Certificate Chain)
│       └── 可选的 CA 证书链
```

### 签名流程

```
1. 加载密钥库
   ↓
2. 读取私钥（使用证书别名和密码）
   ↓
3. 计算文件哈希值（SM3 或 SHA256）
   ↓
4. 使用私钥对哈希值进行签名（SM2 或 RSA）
   ↓
5. 将签名值附加到文件或单独存储
   ↓
6. 记录签章日志（签名时间、证书信息等）
```

### 验签流程

```
1. 加载密钥库
   ↓
2. 读取证书（使用证书别名）
   ↓
3. 提取公钥
   ↓
4. 重新计算文件哈希值
   ↓
5. 使用公钥验证签名值
   ↓
6. 检查证书有效性（有效期、吊销状态）
   ↓
7. 返回验签结果（VALID/INVALID）
```

## 💼 实际应用场景

### 场景 1：归档凭证自动签章

当凭证归档完成后，系统自动使用密钥库中的私钥对凭证文件进行签名：

```java
// 自动签章流程
ArchiveCompletedEvent → SignatureTimestampListener
  → SignatureAdapter.sign(data, certAlias)
    → 从密钥库加载私钥
    → 生成数字签名
    → 保存签章日志
```

### 场景 2：PDF/OFD 文件签章

对 PDF 或 OFD 格式的会计档案进行可视化签章：

```java
// PDF 签章
signatureAdapter.signPdf(pdfFile, keystorePath, keystorePassword);

// OFD 签章
ofdSignatureHelper.signOfd(inputPath, outputPath, keystorePath, keystorePassword);
```

### 场景 3：档案完整性验证

在档案检索或预览时，自动验证签名：

```java
// 验证签名
VerifyResult result = signatureAdapter.verify(data, signature, certAlias);

if (result.isValid()) {
    // 档案完整，未被篡改
} else {
    // 档案可能被篡改，记录告警
}
```

## 🔧 系统配置

### application.yml 配置

```yaml
signature:
  keystore:
    path: ./keystore/signature.p12  # 密钥库文件路径
    password: changeit               # 密钥库密码
  auto-sign: true                   # 是否自动签章
  default-cert-alias: signing-cert  # 默认证书别名
```

### 环境变量配置

```bash
# 密钥库路径（可选，覆盖配置文件）
export SIGNATURE_KEYSTORE_PATH=/path/to/signature.p12

# 密钥库密码（可选，覆盖配置文件）
export SIGNATURE_KEYSTORE_PASSWORD=your_strong_password
```

## 🔒 安全注意事项

### 1. **密钥库密码**

- ✅ 使用强密码（至少 16 位，包含大小写字母、数字、特殊字符）
- ❌ 不要使用默认密码 `changeit`（生产环境）
- ✅ 定期更换密码

### 2. **文件权限**

```bash
# 设置严格的文件权限
chmod 600 keystore/signature.p12
```

### 3. **备份策略**

- ✅ 定期备份密钥库文件
- ✅ 备份文件加密存储
- ✅ 备份文件与主文件分开存放

### 4. **证书有效期**

- ✅ 监控证书有效期
- ✅ 在证书过期前更新
- ✅ 使用长期有效证书（建议 10 年）

### 5. **访问控制**

- ✅ 限制密钥库文件的访问权限
- ✅ 不要将密钥库提交到版本控制系统
- ✅ 使用环境变量传递密码（不要硬编码）

## 📝 相关 API 端点

### 签章相关

- `POST /api/signature/sign` - 对数据进行签名
- `POST /api/signature/verify` - 验证签名
- `POST /api/signature/verify-pdf` - 验证 PDF 签章
- `POST /api/signature/verify-ofd` - 验证 OFD 签章
- `GET /api/signature/logs` - 查询签章日志
- `GET /api/signature/status` - 检查签章服务状态

### 证书管理

- `GET /api/certificate` - 查询所有证书
- `GET /api/certificate/{alias}` - 查询指定证书
- `POST /api/certificate/{alias}/verify` - 验证证书有效性

## 🎓 总结

**密钥库是电子签章功能的核心组件**，它：

1. **存储私钥和证书**：用于生成和验证数字签名
2. **确保合规性**：满足 DA/T 94-2022 和《电子签名法》要求
3. **保障档案安全**：提供真实性、完整性、不可否认性保证
4. **支持信创环境**：支持 SM2/SM3 国密算法

**没有密钥库，电子签章功能将无法工作！**
