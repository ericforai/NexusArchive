## 1. 后端管理与配置

### 1.1 配置文件

**主配置文件**: `nexusarchive-java/src/main/resources/application.yml`

**关键配置项**:

```yaml
# 服务端口
server:
  port: 8080

# 数据库 (通过环境变量配置)
spring:
  datasource:
    url: jdbc:postgresql://${DB_HOST}:${DB_PORT}/${DB_NAME}
    username: ${DB_USER}
    password: ${DB_PASSWORD}

# 文件存储路径
archive:
  root:
    path: ${ARCHIVE_ROOT_PATH:./data/archives}

# JWT 配置
jwt:
  secret: ${JWT_SECRET}
  expiration: 86400000  # 24小时
```

### 1.2 环境变量

生产环境通过 `.env` 文件配置:

```bash
# /opt/nexusarchive/.env

# 数据库类型选择
SPRING_PROFILES_ACTIVE=prod          # PostgreSQL
# SPRING_PROFILES_ACTIVE=prod-dameng   # 达梦
# SPRING_PROFILES_ACTIVE=prod-kingbase # 人大金仓

# 数据库连接
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=nexusarchive
DB_USER=nexus
DB_PASSWORD=YourStrongPassword

# 安全密钥
JWT_SECRET=your-256-bit-secret-key

# 存储路径
ARCHIVE_ROOT_PATH=/opt/nexusarchive/data/archives
```

### 1.3 日志管理

**日志位置**:
- 后端日志: `journalctl -u nexusarchive`
- Nginx 访问日志: `/var/log/nginx/access.log`
- Nginx 错误日志: `/var/log/nginx/error.log`

**查看实时日志**:
```bash
journalctl -u nexusarchive -f
```

### 1.4 服务管理

```bash
# 启动服务
systemctl start nexusarchive

# 停止服务
systemctl stop nexusarchive

# 重启服务
systemctl restart nexusarchive

# 查看状态
systemctl status nexusarchive

# 设置开机自启
systemctl enable nexusarchive
```

## 2. 数据库配置

### 2.1 PostgreSQL

**创建数据库**:
```bash
# 登录 PostgreSQL
sudo -u postgres psql

# 创建用户和数据库
CREATE USER nexus WITH PASSWORD 'YourPassword';
CREATE DATABASE nexusarchive OWNER nexus;
GRANT ALL PRIVILEGES ON DATABASE nexusarchive TO nexus;
\q
```

**配置连接**:
```ini
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=nexusarchive
DB_USER=nexus
DB_PASSWORD=YourPassword
```

### 2.2 达梦数据库 (DM8)

**创建表空间和用户**:
```sql
-- 登录 DBA
CREATE TABLESPACE NEXUS_TS DATAFILE '/dm8/data/nexus.dbf' SIZE 100M;
CREATE USER NEXUS IDENTIFIED BY "YourPassword" DEFAULT TABLESPACE NEXUS_TS;
GRANT DBA TO NEXUS;
```

**配置连接**:
```ini
SPRING_PROFILES_ACTIVE=prod-dameng
DB_HOST=127.0.0.1
DB_PORT=5236
DB_NAME=NEXUSARCHIVE
DB_USER=NEXUS
DB_PASSWORD=YourPassword
```

### 2.3 人大金仓 (Kingbase)

**创建数据库**:
```bash
# 使用 ksql 连接
CREATE DATABASE nexusarchive OWNER system;
```

**配置连接**:
```ini
SPRING_PROFILES_ACTIVE=prod-kingbase
DB_HOST=127.0.0.1
DB_PORT=54321
DB_NAME=nexusarchive
DB_USER=system
DB_PASSWORD=YourPassword
```

### 2.4 数据库迁移

系统使用 **Flyway** 自动管理数据库版本:
- 首次启动时自动创建所有表
- 升级时自动执行增量迁移
- 迁移脚本位于: `src/main/resources/db/migration/`

## 3. 服务器部署

### 3.1 部署方式对比

| 方式 | 适用场景 | 优点 | 缺点 |
|:---|:---|:---|:---|
| 离线安装包 | 无外网环境 | 一键部署 | 需预先下载 |
| 在线部署脚本 | 有外网环境 | 自动构建 | 需要构建工具 |
| Docker | 容器化环境 | 快速扩展 | 需要 Docker 环境 |

### 3.2 离线安装包部署

详见 [第4章 离线安装包](#4-离线安装包)

### 3.3 在线部署

**步骤1: 服务器初始化**
```bash
./deploy/init_server.sh 服务器IP
```

**步骤2: 安装数据库**
```bash
./deploy/install_db.sh 服务器IP
```

**步骤3: 部署应用**
```bash
./deploy/deploy.sh
```

### 3.4 Nginx 配置

```nginx
server {
    listen 80;
    server_name _;
    
    # 前端静态文件
    location / {
        root /opt/nexusarchive/frontend;
        try_files $uri $uri/ /index.html;
    }
    
    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        client_max_body_size 100M;
    }
}
```

## 4. 离线安装包

离线安装包支持在**完全无外网**的环境下一键部署 NexusArchive 系统。

### 4.1 安装包内容

#### 4.1.1 核心脚本

| 脚本 | 说明 |
|:---|:---|
| `install.sh` | 主安装脚本，含交互式配置向导 |
| `install.conf.template` | 配置文件模板（用于静默安装） |
| `uninstall.sh` | 卸载脚本 |
| `upgrade.sh` | 升级脚本（保留数据） |

#### 4.1.2 目录结构

```
nexusarchive-offline-installer-2.0.0/
├── install.sh              # 一键安装脚本
├── install.conf.template   # 配置模板
├── uninstall.sh           # 卸载脚本
├── upgrade.sh             # 升级脚本
├── bin/                    # 后端 JAR 文件
│   └── nexusarchive-backend-2.0.0.jar
├── frontend/               # 前端静态资源
├── config/                 # 配置文件模板
│   ├── nexusarchive.service
│   └── nginx.conf
├── db/migration/           # 数据库迁移脚本
├── deps/                   # 离线依赖 (JDK 等)
│   └── README.md
└── docs/                   # 安装文档
    └── 安装手册.md
```

### 4.2 安装脚本功能

#### 4.2.1 环境检测
- **操作系统识别**: CentOS / Ubuntu / 麒麟 V10 / 统信 UOS
- **CPU 架构检测**: x86_64 / ARM64 (aarch64)
- **磁盘空间检查**: 自动验证是否有足够空间
- **Java 版本检测**: 自动检测或使用离线 JDK

#### 4.2.2 配置向导

安装脚本支持两种模式：

**交互式模式** (默认):
```
========================================
   NexusArchive 安装配置向导
========================================

安装目录 [/opt/nexusarchive]: 
数据存储目录 [/opt/nexusarchive/data]: 
服务端口 [8080]: 

--- 数据库配置 ---
数据库类型:
  1) PostgreSQL
  2) 达梦 (DM8)
  3) 人大金仓 (Kingbase)
选择 [1]: 
```

**配置文件模式** (批量部署):
预先编辑 `install.conf` 文件，脚本自动读取配置。

#### 4.2.3 信创支持

安装脚本自动识别信创环境并进行适配：
- 🇨🇳 **麒麟 V10**: 自动识别，优化 ARM64 兼容性
- 🇨🇳 **统信 UOS**: 自动识别，处理 deb 包差异
- 🏛️ **达梦/金仓**: 自动选择对应的 Spring Profile

### 4.3 构建安装包

在开发环境执行:
```bash
./deploy/build_offline_package.sh
```

构建过程:
1. 编译后端 JAR (`mvn package`)
2. 编译前端资源 (`npm run build`)
3. 复制配置文件和迁移脚本
4. 生成卸载/升级脚本
5. 打包成 tar.gz

### 4.5 静默安装 (批量部署)

适用于需要部署多台服务器的场景：

```bash
# 1. 复制配置模板
cp install.conf.template install.conf

# 2. 编辑配置文件
cat > install.conf << EOF
INSTALL_DIR=/opt/nexusarchive
DATA_DIR=/opt/nexusarchive/data
SERVER_PORT=8080
DB_TYPE=postgresql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=nexusarchive
DB_USER=nexus
DB_PASSWORD=YourStrongPassword
EOF

# 3. 执行安装
sudo ./install.sh
```

### 4.6 升级

```bash
# 升级会自动备份旧版本
sudo ./upgrade.sh
```

升级过程:
1. 备份当前 JAR 和前端文件
2. 停止服务
3. 替换新版本文件
4. 启动服务
5. 验证启动成功

### 4.7 卸载

```bash
sudo ./uninstall.sh
```

> ⚠️ **注意**: 卸载不会删除数据库和归档文件，需手动清理。

### 4.8 添加离线依赖

如果目标服务器没有 Java，可将 JDK 放入 `deps/` 目录：

1. 下载 OpenJDK 17: https://adoptium.net/
2. 重命名为: `jdk-17_linux-x86_64.tar.gz` (或 `aarch64`)
3. 放入 `deps/` 目录
4. 安装脚本会自动使用

## 5. 故障排除

### 5.1 快速检查清单

遇到问题时按此顺序检查:

- [ ] 后端服务运行中？ `systemctl status nexusarchive`
- [ ] Nginx 运行中？ `systemctl status nginx`
- [ ] 数据库连接正常？ 检查 `.env` 配置
- [ ] Redis 运行中？ `redis-cli ping` (如启用)
- [ ] 磁盘空间充足？ `df -h`
- [ ] 端口未被占用？ `ss -tlnp | grep 8080`

### 5.2 常见问题

#### 5.2.1 登录后菜单消失 / 401 错误

**原因**: Redis 连接失败导致 Token 验证异常

**解决方案**:
```bash
# 方案1: 启动 Redis
systemctl start redis

# 方案2: 禁用 Redis (修改配置)
# spring.data.redis.enabled: false
```

#### 5.2.2 前端 404 错误

**原因**: Nginx 配置问题或前端资源未正确部署

**解决方案**:
1. 检查前端文件是否存在: `ls /opt/nexusarchive/frontend/`
2. 检查 Nginx 配置: `nginx -t`
3. 确保 `try_files` 配置正确

#### 5.2.3 上传文件失败 (413 错误)

**原因**: Nginx 上传大小限制

**解决方案**:
```nginx
# 在 nginx.conf 中添加
client_max_body_size 100M;
```

#### 5.2.4 数据库迁移失败

**原因**: Flyway 版本冲突或 SQL 语法错误

**解决方案**:
```bash
# 查看迁移历史
PGPASSWORD=xxx psql -h localhost -U nexus -d nexusarchive \
  -c "SELECT * FROM flyway_schema_history ORDER BY installed_rank DESC LIMIT 5;"

# 删除失败的记录
PGPASSWORD=xxx psql -h localhost -U nexus -d nexusarchive \
  -c "DELETE FROM flyway_schema_history WHERE success = false;"
```

#### 5.2.5 后端启动失败

**诊断步骤**:
```bash
# 1. 查看日志
journalctl -u nexusarchive -n 100

# 2. 手动运行查看错误
cd /opt/nexusarchive
java -jar app.jar
```
