## 1. 系统概述

### 1.1 产品简介

NexusArchive 是一款符合中国会计档案法规（《会计档案管理办法》财政部79号令、GB/T 39362-2020）的**电子会计档案管理系统**，专为企业级私有化部署设计。

### 1.2 核心特性

| 特性 | 说明 |
|:---|:---|
| 📁 **电子档案管理** | 会计凭证、账簿、财务报告的全生命周期管理 |
| 🔒 **四性检测** | 真实性、完整性、可用性、安全性自动检测 |
| 🇨🇳 **信创支持** | 支持麒麟/统信 OS、达梦/金仓数据库、国密 SM2/SM3 |
| 📊 **智能分析** | 档案全景分析、统计报表、关系图谱 |
| 🔐 **等保合规** | 三员分立、审计日志、权限控制 |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      前端 (React + TypeScript)               │
├─────────────────────────────────────────────────────────────┤
│                      Nginx (反向代理)                        │
├─────────────────────────────────────────────────────────────┤
│                  后端 (Spring Boot 3.x)                      │
├─────────────────────────────────────────────────────────────┤
│           数据库 (PostgreSQL / 达梦 / 人大金仓)              │
└─────────────────────────────────────────────────────────────┘
```

## 3. 前端功能使用指南

### 3.1 登录与首页

1. 打开浏览器访问系统地址
2. 输入用户名和密码
3. 点击"登录"按钮

**首页仪表盘** 显示:
- 档案总量统计
- 各类型档案占比

### 3.2 档案管理

#### 3.2.1 档案列表
- **路径**: 侧边栏 → 档案管理 → 档案列表
- **功能**: 查看、搜索、筛选所有电子档案

**操作说明**:
| 操作 | 说明 |

## 4. 后端管理与配置

### 4.1 配置文件

**主配置文件**: `nexusarchive-java/src/main/resources/application.yml`

**关键配置项**:

```yaml
# 服务端口
server:
  port: 8080

# 数据库 (通过环境变量配置)
spring:
  datasource:
    url: jdbc:postgresql://${DB_HOST}:${DB_PORT}/${DB_NAME}
    username: ${DB_USER}
    password: ${DB_PASSWORD}

# 文件存储路径
archive:
  root:
    path: ${ARCHIVE_ROOT_PATH:./data/archives}

# JWT 配置
jwt:
  secret: ${JWT_SECRET}
  expiration: 86400000  # 24小时
```

### 4.2 环境变量

生产环境通过 `.env` 文件配置:

```bash
# /opt/nexusarchive/.env

# 数据库类型选择
SPRING_PROFILES_ACTIVE=prod          # PostgreSQL
# SPRING_PROFILES_ACTIVE=prod-dameng   # 达梦
# SPRING_PROFILES_ACTIVE=prod-kingbase # 人大金仓

# 数据库连接
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=nexusarchive
DB_USER=nexus
DB_PASSWORD=YourStrongPassword

# 安全密钥
JWT_SECRET=your-256-bit-secret-key

# 存储路径
ARCHIVE_ROOT_PATH=/opt/nexusarchive/data/archives
```

### 4.3 日志管理

**日志位置**:
- 后端日志: `journalctl -u nexusarchive`
- Nginx 访问日志: `/var/log/nginx/access.log`
- Nginx 错误日志: `/var/log/nginx/error.log`

**查看实时日志**:
```bash
journalctl -u nexusarchive -f
```

### 4.4 服务管理

```bash
# 启动服务
systemctl start nexusarchive

# 停止服务
systemctl stop nexusarchive

# 重启服务
systemctl restart nexusarchive

# 查看状态
systemctl status nexusarchive

# 设置开机自启
systemctl enable nexusarchive
```

## 5. 数据库配置

### 5.1 PostgreSQL

**创建数据库**:
```bash
# 登录 PostgreSQL
sudo -u postgres psql

# 创建用户和数据库
CREATE USER nexus WITH PASSWORD 'YourPassword';
CREATE DATABASE nexusarchive OWNER nexus;
GRANT ALL PRIVILEGES ON DATABASE nexusarchive TO nexus;
\q
```

**配置连接**:
```ini
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=nexusarchive
DB_USER=nexus
DB_PASSWORD=YourPassword
```

### 5.2 达梦数据库 (DM8)

**创建表空间和用户**:
```sql
-- 登录 DBA
CREATE TABLESPACE NEXUS_TS DATAFILE '/dm8/data/nexus.dbf' SIZE 100M;
CREATE USER NEXUS IDENTIFIED BY "YourPassword" DEFAULT TABLESPACE NEXUS_TS;
GRANT DBA TO NEXUS;
```

**配置连接**:
```ini
SPRING_PROFILES_ACTIVE=prod-dameng
DB_HOST=127.0.0.1
DB_PORT=5236
DB_NAME=NEXUSARCHIVE
DB_USER=NEXUS
DB_PASSWORD=YourPassword
```

### 5.3 人大金仓 (Kingbase)

**创建数据库**:
```bash
# 使用 ksql 连接
CREATE DATABASE nexusarchive OWNER system;
```

**配置连接**:
```ini
SPRING_PROFILES_ACTIVE=prod-kingbase
DB_HOST=127.0.0.1
DB_PORT=54321
DB_NAME=nexusarchive
DB_USER=system
DB_PASSWORD=YourPassword
```

### 5.4 数据库迁移

系统使用 **Flyway** 自动管理数据库版本:
- 首次启动时自动创建所有表
- 升级时自动执行增量迁移
- 迁移脚本位于: `src/main/resources/db/migration/`

### 6.4 Nginx 配置

```nginx
server {
    listen 80;
    server_name _;
    
    # 前端静态文件
    location / {
        root /opt/nexusarchive/frontend;
        try_files $uri $uri/ /index.html;
    }
    
    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        client_max_body_size 100M;
    }
}
```

## 7. 离线安装包

离线安装包支持在**完全无外网**的环境下一键部署 NexusArchive 系统。

### 7.1 安装包内容

#### 7.1.1 核心脚本

| 脚本 | 说明 |
|:---|:---|
| `install.sh` | 主安装脚本，含交互式配置向导 |
| `install.conf.template` | 配置文件模板（用于静默安装） |
| `uninstall.sh` | 卸载脚本 |
| `upgrade.sh` | 升级脚本（保留数据） |

#### 7.1.2 目录结构

```
nexusarchive-offline-installer-2.0.0/
├── install.sh              # 一键安装脚本
├── install.conf.template   # 配置模板
├── uninstall.sh           # 卸载脚本
├── upgrade.sh             # 升级脚本
├── bin/                    # 后端 JAR 文件
│   └── nexusarchive-backend-2.0.0.jar
├── frontend/               # 前端静态资源
├── config/                 # 配置文件模板
│   ├── nexusarchive.service
│   └── nginx.conf
├── db/migration/           # 数据库迁移脚本
├── deps/                   # 离线依赖 (JDK 等)
│   └── README.md
└── docs/                   # 安装文档
    └── 安装手册.md
```

### 7.2 安装脚本功能

#### 7.2.1 环境检测
- **操作系统识别**: CentOS / Ubuntu / 麒麟 V10 / 统信 UOS
- **CPU 架构检测**: x86_64 / ARM64 (aarch64)
- **磁盘空间检查**: 自动验证是否有足够空间
- **Java 版本检测**: 自动检测或使用离线 JDK

#### 7.2.2 配置向导

安装脚本支持两种模式：

**交互式模式** (默认):
```

数据存储目录 [/opt/nexusarchive/data]:

  1) PostgreSQL
  2) 达梦 (DM8)
  3) 人大金仓 (Kingbase)
```

#### 7.2.3 信创支持

安装脚本自动识别信创环境并进行适配：
- 🇨🇳 **麒麟 V10**: 自动识别，优化 ARM64 兼容性
- 🇨🇳 **统信 UOS**: 自动识别，处理 deb 包差异


1. 备份当前 JAR 和前端文件
2. 停止服务
3. 替换新版本文件
4. 启动服务

1. 下载 OpenJDK 17: https://adoptium.net/
2. 重命名为: `jdk-17_linux-x86_64.tar.gz` (或 `aarch64`)
3. 放入 `deps/` 目录
4. 安装脚本会自动使用

## 8. 故障排除

### 8.1 快速检查清单

遇到问题时按此顺序检查:

- [ ] 后端服务运行中？ `systemctl status nexusarchive`
- [ ] Nginx 运行中？ `systemctl status nginx`
- [ ] 数据库连接正常？ 检查 `.env` 配置
- [ ] Redis 运行中？ `redis-cli ping` (如启用)
- [ ] 磁盘空间充足？ `df -h`
- [ ] 端口未被占用？ `ss -tlnp | grep 8080`

### 8.2 常见问题

#### 8.2.1 登录后菜单消失 / 401 错误

**原因**: Redis 连接失败导致 Token 验证异常

**解决方案**:
```bash
# 方案1: 启动 Redis
systemctl start redis

# 方案2: 禁用 Redis (修改配置)
# spring.data.redis.enabled: false
```

#### 8.2.2 前端 404 错误

**原因**: Nginx 配置问题或前端资源未正确部署

**解决方案**:
1. 检查前端文件是否存在: `ls /opt/nexusarchive/frontend/`
2. 检查 Nginx 配置: `nginx -t`
3. 确保 `try_files` 配置正确

#### 8.2.3 上传文件失败 (413 错误)

**原因**: Nginx 上传大小限制

**解决方案**:
```nginx
# 在 nginx.conf 中添加
client_max_body_size 100M;
```

#### 8.2.4 数据库迁移失败

**原因**: Flyway 版本冲突或 SQL 语法错误

**解决方案**:
```bash
# 查看迁移历史
PGPASSWORD=xxx psql -h localhost -U nexus -d nexusarchive \
  -c "SELECT * FROM flyway_schema_history ORDER BY installed_rank DESC LIMIT 5;"

# 删除失败的记录
PGPASSWORD=xxx psql -h localhost -U nexus -d nexusarchive \
```

## 9. 常见问题 FAQ

### Q1: 系统支持哪些浏览器？

A: 推荐使用 Chrome、Edge、Firefox 最新版本。不支持 IE 浏览器。

### Q2: 如何修改管理员密码？

A: 登录后进入 系统设置 → 个人中心 → 修改密码。

### Q3: 档案存储在哪里？

A: 默认存储路径为 `/opt/nexusarchive/data/archives/`，可通过 `ARCHIVE_ROOT_PATH` 环境变量修改。

### Q4: 如何备份数据？

A: 
1. 备份数据库: `pg_dump -h localhost -U nexus nexusarchive > backup.sql`
2. 备份文件: `tar -czf files_backup.tar.gz /opt/nexusarchive/data/`

### Q5: 支持集群部署吗？

A: 当前版本为单节点部署。集群支持计划在 v3.0 版本实现。

### Q6: 如何对接 ERP 系统？

A: 系统支持用友 YonSuite 对接。配置方法详见 `application.yml` 中的 `yonsuite` 配置节。

### Q7: 保管期限有哪些选项？

A: 根据《会计档案管理办法》:
- **10年**: 其他财务资料
- **30年**: 会计凭证、会计账簿
- **永久**: 财务报告
