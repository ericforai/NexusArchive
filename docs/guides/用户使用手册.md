## 1. 系统概述

NexusArchive 是一款专为企业级财务归档设计的私有化部署解决方案，严格遵循国家档案局 **DA/T 94-2022《电子会计档案管理规范》** 及 **DA/T 104-2024《ERP系统电子文件归档和电子档案管理规范》**。系统采用高性能模块化单体架构，深度适配信创环境，提供从凭证采集、四性检测、归档整理、AIP长期保存到借阅利用的全生命周期管理能力。

### 核心价值
*   **合规性**: 内置"四性检测"及《会计档案管理办法》符合性检查。
*   **安全性**: 
    *   **国密支持**: 完整支持 **SM2** (签名/验签)、**SM3** (哈希摘要)、**SM4** (敏感数据字段级加密)。
    *   **信创适配**: 兼容麒麟/统信操作系统及达梦/人大金仓数据库。
*   **高性能**: 集成 Redis 缓存加速，支持千万级档案秒级检索。
*   **标准化**: 支持导出符合 **GB/T 39674** 标准的版式文档及结构化 AIP 包。

## 2. 快速入门

### 2.1 登录与导航
1.  **访问**: 在浏览器输入 URL 访问系统。
2.  **深层链接 (Deep Linking)**: 系统支持 URL 驱动导航。您可以将任意页面（如某个特定档案的预览页）收藏为书签，点击即可直达，无需从首页逐级点击。
3.  **历史记录**: 完整支持浏览器的"前进/后退"按钮，操作体验与标准网页一致。

### 2.2 界面主页
登录后默认进入**工作台 (Dashboard)**：
*   **健康状态**: 系统最近一次"四性检测"和"符合性检查"的通过率。
*   **待办事项**: 审批任务提醒。
*   **数据概览**: 档案总量、今日新增量、电子凭证池状态。

## 3. 功能指南

### 3.1 工作台 (Dashboard)
提供全局运营看板，包括档案统计、趋势图及存储空间预警。

### 3.2 档案采集 (Collection)

#### 3.2.1 原始凭证库 (Original Voucher)
*   **独立管理**: 作为一个独立的资产库，管理所有尚未生成记账凭证的原始单据（发票、银行回单、合同等）。
*   **版本控制**: 支持同一单据的多版本上传与历史追溯。
*   **格式支持**: 重点支持 **OFD**、PDF 及图片格式，上传时自动进行哈希计算。
  - **OFD 在线预览**: 支持符合 GB/T 33190 标准的 OFD 版式文档直接在线预览，无需转换。
  - **多附件切换**: 档案详情页支持在多个附件间快速切换浏览。
  - **预览工具栏**: 提供缩放、旋转、下载等便捷操作。

#### 3.2.2 电子凭证池集成
*   **多源异构集成**:
    *   **YonSuite (用友)**: 支持通过 API 自动同步凭证、辅助核算及附件。
    *   **SAP / 金蝶 / 泛微 OA**: 内置标准连接器。
*   **深度同步 (Deep Sync)**: 针对 YonSuite 等系统，支持同步完整的凭证分录信息（摘要、科目编码、辅助核算），并存储于元数据中供检索。

#### 3.2.2 智能关联 (Auto-Correlation)
系统每日凌晨自动运行，或手动触发。
*   **智能匹配**: 自动识别凭证与发票、回单、合同的关联。
*   **关联全景**: 可视化展示"原始凭证-记账凭证-依据文件"证据链，支持点击穿透。

#### 3.2.3 OCR 识别
支持 PDF/JPG 智能识别，自动提取元数据。

#### 3.2.4 异常数据处理
人工修正校验失败或 OCR 错误的数据。

### 3.3 整理编目 (Cataloging)

#### 3.3.1 案卷管理
*   **自动组卷**: 按月将凭证自动组卷。
*   **合规元数据**: 自动生成符合 DA/T 104 的案卷号与标题。
*   **归档流程**: 组卷(Draft) -> 审核(Pending) -> 封存(Archived)。
*   **归档登记表**: 自动生成 GB/T 18894 格式登记表。

### 3.4 档案库房 (Repository)

#### 3.4.1 列表与预览
*   **增强预览**:
    *   **电子凭证翻阅器**: 分屏对照、X光透视（金额/科目高亮）、动态安全水印。
    *   **账簿/报表**: 结构化还原展示。
*   **AIP 导出**: 导出符合 GB/T 39674 的结构化包（含 XML 索引与版式文件）。

#### 3.4.2 四性检测与合规
*   **四性检测**: 真实性(SM2/SM3校验)、安全性(病毒扫描)、可用性(格式检查)、完整性(元数据校验)。
*   **符合性检查**: 自动校验《会计档案管理办法》条款（保管期限、归档时间等）。

### 3.5 档案利用 (Utilization)

#### 3.5.1 全库检索
*   **快捷键**: `Cmd/Ctrl + K` 随时唤起。
*   **智能搜索**: 支持全文模糊检索，结果自动分类导航。

#### 3.5.2 借阅与全景
*   **借阅**: 严格的审批流程与有效期控制，支持受控下载。
*   **全景视图**: 多维展示档案血缘关系。

### 3.6 鉴定销毁
支持在线鉴定流程与销毁执行，操作记录永久审计。

### 3.7 统计分析
提供归档率、数字化率等多维度报表。

### 3.8 系统设置 (Settings)
*(管理员功能)*

*   **全宗管理**: 维护多全宗体系及根基设置。
*   **系统监控**:
    *   内置 **Prometheus + Grafana** 监控面板，实时查看 CPU、内存、JVM 状态及磁盘用量。
*   **审计日志**: GB/T 39784 标准全量审计，防篡改记录。

### 3.9 运维管理 (Operations)

#### 3.9.1 备份与恢复
*   **自动备份**: 系统支持定时全量/增量数据库备份。
*   **灾难恢复**: 提供一键恢复脚本，确保数据安全。

#### 3.9.2 离线管理
*   **离线安装**: 支持在无外网的内网环境通过离线包部署。
*   **License 管理**:
    *   **生成与导入**: 系统管理员可使用 `generate_license.py` 工具生成专用 License。
    *   **授权控制**: License 包含最大用户数、节点数及有效期限制。过期将限制系统访问。
    *   **持久化**: License 文件在服务器本地加密存储，重启不丢失。

## 4. 常见问题 (FAQ)

**Q: 系统支持哪些国产数据库？**
A: 目前已完整适配达梦 (Dameng) 8/9 和人大金仓 (Kingbase) V8。

**Q: 为什么上传 PDF 后四性检测"真实性"未通过？**
A: 1. 文件哈希不匹配；2. 数字签名无效。请检查源文件。

**Q: 忘记密码如何重置？**
A: 请联系管理员。

## 5. 术语表

*   **SM4**: 国密分组密码算法，用于数据加密。
*   **AIP**: 归档信息包。
*   **Deep Sync**: 深度同步，指全量获取凭证分录详情的技术。
