# 离线部署与激活简易手册

本手册旨在帮助非技术人员在完全离线环境下快速部署 NexusArchive 系统，并解决 License 激活问题。

---

## 1. Mac 电脑部署 (演示/试用)

适用于在个人 MacBook 上进行演示或离线试用。

### 1.1 准备工作
- 确保电脑已安装 Java 17+ (在终端输入 `java -version` 检查)
- 确保也就是本地已安装 PostgreSQL 数据库 (如果是演示，建议安装 Postgres.app)

### 1.2 启动步骤
1. 解压离线安装包 `nexusarchive-offline-installer-2.0.0.tar.gz`
2. 打开终端 (Terminal)，进入解压后的目录
3. 运行启动脚本:
   ```bash
   ./deploy/start_for_mac.sh
   ```
4. 脚本会自动检查环境并生成默认配置文件 `.env`。如果需修改数据库密码，请编辑该文件。
5. 等待日志显示 `Started NexusArchiveApplication` 即表示启动成功。
6. 浏览器访问 `http://localhost:8080` (取决于配置端口)

---

## 2. Linux 服务器部署 (生产环境)

适用于正式的服务器环境（如 CentOS, Ubuntu, 麒麟, 统信）。

### 2.1 安装步骤
1. 将安装包上传至服务器 `/tmp` 目录
2. 执行安装命令:
   ```bash
   tar -xzf nexusarchive-offline-installer-2.0.0.tar.gz
   cd nexusarchive-offline-installer-2.0.0
   sudo ./install.sh
   ```
3. 按照屏幕提示输入安装路径和数据库信息。

## 3. License 激活 (重要)

由于离线包默认未附带有效 License，您需要使用自带的生成工具创建一个。

### 3.1 生成新的 License
在**任意**安装了 Java 的电脑上 (可以是 Mac 或 Linux):

1. 进入 `tools` 目录 (离线包解压后包含)
2. 编译并运行生成器:
   ```bash
   # 方法 A: 直接运行 (需 Java 11+)
   java tools/LicenseGenerator.java
   
   # 方法 B: 编译运行
   javac tools/LicenseGenerator.java
   java -cp tools LicenseGenerator
   ```
3. **交互式配置**:
   - 程序会询问 **过期时间** (例如输入 `2026-12-31`)
   - 程序会询问 **最大用户数** (例如输入 `100`)
   - 程序会询问 **最大节点数** (例如输入 `10`)

4. **保存结果**:
   - 程序会生成 `license.json` 文件。
   - 程序会屏幕打印一大串 **公钥 (Public Key)**。

### 3.2 应用 License

#### 第一步: 配置公钥
打开服务器上的配置文件 (Mac为 `.env` 或 `application.yml`，Linux通常在 `/opt/nexusarchive/.env` 或配置中心):

修改或添加以下配置 (将 `...` 替换为刚才生成的公钥字符串):
```yaml
license:
  public-key: "MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA..."
```
*(注意: YAML缩进要正确，不要有换行符)*

#### 第二步: 上传 License 文件
将生成的 `license.json` 复制到数据目录下的根目录。
- **Mac**: 默认为 `nexusarchive/data/license.json`
- **Linux**: 默认为 `/opt/nexusarchive/data/license.json`

#### 第三步: 重启服务
```bash
# Mac
Ctrl+C 停止后重新运行 ./deploy/start_for_mac.sh

# Linux
systemctl restart nexusarchive
```

重启后，系统将自动加载并验证新的 License。
