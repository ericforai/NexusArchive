# 智能凭证关联功能操作手册

一旦我所属的文件夹有所变化，请更新我。

## 功能概述

智能凭证关联是一个基于规则引擎的自动化工具，用于将**记账凭证**与其对应的**原始凭证**（发票、合同、银行回单等）进行智能匹配关联，提高档案归档的完整性和合规性。

---

## 核心功能点

### 1. 业务场景自动识别
系统根据凭证的**会计科目**自动识别业务场景：

| 场景 | 借方科目 | 贷方科目 | 应关联文档 |
|------|---------|---------|-----------|
| 付款业务 | 应付账款/费用 | 银行存款/现金 | 发票、银行回单、合同 |
| 收款业务 | 银行存款/现金 | 应收账款/收入 | 发票、银行回单 |
| 费用报销 | 费用类科目 | 现金/其他应付 | 发票、报销单 |
| 采购入库 | 存货/资产 | 应付账款 | 采购订单、入库单、发票 |
| 销售出库 | 成本类科目 | 存货 | 销售订单、出库单 |

### 2. 多维度智能评分
系统对每个候选文档从多个维度进行评分：

- **金额匹配**（权重 40%）：支持精确匹配和容差匹配（±1%）
- **日期匹配**（权重 30%）：凭证日期与单据日期的接近度
- **交易对手匹配**（权重 30%）：客商名称的模糊匹配

### 3. 三级关联强度
根据业务规则定义文档的必要性级别：

| 级别 | 标识 | 说明 | 举例 |
|------|------|------|------|
| 🔴 必关联 | MUST_LINK | 缺失将影响合规 | 付款业务必须有发票 |
| 🟠 应关联 | SHOULD_LINK | 建议关联 | 大额付款应有合同 |
| 🔵 可关联 | MAY_LINK | 可选增强 | 差旅可附行程单 |

## 使用流程

### 步骤一：初始化配置（首次使用）

**入口**：侧边栏 → **智能匹配** → **初始化向导**

1. **扫描数据**：系统自动扫描您的科目表和单据类型
2. **应用预置规则**：选择适合您企业的行业预置包
3. **确认映射**：确认科目角色和单据类型角色的映射关系
4. **完成**：初始化完成，可开始使用智能关联

### 步骤二：执行智能关联

**入口**：侧边栏 → **智能匹配** → **智能关联**

1. 选择待关联的记账凭证
2. 点击 **「开始匹配」** 按钮
3. 系统自动执行匹配（异步处理，通常 2-5 秒）
4. 查看匹配结果：
   - ✅ **已匹配**：自动关联成功
   - ⚠️ **需确认**：有多个候选，需人工选择
   - ❌ **待补证**：未找到匹配文档，需补传

### 步骤三：处理待确认项

当系统找到多个候选文档时：
1. 点击 **「选择候选」** 按钮
2. 查看候选列表（含评分和匹配原因）
3. 选择正确的文档点击 **「确认」**

### 步骤四：查看合规报告

**入口**：侧边栏 → **智能匹配** → **合规报告**

- 查看关联统计（总数、已匹配、待补证、合规率）
- 导出待补证清单（可按部门、期间筛选）

## API 接口（开发者）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/matching/execute/{voucherId}` | POST | 执行单凭证匹配 |
| `/api/matching/execute/batch` | POST | 批量匹配 |
| `/api/matching/task/{taskId}` | GET | 查询异步任务结果 |
| `/api/matching/confirm` | POST | 确认关联 |
| `/api/matching/templates` | GET | 获取规则模板 |

## 常见问题

### Q1：匹配不到发票怎么办？
1. 检查发票是否已上传到系统
2. 确认发票金额、日期是否与凭证一致
3. 检查客商名称是否匹配（支持模糊匹配）

### Q2：如何自定义匹配规则？
目前版本使用预置规则模板，后续版本将支持自定义规则配置。

### Q3：支持批量匹配吗？
支持。可通过 `/api/matching/execute/batch` 接口批量提交凭证 ID。

- **DA/T 104-2024**：归档接口规范
