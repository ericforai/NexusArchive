# 安全指南

> NexusArchive 安全配置与最佳实践

---

## 🔐 安全架构

NexusArchive 采用多层安全防护机制，符合等保 2.0 三级要求。

### 核心安全组件

| 组件 | 文件 | 说明 |
|------|------|------|
| 安全配置 | `SecurityConfig.java` | Spring Security 配置 |
| 启动验证 | `SecurityConfigValidator.java` | 生产环境配置检查 |
| 请求限流 | `RateLimitFilter.java` | 令牌桶算法防 DDoS |
| XSS 过滤 | `XssFilter.java` | 跨站脚本攻击防护 |
| 密码策略 | `PasswordPolicyValidator.java` | 强密码验证 |
| 路径安全 | `PathSecurityUtils.java` | 路径遍历防护 |
| 登录锁定 | `LoginAttemptService.java` | Redis 分布式锁定 |

## 🔑 敏感信息外置

**重要**: 生产环境必须通过环境变量配置敏感信息。

### 必需环境变量

| 变量名 | 说明 | 安全级别 |
|--------|------|----------|
| `JWT_SECRET` | JWT 签名密钥（≥32字符） | 🔴 关键 |
| `DB_PASSWORD` | 数据库密码 | 🔴 关键 |
| `SM4_KEY` | SM4 加密密钥（32位Hex） | 🔴 关键 |

### 推荐环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `CORS_ALLOWED_ORIGINS` | 允许跨域的源 | `*` (开发) |
| `YONSUITE_APP_KEY` | 用友 AppKey | - |
| `YONSUITE_APP_SECRET` | 用友 AppSecret | - |

### 配置示例

```bash
# 生成安全的 JWT 密钥
openssl rand -base64 32

# 生成 SM4 密钥
openssl rand -hex 16

# 设置环境变量
export JWT_SECRET="your-secure-jwt-secret-key-here"
export DB_PASSWORD="secure-database-password"
export SM4_KEY="your-32-char-hex-key"
export CORS_ALLOWED_ORIGINS="https://your-domain.com"
```

参考模板: [/.env.template](/.env.template)

## 🛡️ 安全配置

### 1. 请求频率限制

系统使用令牌桶算法防止暴力破解和 DDoS 攻击：

| 接口类型 | 限制 | 封禁时长 |
|----------|------|----------|
| 普通接口 | 100 次/分钟 | 1 分钟 |
| 登录接口 | 10 次/分钟 | 5 分钟 |

配置类: `RateLimitFilter.java`

### 2. 登录保护

- **失败锁定**: 5 次失败后锁定账户 15 分钟
- **分布式支持**: 使用 Redis 存储锁定状态
- **优雅降级**: Redis 不可用时自动切换内存存储

配置类: `LoginAttemptService.java`

### 3. 密码策略

强制密码复杂度要求：

- ✅ 最小长度 8 位
- ✅ 包含大写字母
- ✅ 包含小写字母
- ✅ 包含数字
- ✅ 包含特殊字符
- ❌ 禁止常见弱密码
- ❌ 禁止连续字符

配置类: `PasswordPolicyValidator.java`

### 4. XSS 防护

自动过滤危险内容：

- `<script>` 标签
- `javascript:` 协议
- `onXXX` 事件属性
- Base64 编码攻击

配置类: `XssFilter.java`

### 5. 路径安全

防止路径遍历攻击：

- 规范化路径处理
- 检测 `../` 遍历
- 验证根目录边界
- 危险扩展名过滤

配置类: `PathSecurityUtils.java`

## 📋 审计日志防篡改

NexusArchive 采用 **SM3 哈希链** 确保审计日志不可篡改。

### 哈希链机制

```
日志1 → SM3(内容+NULL) → Hash1
日志2 → SM3(内容+Hash1) → Hash2
日志3 → SM3(内容+Hash2) → Hash3
...
```

每条日志记录：
- `log_hash`: 当前日志的 SM3 哈希值
- `prev_log_hash`: 前一条日志的哈希值

### 验证日志完整性

```java
// 验证指定时间范围的日志链
AuditLogService.LogChainVerifyResult result = 
    auditLogService.verifyLogChain(startDate, endDate);

if (!result.isValid()) {
    // 日志被篡改！
    log.error("日志链验证失败: {}", result.getMessage());
}
```

### 数据库字段

```sql
-- V24__enhance_audit_log.sql
ALTER TABLE sys_audit_log ADD COLUMN prev_log_hash VARCHAR(64);
ALTER TABLE sys_audit_log ADD COLUMN log_hash VARCHAR(64);
CREATE INDEX idx_audit_log_hash ON sys_audit_log(log_hash);
```

## 🔏 国密算法支持

### SM3 摘要算法

用于审计日志哈希链和文件完整性校验：

```java
// 计算哈希
String hash = sm3Utils.hash("content");

// 验证文件
String fileHash = sm3Utils.hash(fileBytes);
```

### SM4 对称加密

用于敏感字段加密存储：

```java
// 加密 (自动配合 TypeHandler)
archive.setTitle("机密文档标题"); // 自动加密存储

// 解密 (自动)
String title = archive.getTitle(); // 自动解密读取
```

加密字段：
- `acc_archive.title` (档案题名)

### SM2 签名算法

用于电子签章验签：

```java
// 签名
SignResult result = sm2SignatureService.sign(data, certAlias);

// 验签
VerifyResult verify = sm2SignatureService.verify(data, signature, certAlias);
```
