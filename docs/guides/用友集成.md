# 用友云 YonSuite 集成指南

本文档详细介绍如何配置和使用 NexusArchive 与用友云 YonSuite 的集成功能。

---

## 凭证同步

系统支持自动从用友云 YonSuite 拉取会计凭证数据。

### 配置项

在 `application.yml` 中配置以下参数：

```yaml
yonsuite:
  base-url: https://dbox.yonyoucloud.com  # 用友 API 网关地址
  app-key: your-app-key                    # 应用密钥
  app-secret: your-app-secret              # 应用密钥密码
```

### 同步能力

- 按账簿代码 (`accbookCode`) 筛选
- 按会计期间范围同步 (`periodStart` ~ `periodEnd`)
- 支持增量同步和全量同步

### 数据映射

自动映射用友凭证字段到 EAA 档案结构，生成标准档号。

### 凭证分录完整同步

- 自动同步凭证的完整分录信息，包括 **摘要 (description)** 和 **科目 (accsubject)**
- 分录数据存储于 `custom_metadata` 字段 (PostgreSQL JSONB 类型)
- 导出 AIP 包时，PDF 凭证将展示真实的业务摘要和科目名称

## Webhook 集成

本系统支持通过 Webhook 实时接收 YonSuite 的凭证变更通知。

### 配置前提

YonSuite 服务器必须能够访问您的 NexusArchive 实例：
- **生产环境**: 确保服务器拥有公网 IP 或域名。
- **本地测试**: 使用内网穿透工具 (如 ngrok, cpolar) 生成公网 URL。

### 控制台配置

1. 登录 **YonSuite 开发者中心** (或集成平台)。
2. 进入 **消息推送** 或 **Webhook 订阅**。
3. 创建新订阅:
   - **事件类型**: 选择 `凭证审核` (Voucher Audit) 或 `凭证保存` (Voucher Save)。
   - **回调地址 (Callback URL)**: 填入您的公网地址 + `/api/integration/yonsuite/webhook`。
     - 例如: `http://180.184.54.214/api/integration/yonsuite/webhook`
     - **注意**: 系统上下文路径为 `/api`，因此完整 URL 必须包含 `/api`。
   - **签名算法**: 选择 `HMAC-SHA256`。
   - **密钥 (Secret)**: 确保密钥与 `application.yml` 中的 `yonsuite.app-secret` 一致。

## 故障排除

### "系统访问回调地址失败" 问题

如果在点击 YonSuite 控制台的 "测试" 按钮时遇到此错误：

1. **检查 Nginx 配置**:
   - **不要强行开启 HTTPS**，除非服务器上真的有有效证书。
   - 如果使用 IP 访问，请使用监听 80 端口的简单 HTTP 配置。
   - 确保云服务器安全组 (防火墙) 允许入方向 TCP 80 流量。

2. **理解 "测试" 按钮行为**:
   - YonSuite 的 "测试" 按钮是一个连通性检查 (Health Check)，而非完整的功能测试。
   - 它可能会发送 **GET** 请求。
   - 它可能会发送 **空包体 (Empty Body)** 的 POST 请求。
   - 它可能**不带签名**。
   - **关键**: 您的控制器必须对 GET 请求和空 Body 返回 `200 OK`，否则 YonSuite 会认为服务不可用。

3. **控制器健壮性**:
   - 实现 `@GetMapping` 端点处理健康检查。
   - 在 `@PostMapping` 中检查 `null` body 并立即返回 200。

## 已解决的技术问题

### 1. Connection Refused 问题

- **症状**: Java 应用无法连接 `https://dbox.yonyoucloud.com`，报 `ConnectException: Connection refused`
- **根因**: Java 默认使用系统代理设置 (`java.net.useSystemProxies`)，而代理服务器未运行
- **解决**: 启动时添加 JVM 参数 `-Djava.net.useSystemProxies=false` 禁用系统代理

### 2. JSONB 类型不匹配问题

- **症状**: 保存凭证分录时报错 `column "custom_metadata" is of type jsonb but expression is of type character varying`
- **根因**: MyBatis-Plus 默认将 String 类型直接插入 JSONB 列，PostgreSQL 拒绝隐式转换
- **解决**: 创建 `PostgresJsonTypeHandler.java`，使用 `Types.OTHER` 正确处理 JSONB 类型转换
