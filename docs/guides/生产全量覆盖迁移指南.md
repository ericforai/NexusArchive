# 生产全量覆盖迁移指南（整库 + 文件）

## 适用场景
- 需要将本地完整数据（数据库 + 附件存储）覆盖到服务器。
- 明确接受“目标环境现有数据会被覆盖”。

## 一、生成迁移包（本地）

```bash
cd /Users/user/nexusarchive
bash scripts/create_full_migration_package.sh
```

默认会导出：
- DB: `nexusarchive`（来自 `nexus-db` 容器）
- 文件: `ARCHIVE_ROOT_PATH`（默认 `./nexusarchive-java/data/archives`）

脚本会输出：
- 压缩包路径（`full_migration_*.tar.gz`）
- 包 SHA256（用于工作流校验）

## 二、上传迁移包到服务器本地路径

将生成的 `full_migration_*.tar.gz` 上传到生产服务器，例如：

```bash
scp /Users/user/nexusarchive/migration_packages/full_migration_*.tar.gz \
  <user>@<server>:/opt/nexusarchive/migration_packages/
```

记录该文件的绝对路径（例如 `/opt/nexusarchive/migration_packages/full_migration_20260212_143000.tar.gz`）。

## 三、执行 GitHub Actions

工作流：`Migrate Prod Full Overwrite via SSH`

输入参数：
1. `server_package_path`: 迁移包在服务器上的绝对路径
2. `migration_package_sha256`: 迁移包 SHA256
3. `confirm_text`: 必须填写 `I_UNDERSTAND_FULL_OVERWRITE`
4. `healthcheck_url`: 健康检查地址（默认 `https://www.digivoucher.cn/api/health`）
5. `archive_root_path`:
   - 留空：自动读取 `/etc/nexusarchive/nexusarchive.env` 的 `ARCHIVE_ROOT_PATH`
   - 或手动指定绝对路径

## 四、工作流执行内容

1. 下载迁移包并校验 SHA256
1. 从 `server_package_path` 复制迁移包并校验 SHA256
2. 备份当前线上：
   - DB: `/opt/nexusarchive/migration_backups/db_before_migrate_*.sql`
   - Files: `/opt/nexusarchive/migration_backups/files_before_migrate_*.tgz`
3. 停服务 `nexusarchive`
4. 覆盖恢复 DB（drop/create + restore）
5. 覆盖恢复 `ARCHIVE_ROOT_PATH`
6. 启服务并健康检查

## 五、迁移后验收（建议）

1. 健康检查
2. 登录系统手工验证关键档案预览
3. 抽样检查下载接口（HTTP 200 + Content-Length 正常）

## 六、回滚方案

若迁移后异常：
1. 停服务
2. 用备份 SQL 恢复数据库
3. 用备份 tgz 恢复 `ARCHIVE_ROOT_PATH`
4. 启服务并健康检查

> 备份文件由工作流自动生成，路径见执行日志末尾。
