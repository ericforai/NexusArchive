# 历史数据导入使用指南

> **版本**: v1.0.0  
> **用途**: 历史数据导入功能的使用说明  
> **适用对象**: 系统管理员

---

## 📋 功能概述

历史数据导入功能允许系统管理员将历史档案数据批量导入系统，支持 CSV 和 Excel 格式，系统会自动识别并创建全宗结构。

**访问路径**: 系统管理 → 历史数据导入

---

## 📁 文件格式要求

### 支持的格式

- **CSV文件** (.csv)
  - 编码：UTF-8（支持BOM）
  - 分隔符：逗号（,）
  - 第一行必须为表头（字段名）

- **Excel文件** (.xlsx, .xls)
  - 第一个工作表（Sheet）包含数据
  - 第一行必须为表头（字段名）

### 文件大小限制

- 最大文件大小：100MB
- 建议单次导入行数：不超过 10,000 行（大数据量建议分批导入）

---

## 📊 数据字段要求

### 必需字段

以下字段为必需字段，导入文件中必须包含：

| 字段名 | 类型 | 说明 | 示例 | 备注 |
|--------|------|------|------|------|
| `fonds_no` | 字符串 | 全宗号 | JD-001 | 长度1-50，字母数字下划线 |
| `fonds_name` | 字符串 | 全宗名称 | 京东集团 | 不能为空 |
| `archive_year` | 整数 | 归档年度 | 2024 | 有效年份：1900-2100 |
| `doc_type` | 字符串 | 档案类型 | 凭证、报表、账簿等 | 必须在允许的类型列表中 |
| `title` | 字符串 | 档案标题 | 2024年1月记账凭证 | 长度1-255 |
| `retention_policy_name` | 字符串 | 保管期限名称 | 永久、30年、10年等 | 必须在系统中存在 |

### 可选字段

以下字段为可选字段，根据需要填写：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `entity_name` | 字符串 | 法人实体名称 | 北京京东世纪贸易有限公司 |
| `entity_tax_code` | 字符串 | 统一社会信用代码 | 91110000MA01234567 |
| `doc_date` | 日期 | 形成日期 | 2024-01-15 |
| `amount` | 数字 | 金额 | 1000000.00 |
| `counterparty` | 字符串 | 对方单位 | 供应商A |
| `voucher_no` | 字符串 | 凭证号 | V-202401-001 |
| `invoice_no` | 字符串 | 发票号 | INV-202401-001 |
| `custom_metadata` | JSON字符串 | 扩展元数据 | `{"key":"value"}` |
| `file_path` | 字符串 | 文件路径 | /data/archives/xxx.pdf |
| `file_hash` | 字符串 | 文件哈希值 | abc123... |

## 🔄 导入流程

### 步骤1: 准备数据文件

1. 按照上述格式要求准备CSV或Excel文件
2. 确保必需字段已填写
3. 检查数据格式是否正确（日期格式、数字格式等）

### 步骤2: 上传文件

1. 登录系统，进入 **系统管理 → 历史数据导入**
2. 点击"选择文件"或拖拽文件到上传区域
3. 支持的文件格式：CSV、Excel (.xlsx, .xls)

### 步骤3: 预览数据

1. 点击"预览数据"按钮
2. 系统会解析文件并显示预览结果：
   - 总行数
   - 有效行数
   - 无效行数
   - 全宗数量
   - 验证错误列表（如有）
3. 检查预览结果，确认数据正确

### 步骤4: 执行导入

1. 确认预览结果无误后，点击"确认导入"按钮
2. 系统开始执行导入（可能需要较长时间，请耐心等待）
3. 导入完成后显示结果摘要：
   - 成功导入行数
   - 失败行数
   - 自动创建的全宗列表
   - 错误报告下载链接（如有错误）

### 步骤5: 查看导入历史

1. 切换到"导入历史"标签页
2. 查看历史导入记录：
   - 文件名
   - 操作人
   - 导入时间
   - 导入结果（成功/失败/部分成功）
   - 错误报告下载（如有错误）

## ⚠️ 注意事项

### 数据验证规则

1. **全宗号格式**
   - 只能包含字母、数字、下划线
   - 长度限制：1-50 个字符
   - 示例：`JD-001`、`FONDS_001`、`ABC123`

2. **归档年度**
   - 必须是有效年份：1900-2100
   - 必须与 `doc_date` 的年份一致（或接近）

3. **档案类型**
   - 必须在系统允许的类型列表中
   - 常见类型：凭证、报表、账簿、合同、发票等

4. **保管期限**
   - 必须在系统中存在
   - 常见值：永久、30年、10年等

5. **日期格式**
   - 标准格式：`YYYY-MM-DD`
   - 示例：`2024-01-15`

6. **金额格式**
   - 数字格式，支持小数
   - 示例：`1000000.00`、`1234.56`

### 业务规则

1. **全宗自动创建**
   - 如果 `fonds_no` 不存在，系统会自动创建全宗
   - 全宗状态默认为 `ACTIVE`
   - 全宗有效期：从当前日期开始，永久有效

2. **实体自动创建**
   - 如果 `entity_name` 和 `entity_tax_code` 对应的实体不存在，系统会自动创建
   - 优先使用 `entity_tax_code` 进行匹配（精确匹配）
   - 如果 `entity_tax_code` 为空，使用 `entity_name` 进行模糊匹配

3. **同一批导入限制**
   - 建议同一批导入的数据使用相同的 `fonds_no`
   - 跨全宗导入需要相应权限

4. **事务处理**
   - 导入过程在一个事务中执行
   - 如果发生错误，支持全部回滚
   - 支持部分成功导入（成功行导入，失败行记录错误）

## 🐛 常见问题

### Q1: 导入失败怎么办？

**A**: 查看导入结果中的错误报告，根据错误提示修改数据后重新导入。

常见错误：
- 必需字段缺失
- 字段格式不正确
- 业务规则验证失败

### Q2: 如何修改字段映射？

**A**: 如果CSV/Excel中的列名与标准字段名不一致，可以在上传时配置字段映射。目前版本暂不支持前端配置，需要后端API支持。

### Q3: 可以导入大文件吗？

**A**: 支持，但建议：
- 单次导入不超过 10,000 行
- 大文件可以拆分为多个小文件分批导入
- 文件大小限制：100MB

### Q4: 导入后如何查看导入的数据？

**A**: 导入成功后，数据会出现在对应的档案列表中。可以：
- 使用档案检索功能查询
- 按全宗号、归档年度、档案类型等条件筛选

### Q5: 导入错误报告在哪里下载？

**A**: 
- 导入完成后，如果有错误，会在导入结果中显示"下载错误报告"按钮
- 在"导入历史"标签页中，每条导入记录（如有错误）都会显示"错误报告"下载链接
