# 端到端测试流程（凭证归档主链路）

## 0. 目标
- 覆盖从 ERP 同步到归档完成的全链路。
- 每一步都有可复核证据：API 响应、页面截图、SQL 结果、日志条目。
- 保障可清理、可复跑、可审计。

## 1. 前置条件
- 环境：非生产环境。
- 服务：前后端服务在线，数据库可访问。
- 权限：ERP 同步、归档、审批、审计权限均可用。
- 工具：浏览器、SQL 客户端、日志查看工具。

## 2. 统一标记
- `RUN_ID`：例如 `20250105-1430`。
- `T0`：开始时间戳（精确到秒）。
- 测试数据命名：凭证号/标题/备注包含 `RUN_ID`，用于清理与追溯。

### RUN_ID 落点规则（必须遵守）
- `acc_archive.unique_biz_id` 必须包含 `RUN_ID`（首要清理口径）。
- `acc_archive.archive_code` 建议包含 `RUN_ID`（便于 UI/人工核对）。
- `arc_file_content.business_doc_no` 必须包含 `RUN_ID`（文件维度清理口径）。
- `arc_file_content.erp_voucher_no`/`archival_code`/`file_name` 可选包含 `RUN_ID`（兼容不同场景检索）。
- `acc_archive_relation`/`acc_archive_attachment` 不直接存 `RUN_ID`，通过 `archive_id`/`file_id` 追溯。
- `sys_audit_log.details` 或 `resource_id` 建议包含 `RUN_ID`（便于审计检索）。

## 3. 执行步骤总表

| 步骤 | 测试点 | 执行动作 | 验证内容 |
| --- | --- | --- | --- |
| 1 | 数据清理 | 物理删除所有测试数据 | 相关表清零，仅影响 `RUN_ID` |
| 2 | ERP 同步 | 触发 `VOUCHER_SYNC` | API 返回成功，日志可追溯 |
| 3 | 电子凭证池 | 进入凭证池/查库 | `arc_file_content` 有数据 |
| 4 | 凭证关联 | 进入档案列表/查库 | `acc_archive` 有数据，前端可见 |
| 5 | 创建原始凭证 | 创建发票/合同测试数据 | 前端可见、字段含 `RUN_ID` |
| 6 | 智能匹配 | 执行匹配 | 关联成功，有关系记录 |
| 7 | 四性检测 | 执行检测 | 真实性/完整性/可用性/安全性通过 |
| 8 | 提交归档 | 提交到审批流 | 状态为待审批 |
| 9 | 归档审批 | 审批通过 | 状态为已归档 |
| 10 | 会计档案查询 | 查询列表 | 记账凭证列表可见 |
| 11 | 全景视图 | 查看关联链 | 凭证可检索、链路完整 |
| 12 | 日志审计 | 审计查询 | 操作日志完整记录 |

## 4. 步骤细则（含证据）

### 1) 数据清理（物理删除）
- **动作**：以 `RUN_ID` 或 `T0` 为边界进行清理，先查后删。
- **验证**：删除后相关表 `count=0`，无越界删除。
- **证据**：SQL 执行记录 + 行数截图。
- **注意**：无法精确定位数据时，禁止执行删除。

### 2) ERP 同步
- **动作**：触发 `VOUCHER_SYNC`（UI 或 API）。
- **验证**：接口返回成功；后端日志记录同步开始/结束。
- **证据**：接口响应截图 + 日志条目。

### 3) 电子凭证池
- **动作**：进入凭证池页面或查询 `arc_file_content`。
- **验证**：存在 `T0` 后新增记录。
- **证据**：列表截图 + SQL 结果。

### 4) 凭证关联
- **动作**：进入档案列表或查询 `acc_archive`。
- **验证**：`acc_archive` 有新增记录，UI 列表可见。
- **证据**：列表截图 + SQL 结果。

### 5) 创建原始凭证
- **动作**：创建发票/合同等原始凭证（标题含 `RUN_ID`）。
- **验证**：前端列表可见；关键字段包含 `RUN_ID`。
- **证据**：详情页截图或接口返回。

### 6) 智能匹配
- **动作**：对目标凭证执行智能匹配。
- **验证**：匹配成功；关系表有记录。
- **证据**：匹配结果页面 + 关系表 SQL 结果。

### 7) 四性检测
- **动作**：对目标档案执行四性检测。
- **验证**：真实性/完整性/可用性/安全性均通过。
- **证据**：检测结果页面/报告截图。

### 8) 提交归档
- **动作**：提交归档流程。
- **验证**：状态变为待审批；流程实例生成。
- **证据**：流程列表截图 + 状态字段。

### 9) 归档审批
- **动作**：审批通过。
- **验证**：状态为已归档/已通过；流程结束。
- **证据**：审批记录截图 + 状态字段。

### 10) 会计档案查询
- **动作**：进入会计档案查询列表。
- **验证**：记账凭证列表可见（包含 `RUN_ID`）。
- **证据**：列表截图。

### 11) 全景视图
- **动作**：打开全景视图/关联链。
- **验证**：凭证可检索、链路完整。
- **证据**：全景视图截图。

### 12) 日志审计
- **动作**：查询审计日志（按操作人/时间）。
- **验证**：关键操作均有记录。
- **证据**：审计日志截图或 SQL 结果。

## 5. SQL 模板（按需替换 RUN_ID/T0）

> 字段依据 `docs/database/数据库设计.md` 与实际迁移脚本：`acc_archive` 使用 `unique_biz_id`，时间字段为 `created_time`；`arc_file_content` 时间字段为 `created_time`；`acc_archive_relation`/`acc_archive_attachment` 使用 `created_at`；`sys_audit_log` 使用 `username`/`user_id` + `created_time`。

```sql
-- 删除前计数 (优先用 unique_biz_id 或 archive_code 追踪 RUN_ID)
select count(*) from acc_archive
where unique_biz_id like '%RUN_ID%' or archive_code like '%RUN_ID%';

select count(*) from arc_file_content
where business_doc_no like '%RUN_ID%'
   or erp_voucher_no like '%RUN_ID%'
   or archival_code like '%RUN_ID%'
   or file_name like '%RUN_ID%';

-- 时间兜底 (created_time)
select count(*) from acc_archive where created_time >= 'T0';
select count(*) from arc_file_content where created_time >= 'T0';

-- 清理关系表 (acc_archive_relation: source_id/target_id)
delete from acc_archive_relation
where source_id in (
    select id from acc_archive where unique_biz_id like '%RUN_ID%' or archive_code like '%RUN_ID%'
)
or target_id in (
    select id from acc_archive where unique_biz_id like '%RUN_ID%' or archive_code like '%RUN_ID%'
);

delete from acc_archive_attachment where archive_id in (
  select id from acc_archive where unique_biz_id like '%RUN_ID%' or archive_code like '%RUN_ID%'
);

-- 清理主表
delete from acc_archive where unique_biz_id like '%RUN_ID%' or archive_code like '%RUN_ID%';
delete from arc_file_content
where business_doc_no like '%RUN_ID%'
   or erp_voucher_no like '%RUN_ID%'
   or archival_code like '%RUN_ID%'
   or file_name like '%RUN_ID%';

-- 审计日志（可选）
delete from sys_audit_log
-- username/user_id 可能不同，请替换为实际值
where (username = 'zhangsan' or user_id = 'zhangsan')
  and created_time >= 'T0';
```

```sql
-- 凭证池
select count(*) from arc_file_content where created_time >= 'T0';

-- 档案主表
select count(*) from acc_archive where created_time >= 'T0';

-- 关联关系
select count(*) from acc_archive_relation where created_at >= 'T0';
```
