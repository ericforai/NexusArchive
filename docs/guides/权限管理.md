# 权限管理指南

本文档详细介绍 NexusArchive 系统的用户、角色、权限管理功能。

---

## 用户管理

### 后端接口

需 `manage_users` 权限或 `system_admin` 角色：

| 操作 | 接口 | 说明 |
|------|------|------|
| 分页查询 | `GET /api/admin/users?page=1&limit=10&search=xx&status=active` | 支持搜索和状态筛选 |
| 创建用户 | `POST /api/admin/users` | 创建新用户 |
| 更新用户 | `PUT /api/admin/users/{id}` | 修改用户信息 |
| 删除用户 | `DELETE /api/admin/users/{id}` | 删除用户 |
| 重置密码 | `POST /api/admin/users/{id}/reset-password` | body: `{ "newPassword": "xxx" }` |
| 状态切换 | `PUT /api/admin/users/{id}/status` | body: `{ "status": "active\|disabled\|locked" }` |

### 前端操作

位置：`/system` → "系统设置"

- **创建用户**：填写基本信息、选择角色，提交后自动刷新用户列表。
- **用户管理**：分页列表、下拉修改状态、重置密码（弹框输入）、刷新。

---

## 角色管理

### 前端操作

位置：`/system` → "系统设置" → "角色管理"

- 创建/编辑/删除角色（系统角色禁删）
- 选择类别、类型、描述和权限
- 权限从 `/admin/permissions` 动态加载，分组复选

### 权限动态加载

角色管理优先读取 `sys_permission` 表（空表时回落默认权限）。

---

## 组织/部门管理

### 前端操作

位置：`/system` → "系统设置" → "组织/部门"

- 创建组织/部门
- 查看组织树
- 列表删除
- 排序（上下调 orderNum）
- 支持批量导入 JSON / CSV / Excel

### 批量导入

- 后端接口: `/admin/org/import`
- 支持格式: `.csv`, `.xls`, `.xlsx`
- 模板说明: `/admin/org/import/template`
- 校验: 必填字段（name）、类型/排序解析
- 返回: 成功/失败统计与错误行列表

### 排序

前端"上/下"按钮调整 orderNum，调用 `/admin/org/{id}/order`。

---

## 系统设置

位置：`/system` → "系统设置" → "系统设置"

可配置项：
- 系统名称
- 档号前缀
- 存储类型/路径
- 默认保管期限

对接接口: `/admin/settings`

## 表结构

组织/部门/权限/设置相关表结构已提供示例 DDL：

| 数据库 | 文件 |
|--------|------|
| PostgreSQL | `docs/database/auth_schema.sql` |
| MySQL | `docs/database/auth_schema_mysql.sql` |
| Kingbase | `docs/database/auth_schema_kingbase.sql` |
| 达梦 | `docs/database/auth_schema_dameng.sql` |

### 表说明

- `sys_org`：组织/部门（id/name/code/parent_id/type/order_num/逻辑删）
- `sys_role`：角色（code 唯一，role_category、permissions JSON 字符串、type system/custom）
- `sys_user`：用户（status active/disabled/locked 等）
- `sys_user_role`：用户-角色关联
- `sys_permission`：权限配置（键/分组/排序/启用标记）
- `sys_setting`：系统配置（键/值/描述/分类）

## 审计与安全

- **审计注解** `@ArchivalAudit` 应用于用户 CRUD、角色 CRUD、系统配置更新
- **日志落表** `sys_audit_log`（切面负责写入）
- **Token 黑名单/登录失败锁定**: 见 AuthService / LoginAttemptService / TokenBlacklistService
- **密码策略**: 创建/重置时校验长度≥8，需包含大小写字母、数字、特殊字符
- **数据域**: 档案列表非 system_admin 仅可查看本人创建的档案

---

## 前端登录与鉴权行为

- 进入 `/system` 时会自动调用 `/auth/me` 校验 token
- 无效/过期 token 将清除并回到登录页
- 任意接口返回 401 时自动跳转回 `/system` 登录
- 登出调用 `/auth/logout`，同时清理本地存储的 token/user
- 登录页增加加载态与错误提示
