# 签章和时间戳配置指南

## 📋 快速配置步骤

### 1. 生成密钥库

#### 方法一：使用脚本（推荐）

```bash
cd nexusarchive-java
bash scripts/generate_keystore.sh
```

**环境变量**（可选）：
```bash
export KEYSTORE_DIR=./keystore
export KEYSTORE_FILE=signature.p12
export KEYSTORE_PASSWORD=changeit
export CERT_ALIAS=signing-cert
export VALIDITY_DAYS=3650
bash scripts/generate_keystore.sh
```

#### 方法二：手动生成

```bash
# 使用 keytool 生成 PKCS12 格式的密钥库
keytool -genkeypair \
  -alias signing-cert \
  -keyalg RSA \
  -keysize 2048 \
  -sigalg SHA256withRSA \
  -validity 3650 \
  -keystore ./keystore/signature.p12 \
  -storetype PKCS12 \
  -storepass changeit \
  -keypass changeit \
  -dname "CN=NexusArchive Signing Certificate, OU=IT Department, O=NexusArchive, L=Beijing, ST=Beijing, C=CN"
```

#### 方法三：使用现有证书

如果您已有 PKCS12 格式的证书文件，直接复制到 `./keystore/signature.p12` 即可。

### 2. 配置 application.yml

在 `application.yml` 或 `application-dev.yml` 中添加：

```yaml
# 电子签章配置
signature:
  keystore:
    path: ./keystore/signature.p12  # 相对路径或绝对路径
    password: changeit
  auto-sign: true  # 是否在归档时自动签章
  default-cert-alias: signing-cert  # 默认证书别名

# 时间戳服务配置
timestamp:
  enabled: false  # 是否启用时间戳服务（需要配置TSA URL）
  tsa:
    url: http://tsa.example.com/tsp  # TSA服务器URL（可选）
    username: tsa_user  # TSA用户名（可选）
    password: tsa_password  # TSA密码（可选）
    timeout: 5000  # 请求超时时间（毫秒）
  fallback-on-error: true  # TSA不可用时是否使用本地时间
  auto-request: true  # 是否在归档时自动请求时间戳
```

### 3. 配置环境变量（可选）

```bash
# 签章配置
export SIGNATURE_KEYSTORE_PATH=./keystore/signature.p12
export SIGNATURE_KEYSTORE_PASSWORD=changeit
export SIGNATURE_AUTO_SIGN=true
export SIGNATURE_CERT_ALIAS=signing-cert

# 时间戳配置
export TIMESTAMP_ENABLED=false
export TSA_URL=http://tsa.example.com/tsp
export TSA_USERNAME=tsa_user
export TSA_PASSWORD=tsa_password
export TSA_TIMEOUT=5000
export TIMESTAMP_FALLBACK=true
export TIMESTAMP_AUTO_REQUEST=true
```

## 🔧 TSA 服务配置

### 公共 TSA 服务（测试用）

以下是一些可用的公共 TSA 服务（**仅用于测试**）：

1. **FreeTSA** (http://freetsa.org/tsr)
   - 免费，无需认证
   - 仅用于测试

2. **DigiCert TSA** (https://timestamp.digicert.com)
   - 商业服务，需要认证
   - 生产环境推荐

3. **自建 TSA 服务**
   - 使用 OpenSSL 或 BouncyCastle 搭建
   - 完全控制，适合私有化部署

### 配置示例

```yaml
# 使用 FreeTSA（测试）
timestamp:
  enabled: true
  tsa:
    url: http://freetsa.org/tsr
    timeout: 5000
  fallback-on-error: true

# 使用 DigiCert（生产）
timestamp:
  enabled: true
  tsa:
    url: https://timestamp.digicert.com
    username: your_username
    password: your_password
    timeout: 10000
  fallback-on-error: false  # 生产环境建议禁用降级
```

## 📝 集成到业务流程

签章和时间戳已自动集成到归档流程中：

1. **归档完成** → `ProcessingListener.handleCheckPassed()`
2. **发布事件** → `ArchiveCompletedEvent`
3. **自动签章** → `SignatureTimestampListener.performAutoSignature()`
4. **自动时间戳** → `SignatureTimestampListener.performAutoTimestamp()`

### 配置自动执行

在 `application.yml` 中：

```yaml
signature:
  auto-sign: true  # 启用自动签章

timestamp:
  auto-request: true  # 启用自动时间戳
```

### 手动调用

也可以通过 API 手动调用：

```bash
# 手动签章
curl -X POST http://localhost:8080/api/signature/sign \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "data=SGVsbG8gV29ybGQ=" \
  -d "certAlias=signing-cert" \
  -d "archiveId=ARCHIVE-001"

# 手动请求时间戳
curl -X POST http://localhost:8080/api/timestamp/request \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "data=SGVsbG8gV29ybGQ=" \
  -d "archiveId=ARCHIVE-001"
```

## ⚠️ 注意事项

### 1. 密钥库安全

- **生产环境**：密钥库密码必须使用强密码
- **存储位置**：密钥库文件应放在安全位置，设置适当的文件权限
- **备份**：定期备份密钥库文件

### 2. TSA 服务

- **测试环境**：可以使用公共 TSA 服务或禁用（使用降级模式）
- **生产环境**：必须使用可信的 TSA 服务
- **降级模式**：仅在测试环境启用，生产环境应禁用

### 3. 证书管理

- **有效期**：定期检查证书有效期
- **更新**：证书过期前及时更新
- **吊销**：证书被吊销后立即更换

## 🔍 故障排除

### 签章服务不可用

**问题**：`/api/signature/status` 返回 `available: false`

**解决方法**：
1. 检查密钥库路径是否正确
2. 检查密钥库密码是否正确
3. 检查密钥库文件是否存在
4. 检查证书别名是否正确
5. 查看日志中的错误信息

### 时间戳服务不可用

**问题**：`/api/timestamp/status` 返回 `available: false`

**解决方法**：
1. 检查 `timestamp.enabled` 是否为 `true`
2. 检查 `timestamp.tsa.url` 是否配置
3. 如果启用降级，系统会使用本地时间
4. 查看日志中的错误信息

### 自动签章/时间戳未执行

**问题**：归档完成后没有自动签章和时间戳

**解决方法**：
1. 检查 `signature.auto-sign` 是否为 `true`
2. 检查 `timestamp.auto-request` 是否为 `true`
3. 检查签章/时间戳服务是否可用
4. 查看日志中的相关信息
