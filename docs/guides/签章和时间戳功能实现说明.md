# 签章和时间戳功能实现说明

## 📋 概述

本文档说明签章和时间戳功能的实现情况以及如何启用和使用。

## ✅ 已实现的功能

### 1. 电子签章功能

#### 服务层
- ✅ `Sm2SignatureService` - SM2 国密签名服务实现
- ✅ `DigitalSignatureService` - 数字签名验证服务
- ✅ `SignatureAdapter` - 签章适配器接口

#### 控制器层
- ✅ `SignatureController` - 签章API控制器
  - `POST /api/signature/sign` - 对数据进行签名
  - `POST /api/signature/verify` - 验证签名
  - `POST /api/signature/verify-pdf` - 验证PDF文件签章
  - `POST /api/signature/verify-ofd` - 验证OFD文件签章
  - `GET /api/signature/logs` - 查询签章日志
  - `GET /api/signature/status` - 检查签章服务状态

#### 数据层
- ✅ `ArcSignatureLog` - 签章日志实体
- ✅ `ArcSignatureLogMapper` - 签章日志Mapper

### 2. 时间戳功能

#### 服务层
- ✅ `TimestampService` - 时间戳服务实现
  - 支持 RFC 3161 Time-Stamp Protocol (TSP)
  - 支持超时降级（使用本地时间）
  - 支持时间戳验证

#### 控制器层
- ✅ `TimestampController` - 时间戳API控制器
  - `POST /api/timestamp/request` - 请求时间戳
  - `POST /api/timestamp/verify` - 验证时间戳
  - `GET /api/timestamp/status` - 检查时间戳服务状态

### 3. 证书管理功能

#### 控制器层
- ✅ `CertificateController` - 证书管理API控制器
  - `GET /api/admin/certificates` - 查询所有证书
  - `GET /api/admin/certificates/{alias}` - 查询证书详情
  - `POST /api/admin/certificates/{alias}/verify` - 验证证书有效性

## 🔧 配置说明

### 签章服务配置

在 `application.yml` 或 `application.properties` 中配置：

```yaml
# 签章服务配置
signature:
  keystore:
    path: classpath:keystore/signature.p12  # 密钥库路径
    password: changeit  # 密钥库密码
```

### 时间戳服务配置

```yaml
# 时间戳服务配置
timestamp:
  enabled: true  # 是否启用时间戳服务
  tsa:
    url: http://tsa.example.com/tsp  # TSA服务器URL
    username: tsa_user  # TSA用户名（可选）
    password: tsa_password  # TSA密码（可选）
    timeout: 5000  # 请求超时时间（毫秒）
  fallback-on-error: true  # 错误时是否降级使用本地时间
```

## 🚀 使用示例

### 1. 签章数据

```bash
# 对数据进行签名
curl -X POST http://localhost:8080/api/signature/sign \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "data=SGVsbG8gV29ybGQ=" \
  -d "certAlias=signing-cert" \
  -d "archiveId=ARCHIVE-001"
```

### 2. 验证PDF签章

```bash
# 验证PDF文件签章
curl -X POST http://localhost:8080/api/signature/verify-pdf \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@document.pdf" \
  -d "archiveId=ARCHIVE-001"
```

### 3. 请求时间戳

```bash
# 请求时间戳
curl -X POST http://localhost:8080/api/timestamp/request \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "data=SGVsbG8gV29ybGQ=" \
  -d "archiveId=ARCHIVE-001"
```

### 4. 查询证书列表

```bash
# 查询所有证书
curl -X GET http://localhost:8080/api/admin/certificates \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 5. 验证证书

```bash
# 验证证书有效性
curl -X POST http://localhost:8080/api/admin/certificates/signing-cert/verify \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 📊 数据库表

### arc_signature_log 表

如果表不存在，需要创建：

```sql
CREATE TABLE IF NOT EXISTS arc_signature_log (
    id VARCHAR(64) PRIMARY KEY,
    archive_id VARCHAR(64),
    file_id VARCHAR(64),
    signer_name VARCHAR(255),
    signer_cert_sn VARCHAR(255),
    signer_org VARCHAR(255),
    sign_time TIMESTAMP,
    sign_algorithm VARCHAR(50),
    signature_value TEXT,
    verify_result VARCHAR(20),
    verify_time TIMESTAMP,
    verify_message TEXT,
    created_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_archive_id ON arc_signature_log(archive_id);
CREATE INDEX idx_file_id ON arc_signature_log(file_id);
CREATE INDEX idx_verify_result ON arc_signature_log(verify_result);
```

## ⚠️ 注意事项

### 1. 密钥库配置

- 密钥库文件需要放在 `classpath` 或指定路径
- 密钥库格式：PKCS12 (.p12)
- 确保密钥库密码正确

### 2. TSA 服务配置

- TSA URL 必须可访问
- 如果 TSA 不可用，系统会降级使用本地时间（如果 `fallback-on-error=true`）
- 建议在生产环境中使用可信的 TSA 服务

### 3. 证书管理

- 证书存储在密钥库中
- 证书过期检查通过 `checkValidity()` 方法
- 建议定期检查证书有效期

## 🔍 故障排除

### 签章服务不可用

1. 检查密钥库路径是否正确
2. 检查密钥库密码是否正确
3. 检查 BouncyCastle 提供者是否已注册
4. 查看日志中的错误信息

### 时间戳服务不可用

1. 检查 TSA URL 是否可访问
2. 检查网络连接
3. 检查超时设置
4. 如果启用降级，系统会使用本地时间

### 证书验证失败

1. 检查证书是否过期
2. 检查证书别名是否正确
3. 检查密钥库是否包含该证书
