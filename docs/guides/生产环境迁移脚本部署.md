# 生产环境迁移脚本部署指南

> 本指南说明如何将演示数据迁移脚本 `V71__rich_demo_data.sql` 部署到生产环境。

---

## 方法一：自动迁移（推荐）

### 原理
NexusArchive 使用 **Flyway** 进行数据库版本管理，应用启动时会自动检测并执行新的迁移脚本。

### 步骤

#### 1. 将迁移脚本放入正确位置
```bash
# 迁移脚本已在项目中：
nexusarchive-java/src/main/resources/db/migration/V71__rich_demo_data.sql
```

#### 2. 构建生产 JAR 包
```bash
cd nexusarchive-java
mvn clean package -DskipTests
```

#### 3. 上传到生产服务器
```bash
scp target/nexusarchive-backend-2.0.0.jar user@production-server:/opt/nexusarchive/
```

#### 4. 启动应用
```bash
ssh user@production-server
cd /opt/nexusarchive
java -jar nexusarchive-backend-2.0.0.jar
```

> **Flyway 会自动**：
> - 检测 `flyway_schema_history` 表
> - 发现 V71 尚未执行
> - 执行 `V71__rich_demo_data.sql`
> - 记录到历史表中

## 方法二：手动执行 SQL

> 适用于需要先预览或分批执行的场景。

### 步骤

#### 1. 连接生产数据库
```bash
# PostgreSQL
psql -h <数据库地址> -U <用户名> -d nexusarchive

# 或使用 Docker
docker exec -it nexusarchive-db psql -U postgres -d nexusarchive
```

#### 2. 手动执行迁移脚本
```sql
-- 在 psql 中执行
\i /path/to/V71__rich_demo_data.sql
```

#### 3. 记录到 Flyway 历史表
```sql
-- 重要：手动执行后必须记录，否则应用启动时会重复执行
INSERT INTO flyway_schema_history (
    installed_rank, version, description, type, script, 
    checksum, installed_by, installed_on, execution_time, success
) VALUES (
    (SELECT COALESCE(MAX(installed_rank), 0) + 1 FROM flyway_schema_history),
    '71', 'rich demo data', 'SQL', 'V71__rich_demo_data.sql',
    NULL, 'manual', NOW(), 0, true
);
```

## 方法三：使用 Flyway CLI

### 步骤

#### 1. 安装 Flyway CLI
```bash
brew install flyway  # macOS
# 或下载: https://flywaydb.org/download
```

#### 2. 创建配置文件 `flyway.conf`
```properties
flyway.url=jdbc:postgresql://localhost:5432/nexusarchive
flyway.user=postgres
flyway.password=your_password
flyway.locations=filesystem:/path/to/nexusarchive-java/src/main/resources/db/migration
```

#### 3. 执行迁移
```bash
# 预览待执行的迁移
flyway info

# 执行迁移
flyway migrate
```

## 注意事项

### ⚠️ 生产环境检查清单

| 检查项 | 说明 |
|--------|------|
| ✅ 备份数据库 | 执行前务必创建完整备份 |
| ✅ 测试环境验证 | 先在测试环境执行并验证 |
| ✅ 回滚脚本 | 准备好回滚 SQL（见下方） |
| ✅ 低峰期执行 | 建议在业务低峰期操作 |

### 🔙 回滚脚本

如需回滚演示数据，执行：
```sql
-- 删除演示数据
DELETE FROM acc_archive_attachment WHERE id LIKE 'attach-link%';
DELETE FROM arc_file_content WHERE id LIKE 'file-invoice%';
DELETE FROM sys_audit_log WHERE id LIKE 'auditlog%';
DELETE FROM audit_inspection_log WHERE id LIKE 'inspection%';
DELETE FROM biz_destruction WHERE id LIKE 'destruction%';
DELETE FROM biz_borrowing WHERE id LIKE 'borrow%';
DELETE FROM acc_archive_volume WHERE id LIKE 'volume%';
DELETE FROM acc_archive_relation WHERE id LIKE 'rel-%';
DELETE FROM arc_abnormal_voucher WHERE id LIKE 'abnormal%';
DELETE FROM bas_location WHERE id LIKE 'loc%';
DELETE FROM acc_archive WHERE id LIKE 'voucher%' OR id LIKE 'ledger%' 
    OR id LIKE 'report%' OR id LIKE 'other%' OR id LIKE 'pre%' OR id LIKE 'bank%';
DELETE FROM sys_user WHERE id LIKE 'user-%';
DELETE FROM bas_fonds WHERE id LIKE 'fonds-%';

-- 从 Flyway 历史表移除记录
DELETE FROM flyway_schema_history WHERE version = '71';
```
