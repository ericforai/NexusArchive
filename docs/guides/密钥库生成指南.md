# 密钥库生成指南

## 📋 概述

本文档提供多种方法生成 PKCS12 格式的密钥库，用于电子签章功能。

## 🔍 方法一：使用生成脚本（推荐）

### 前提条件

- Java JDK 已安装
- `keytool` 命令可用

### 检查环境

```bash
# 检查 Java 版本
java -version

# 检查 keytool
keytool -help
```

### 执行步骤

```bash
cd nexusarchive-java
bash scripts/generate_keystore.sh
```

### 自定义配置

```bash
# 使用环境变量自定义
export KEYSTORE_DIR=./keystore
export KEYSTORE_FILE=signature.p12
export KEYSTORE_PASSWORD=your_strong_password
export CERT_ALIAS=signing-cert
export VALIDITY_DAYS=3650  # 10年

bash scripts/generate_keystore.sh
```

### 输出

脚本会在 `./keystore/signature.p12` 生成密钥库文件。

## 🔧 方法二：手动使用 keytool

### 基本命令

```bash
# 创建目录
mkdir -p nexusarchive-java/keystore

# 生成密钥库
keytool -genkeypair \
  -alias signing-cert \
  -keyalg RSA \
  -keysize 2048 \
  -sigalg SHA256withRSA \
  -validity 3650 \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storetype PKCS12 \
  -storepass changeit \
  -keypass changeit \
  -dname "CN=NexusArchive Signing Certificate, OU=IT Department, O=NexusArchive, L=Beijing, ST=Beijing, C=CN" \
  -ext "KeyUsage=digitalSignature,nonRepudiation" \
  -ext "ExtendedKeyUsage=codeSigning,emailProtection"
```

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `-alias` | 证书别名 | `signing-cert` |
| `-keyalg` | 密钥算法 | `RSA`（或 `EC` 用于 SM2） |
| `-keysize` | 密钥长度 | `2048`（RSA）或 `256`（EC） |
| `-sigalg` | 签名算法 | `SHA256withRSA` |
| `-validity` | 有效期（天） | `3650`（10年） |
| `-keystore` | 密钥库路径 | `./keystore/signature.p12` |
| `-storetype` | 密钥库类型 | `PKCS12` |
| `-storepass` | 密钥库密码 | `changeit` |
| `-keypass` | 私钥密码 | `changeit` |
| `-dname` | 证书主题 | `CN=..., OU=..., O=...` |

### 交互式生成

如果不想在命令行中指定所有参数，可以交互式生成：

```bash
keytool -genkeypair \
  -alias signing-cert \
  -keyalg RSA \
  -keysize 2048 \
  -validity 3650 \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storetype PKCS12
```

系统会提示输入：
- 密钥库密码
- 私钥密码
- 证书信息（姓名、组织等）

## 🔐 方法三：使用现有证书

### 如果您已有证书文件

#### 3.1 已有 PKCS12 文件

直接复制到目标位置：

```bash
cp your_existing_cert.p12 nexusarchive-java/keystore/signature.p12
```

#### 3.2 已有 PEM 格式证书

转换为 PKCS12：

```bash
# 如果有私钥和证书文件
openssl pkcs12 -export \
  -in certificate.pem \
  -inkey private_key.pem \
  -out nexusarchive-java/keystore/signature.p12 \
  -name signing-cert \
  -passout pass:changeit
```

#### 3.3 已有 JKS 格式密钥库

转换为 PKCS12：

```bash
keytool -importkeystore \
  -srckeystore your_keystore.jks \
  -srcstoretype JKS \
  -srcstorepass your_password \
  -destkeystore nexusarchive-java/keystore/signature.p12 \
  -deststoretype PKCS12 \
  -deststorepass changeit
```

## 🛠️ 方法四：使用 OpenSSL（适用于 SM2）

### 生成 SM2 密钥对

```bash
# 生成 SM2 私钥
openssl ecparam -genkey -name SM2 -out sm2_private_key.pem

# 生成证书请求
openssl req -new -key sm2_private_key.pem -out cert_request.csr

# 自签名证书
openssl x509 -req -days 3650 -in cert_request.csr -signkey sm2_private_key.pem -out certificate.pem

# 转换为 PKCS12
openssl pkcs12 -export \
  -in certificate.pem \
  -inkey sm2_private_key.pem \
  -out nexusarchive-java/keystore/signature.p12 \
  -name signing-cert \
  -passout pass:changeit
```

## ✅ 验证密钥库

### 查看密钥库信息

```bash
keytool -list -v \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storepass changeit
```

### 查看证书详情

```bash
keytool -list -v \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storepass changeit \
  -alias signing-cert
```

### 验证证书有效期

```bash
keytool -list -v \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storepass changeit \
  -alias signing-cert | grep -E "(Valid from|Valid until)"
```

## 🔒 安全建议

### 生产环境

1. **使用强密码**
   ```bash
   # 生成强密码（至少16位，包含大小写字母、数字、特殊字符）
   export KEYSTORE_PASSWORD="YourStrongPassword123!@#"
   ```

2. **设置文件权限**
   ```bash
   chmod 600 nexusarchive-java/keystore/signature.p12
   ```

3. **安全存储**
   - 不要将密钥库提交到版本控制系统
   - 使用安全的备份策略
   - 限制访问权限

4. **定期更新**
   - 在证书过期前更新
   - 监控证书有效期

## 🐛 常见问题

### 问题1：找不到 keytool

**错误**：
```
keytool: command not found
```

**解决方法**：
1. 确保已安装 Java JDK（不是 JRE）
2. 检查 `JAVA_HOME` 环境变量
3. 使用完整路径：`$JAVA_HOME/bin/keytool`

### 问题2：权限被拒绝

**错误**：
```
Permission denied
```

**解决方法**：
```bash
chmod +x scripts/generate_keystore.sh
```

### 问题3：密钥库已存在

**错误**：
```
Keystore file exists
```

**解决方法**：
- 删除旧文件：`rm nexusarchive-java/keystore/signature.p12`
- 或使用不同的文件名

### 问题4：密码错误

**错误**：
```
Keystore was tampered with, or password was incorrect
```

**解决方法**：
- 检查密码是否正确
- 如果忘记密码，需要重新生成密钥库

## 📝 快速生成脚本（简化版）

如果脚本无法运行，可以使用以下简化命令：

```bash
# 创建目录
mkdir -p nexusarchive-java/keystore

# 生成密钥库（交互式）
keytool -genkeypair \
  -alias signing-cert \
  -keyalg RSA \
  -keysize 2048 \
  -validity 3650 \
  -keystore nexusarchive-java/keystore/signature.p12 \
  -storetype PKCS12
```

按提示输入：
- 密钥库密码：`changeit`（或自定义）
- 私钥密码：与密钥库密码相同
- 姓名：`NexusArchive Signing Certificate`
- 组织单位：`IT Department`
- 组织：`NexusArchive`
- 城市：`Beijing`
- 省份：`Beijing`
- 国家代码：`CN`
