## 2. 代码结构速览

### 2.1 前端

- 路由入口：`src/routes/index.tsx`
- 页面层：`src/pages/**`
- 业务模块入口：`src/features/<module>/index.ts`
- UI 组件：`src/components/**`
- 通用能力：`src/api/`, `src/store/`, `src/utils/`, `src/hooks/`, `src/types.ts`

**Settings 模块示例（DDD-ish）**

- 入口：`src/features/settings/index.ts`
- 领域层：`src/features/settings/domain/`
- 应用层：`src/features/settings/application/`
- 基础设施：`src/features/settings/infrastructure/`
- 页面容器：`src/pages/settings/`
- 展示组件：`src/components/settings/`

### 2.2 后端

- 后端工程根：`nexusarchive-java/`
- 模块化根：`nexusarchive-java/src/main/java/com/nexusarchive/modules/`
- Borrowing 模块：`.../modules/borrowing/{api,app,domain,infra}`
- 传统层：`service/`, `controller/`, `mapper/`, `entity/`（逐步收敛中）

---

## 3. 模块边界与守卫规则（必须遵守）

### 前端

- 唯一入口：`src/features/<module>/index.ts`
- `pages/**` 只能 import `features/<module>` 入口（禁止深路径）。
- `components/**` 禁止依赖 `features/pages/api/store`。
- `routes/**` 仅允许引入 `pages` + `layouts/common/auth`。
- 规则配置：`.eslintrc.cjs`（eslint-plugin-boundaries）

### 后端

- Borrowing 对外仅允许依赖：
  - `com.nexusarchive.modules.borrowing.app..`
  - `com.nexusarchive.modules.borrowing.api.dto..`
- 禁止外部访问：
  - `com.nexusarchive.modules.borrowing.domain..`
  - `com.nexusarchive.modules.borrowing.infra..`
- 守卫规则：`nexusarchive-java/src/test/java/com/nexusarchive/architecture/ModuleBoundaryTest.java`

## 4. 开发规范（新人必看）

- **文档规则**：目录结构变化必须更新对应目录 `README.md`；重要变更需更新根 `README.md` 或 `docs/CHANGELOG.md`。
- **文件头注释**：源码与关键配置保持 `Input/Output/Pos` 三行注释。
- **模块化约束**：入口导出 + 边界测试双重约束，禁止绕过。

## 5. 本地运行与验证

```bash
# 前端
npm install
npm run dev

# 后端
cd nexusarchive-java
mvn clean package -DskipTests
java -jar target/nexusarchive-backend-2.0.0.jar
```

**常见问题**

- ESLint v9 需要 `eslint.config.*`，当前为 `.eslintrc.cjs`，可能需要迁移或使用 legacy 模式。
- 后端集成测试依赖本地服务（`localhost:8080`），未启动会报连接拒绝。
