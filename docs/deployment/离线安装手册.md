# NexusArchive 电子会计档案系统 - 离线部署与交付手册 (V2.0)

本手册详细说明了 NexusArchive 系统的离线环境部署流程，涵盖安装包制作、服务器环境准备、系统安装、**License授权**以及常见故障排除。

---

## 1. 部署前准备 (Prerequisites)

### 1.1 硬件要求
*   **CPU**: 最低 4 Core (推荐 8 Core+ 以支持 OCR/转换服务)
*   **内存**: 最低 8 GB (推荐 16 GB+)
*   **磁盘**: 系统盘 50GB+, 数据盘根据档案量规划 (建议 SSD)

### 1.2 软件环境 (信创适配)
*   **操作系统**:
    *   Ubuntu 20.04/22.04/24.04 LTS
    *   CentOS 7.9 / RedHat 8+
    *   麒麟 Kylin V10 SP2+ / 统信 UOS 20+
*   **数据库**:
    *   PostgreSQL 12+ (默认推荐)
    *   达梦 DM8 (信创推荐)
    *   人大金仓 KingbaseES V8
*   **运行环境**:
    *   JDK 17 (安装包内置，自动适配 x86_64/aarch64)

### 1.3 网络要求
*   **入站**: TCP 80 (Web访问), TCP 8080 (后端API, 可选), SSH 22
*   **出站**: 无需外网连接 (完全离线支持)

## 2. 制作离线安装包 (开发侧)

在交付给客户前，需在开发环境构建完整的离线包。

**构建命令**:
```bash
cd deploy
./build_offline_package.sh
```

**产物**:
生成的文件位于 `deploy/target/nexusarchive-offline-installer-[VERSION].tar.gz`。包含：
*   后端 JAR 包
*   前端静态资源 (Dist)
*   JDK 运行环境 (可选打包)
*   安装脚本 (`install.sh`)
*   配置文件模板

---

## 3. 服务器安装步骤 (交付侧)

### 3.1 上传与解压
将 `.tar.gz` 安装包上传至服务器 `/opt` 或 `/tmp` 目录。

```bash
# 1. 解压 (建议使用 sudo 避免权限问题)
tar -xzf nexusarchive-offline-installer-2.0.0.tar.gz

# 2. 进入目录
cd nexusarchive-offline-installer-2.0.0
```

### 3.2 运行安装脚本
脚本会自动检测环境、创建用户、配置服务。

```bash
# 必须使用 sudo 或 root 运行
sudo ./install.sh
```

**交互式配置向导**:
1.  **安装目录**: 默认 `/opt/nexusarchive` (回车确认)
2.  **服务端口**: 默认 `8080`
3.  **数据库类型**: 选择 `postgresql`, `dameng`, 或 `kingbase`
4.  **数据库连接**: 输入 Host, Port, DB Name, User, Password
    *   *注意: 确保数据库已提前创建并允许连接*

### 3.3 验证服务状态
安装完成后，服务会自动启动。

```bash
# 查看服务状态
systemctl status nexusarchive

# 查看实时日志
journalctl -u nexusarchive -f
```

如果看到 "Started NexusArchive Service" 且无错误日志，说明启动成功。

## 4. License 授权 (关键步骤)

本系统启用了严格的 License 验证。**默认安装后，未授权状态下无法登录系统。**

### 4.1 生成 License (仅限授权人员)
使用项目源码中的生成工具为客户签发证书：
*   **工具路径**: `nexusarchive-java/src/main/java/com/nexusarchive/tool/LicenseGenerator.java`
*   **操作**:
    1.  填入客户信息 (到期时间, 用户数, 节点数).
    2.  运行 `main` 方法.
    3.  复制生成的 JSON 字符串.

### 4.2 导入 License (客户端)
1.  访问系统登录页 (如 `http://192.168.1.100`).
2.  使用 `admin` 账号登录.
    *   *初始状态可能允许管理员进入 License 配置页，或通过后端 API直接导入.*
3.  由于目前处于强制验证期，若无法登录，请联系运维人员通过 **API 后门** 或 **数据库直接导入** (紧急模式)。

**授权方式**:
1.  登录系统，进入 **系统管理 -> 系统设置**。
2.  找到 "License 授权管理" 面板。
3.  将生成的 JSON 字符串粘贴到输入框并点击 "验证并导入"。

*(注: V2.0 版本已内置 License 管理界面，无需后台手动操作)*

## 5. 常见故障排除 (Troubleshooting)

### 5.1 前端 403 Forbidden / 500 Internal Server Error
*   **现象**: 访问页面提示 Nginx 500 或 403。
*   **原因**: `/opt/nexusarchive/frontend` 目录权限不正确，Nginx 用户 (`www-data` 或 `nginx`) 无法读取。
*   **解决**:
    ```bash
    sudo chmod -R 755 /opt/nexusarchive
    sudo chown -R nexus:nexus /opt/nexusarchive
    # 确保 Nginx 能读取 (给予其它用户读权限)
    sudo chmod -R o+r /opt/nexusarchive/frontend
    ```

### 5.2 数据库认证失败 (Fatals: password authentication failed)
*   **现象**: `systemctl status` 显示启动失败，日志报错 `password authentication failed`。
*   **原因**: 安装时输入的密码有误，或包含特殊字符被 Shell 转义。
*   **解决**:
    1.  修改配置文件: `sudo vim /opt/nexusarchive/.env`
    2.  更新 `DB_PASSWORD`。
    3.  重启服务: `sudo systemctl restart nexusarchive`

### 5.3 无法登录 (License 限制)
*   **现象**: API 返回 `403 Forbidden`, 提示 "License 未加载" 或 "已过期"。
*   **解决**: 参考第 4 节，使用 `LicenseGenerator` 签发新证书，并通过 API `/api/license/load` (需持有 Token) 或联系后台导入。

### 5.4 统计数据不显示 (显示为 0)
*   **现象**: 仪表盘连接数、接收数为 0。
*   **原因**: 正常现象。新部署系统尚未接入 ERP，确实无数据。
*   **验证**: 检查 `/api/erp/config/stats` 接口返回 200 OK 即可。
