# 故障排除指南

本文档记录常见问题的诊断和解决方案，帮助快速定位和修复问题。

---

## 1. 登录后菜单和数据全部丢失 (401 Unauthorized)

### 症状
- 登录成功后，页面显示"工作台概览"但所有数据显示 `--`
- 控制台出现大量 `401 Unauthorized` 错误
- 所有 API 请求（如 `/user/permissions`、`/stats/dashboard`）都返回 401

### 诊断步骤
1. 检查后端日志：
   ```bash
   grep -i "jwt\|auth\|error" /tmp/backend.log | tail -30
   ```

2. 如果看到 **"Unable to connect to Redis"**，说明是 Redis 服务问题

### 根本原因
`TokenBlacklistService` 使用 Redis 存储 token 黑名单。当 Redis 不可用时，`isBlacklisted()` 方法抛出异常，导致 JWT 验证失败。

### 解决方案
**方案1：启动 Redis**
```bash
# macOS
brew services start redis

# 验证
redis-cli ping
# 应返回: PONG
```

**方案2：代码优雅降级（已实施）**
修改 `TokenBlacklistService.java`，在 Redis 不可用时返回 `false` 而不是抛出异常：
```java
public boolean isBlacklisted(String token) {
    try {
        return Boolean.TRUE.equals(redisTemplate.hasKey(getBlacklistKey(token)));
    } catch (Exception e) {
        log.warn("Redis unavailable, assuming token is valid: {}", e.getMessage());
        return false;  // 降级处理
    }
}
```

## 2. 登录后立即闪退回登录页

### 症状
- 输入用户名密码后登录，界面短暂显示后又跳回登录页
- 形成登录循环

### 可能原因

#### 2.1 前端 localStorage 被阻止
**诊断**：控制台出现 `"Access to storage is not allowed from this context"`

**解决方案**：
- 检查浏览器是否处于隐私模式
- 检查是否有扩展程序阻止 localStorage
- 使用 `safeStorage` 工具类替代直接调用 `localStorage`

#### 2.2 axios 401 拦截器过于激进
**诊断**：登录后立即有 401 请求触发重定向

**解决方案**：
修改 `src/api/client.ts`，401 拦截器不再自动删除 token 和重定向：
```typescript
if (error.response?.status === 401) {
    console.warn('[Axios] 401 for:', error.config?.url);
    // 不再自动删除 token 和重定向
}
```

#### 2.3 后端 /user/permissions 返回格式错误
**诊断**：前端期望 `{ permissions: [], roles: [] }` 但后端返回 `List<String>`

**解决方案**：
修改 `UserController.java` 返回正确格式：
```java
@GetMapping("/permissions")
public Result<Map<String, Object>> getPermissions() {
    Map<String, Object> result = new HashMap<>();
    result.put("permissions", permissions);  // List<String>
    result.put("roles", roles);              // List<Map>
    return Result.success(result);
}
```

## 3. 数据库迁移失败 (Flyway)

### 症状
- 后端启动失败
- 日志显示 `FlywayValidateException` 或 `Migration failed`

### 常见原因

#### 3.1 Checksum 不匹配
**诊断**：日志显示 `checksum mismatch`

**解决方案**：
```bash
PGPASSWORD=postgres psql -h localhost -U postgres -d nexusarchive \
  -c "DELETE FROM flyway_schema_history WHERE version = 'XX';"
```

#### 3.2 外键约束冲突
**诊断**：日志显示 `violates foreign key constraint`

**解决方案**：
- 使用 `ON CONFLICT` 语法处理冲突
- 使用 PL/pgSQL 条件逻辑更新而非删除重建

## 4. 快速检查清单

遇到问题时按此顺序检查：

- [ ] **Redis 运行中？** `redis-cli ping`
- [ ] **后端运行中？** `lsof -ti:8080`
- [ ] **前端运行中？** `lsof -ti:5173`
- [ ] **数据库连接正常？** 检查 `application.yml` 配置
- [ ] **后端日志有错误？** `tail -50 /tmp/backend.log`
- [ ] **浏览器控制台有错误？** F12 检查 Network 和 Console

## 5. 常用调试命令

```bash
# 登录测试
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin", "password":"admin123"}'

# 权限接口测试（需先获取 token）
TOKEN="your_token_here"
curl http://localhost:8080/api/user/permissions \
  -H "Authorization: Bearer $TOKEN"

# 重启后端
lsof -ti:8080 | xargs kill -9 2>/dev/null
cd nexusarchive-java && mvn -Dspring-boot.run.profiles=dev spring-boot:run

# 清理 Flyway 记录
PGPASSWORD=postgres psql -h localhost -U postgres -d nexusarchive \
  -c "DELETE FROM flyway_schema_history WHERE success = false;"
```
