# NexusArchive 全栈启动指南

本文档指导开发人员在本地完整启动 NexusArchive 的前后端及数据库环境。

## 1. 环境准备

确保本地已安装以下基础环境：
- **JDK**: OpenJDK 17+
- **Maven**: 3.8+
- **Node.js**: 18.0+ (推荐使用 nvm 管理)
- **Database**: PostgreSQL 14+ (推荐使用 Docker 或本地安装)
- **Python**: 3.8+ (可选，用于辅助脚本)

## 2. 数据库配置

1.  **创建数据库**:
    ```bash
    createdb nexusarchive
    ```
2.  **配置连接**:
    修改 `nexusarchive-java/src/main/resources/application.yml` (或 `application-dev.yml`):
    ```yaml
    spring:
      datasource:
        url: jdbc:postgresql://localhost:5432/nexusarchive
        username: your_username
        password: your_password
    ```
3.  **数据库迁移**:
    系统启动时会自动运行 `Flyway` 脚本。无需手动建表。

## 3. License 生成 (首次启动必须)

系统启用 License 校验，首次启动需生成并配置 License。

1.  **运行生成脚本**:
    ```bash
    # 在项目根目录
    python3 generate_license.py
    ```
2.  **应用配置**:
    - 脚本会生成 `new_license.json` 和 `new_license_pub.txt`。
    - 将 `new_license_pub.txt` 中的内容复制到 `application.yml` 的 `license.public-key` 字段。
    - 将 `new_license.json` 放置在后端运行目录（如 `nexusarchive-java/`）。

## 4. 启动后端 (Backend)

进入后端目录并启动 Spring Boot 应用：

```bash
cd nexusarchive-java
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

*   **API 文档**: 启动成功后访问 `http://localhost:19090/api/swagger-ui.html`
*   **健康检查**: `http://localhost:19090/api/actuator/health`

## 5. 启动前端 (Frontend)

进入前端目录安装依赖并启动：

```bash
# 1. 安装依赖
npm install

# 2. 启动开发服务器
npm run dev
```

*   **访问地址**: `http://localhost:15175`
*   **登录账号**: `admin` / `admin123` (默认初始化账号)

## 6. 快捷启动脚本

项目根目录提供了快捷脚本，可一键启动开发环境（需安装 `tmux` 或分屏终端支持）：

```bash
# Linux/Mac
./scripts/restart-services.sh
```

## 7. 常见问题

### Q: 启动时报 "License invalid"？
A: 请确保已运行 `generate_license.py` 并正确配置了公钥。详细请参考 `check_key_match.py` 进行密钥对校验。

### Q: 数据库连接失败？
A: 检查 PostgreSQL 服务是否运行，以及 `application.yml` 中的用户名密码是否正确。

### Q: 依赖下载慢？
A: 前端建议配置 taobao 镜像，后端建议配置阿里云 Maven 镜像。

```bash
npm install
```

或者使用 yarn:

```bash
yarn install
```

### 2. 配置环境变量（可选）

如果需要使用 AI 功能（如 OCR），需要配置 Gemini API Key：

1. 创建 `.env.local` 文件（已自动创建）
2. 在文件中添加你的 API Key：
   ```
   GEMINI_API_KEY=your_api_key_here
   ```

**注意**：如果没有 API Key，系统仍可正常运行，只是部分 AI 功能不可用。

### 3. 启动开发服务器

```bash
npm run dev
```

或者使用 yarn:

```bash
yarn dev
```

### 4. 访问系统

启动成功后，在浏览器中打开：

```
http://localhost:15175
```

## 功能说明

### 预归档库功能

1. **电子凭证池** - 查看和管理来自各系统的凭证
2. **OCR识别** - 批量识别和处理文档
3. **凭证关联** - 配置关联规则，实现自动/手动关联
   - 支持规则的增删改查
   - 支持权重调整
   - 支持一对多/多对多关联
   - 支持关联关系可视化

### 系统设置

1. **定时任务管理** - 配置自动执行的任务
   - 支持每天/每周/每月/间隔执行
   - 查看执行历史
   - 手动触发执行

2. **消息提醒规则** - 配置系统通知规则
   - 任务完成提醒
   - 任务失败提醒
   - 匹配阈值提醒
   - 合规预警

### 数据存储

系统使用 localStorage 存储以下数据：
- 关联规则配置
- 定时任务配置
- 消息提醒规则
- 通知历史
- 任务执行历史

**注意**：刷新浏览器不会丢失数据，但清除浏览器缓存会清除所有本地数据。

## 常见问题

### 端口被占用

如果 15175 端口被占用，可以修改 `vite.config.ts` 中的端口配置：

```typescript
server: {
  port: 3001, // 改为其他端口（如 3001, 3002 等）
  host: '0.0.0.0',
}
```

### 依赖安装失败

如果遇到依赖安装问题，可以尝试：

```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```
