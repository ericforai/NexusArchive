# NexusArchive 本地启动指南

## 前置要求

- Node.js 18.0 或更高版本
- npm 或 yarn 包管理器

## 快速启动

### 1. 安装依赖

```bash
npm install
```

或者使用 yarn:

```bash
yarn install
```

### 2. 配置环境变量（可选）

如果需要使用 AI 功能（如 OCR），需要配置 Gemini API Key：

1. 创建 `.env.local` 文件（已自动创建）
2. 在文件中添加你的 API Key：
   ```
   GEMINI_API_KEY=your_api_key_here
   ```

**注意**：如果没有 API Key，系统仍可正常运行，只是部分 AI 功能不可用。

### 3. 启动开发服务器

```bash
npm run dev
```

或者使用 yarn:

```bash
yarn dev
```

### 4. 访问系统

启动成功后，在浏览器中打开：

```
http://localhost:5173
```

## 可用命令

- `npm run dev` - 启动开发服务器（端口 5173）
- `npm run build` - 构建生产版本
- `npm run preview` - 预览生产构建

## 功能说明

### 预归档库功能

1. **电子凭证池** - 查看和管理来自各系统的凭证
2. **OCR识别** - 批量识别和处理文档
3. **凭证关联** - 配置关联规则，实现自动/手动关联
   - 支持规则的增删改查
   - 支持权重调整
   - 支持一对多/多对多关联
   - 支持关联关系可视化

### 系统设置

1. **定时任务管理** - 配置自动执行的任务
   - 支持每天/每周/每月/间隔执行
   - 查看执行历史
   - 手动触发执行

2. **消息提醒规则** - 配置系统通知规则
   - 任务完成提醒
   - 任务失败提醒
   - 匹配阈值提醒
   - 合规预警

### 数据存储

系统使用 localStorage 存储以下数据：
- 关联规则配置
- 定时任务配置
- 消息提醒规则
- 通知历史
- 任务执行历史

**注意**：刷新浏览器不会丢失数据，但清除浏览器缓存会清除所有本地数据。

## 常见问题

### 端口被占用

如果 5173 端口被占用，可以修改 `vite.config.ts` 中的端口配置：

```typescript
server: {
  port: 3001, // 改为其他端口（如 3001, 3002 等）
  host: '0.0.0.0',
}
```

### 依赖安装失败

如果遇到依赖安装问题，可以尝试：

```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 浏览器兼容性

推荐使用以下浏览器：
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## 开发说明

- 项目使用 React 19 + TypeScript + Vite
- UI 框架：Tailwind CSS（通过 CDN）
- 图标库：lucide-react
- 图表库：recharts

## 技术支持

如有问题，请检查：
1. Node.js 版本是否符合要求
2. 依赖是否正确安装
3. 端口是否被占用
4. 浏览器控制台是否有错误信息
