# NexusArchive 全栈启动指南

本文档指导开发人员在本地完整启动 NexusArchive 开发环境。

---

## 🚀 推荐方式：Docker 一键启动

> **无需安装 JDK/Maven/PostgreSQL，只需 Docker Desktop**

```bash
# 启动全栈服务
./scripts/dev-start.sh

# 停止服务
./scripts/dev-stop.sh

# 查看日志
./scripts/dev-logs.sh
```

### 访问地址

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:15175 |
| 后端 API | http://localhost:19090/api |
| API 健康检查 | http://localhost:19090/api/health |
| 数据库 | localhost:54321 (PostgreSQL) |
| Redis | localhost:16379 |

### 默认账号

- **管理员**: `admin` / `admin123`

---

## 备选方式：手动启动

> 适用于需要本地调试、修改代码热重载的场景

### 1. 环境准备

确保本地已安装：
- **JDK**: OpenJDK 17+
- **Maven**: 3.8+
- **Node.js**: 18.0+
- **Docker**: 用于运行数据库和 Redis

### 2. 启动依赖服务

```bash
docker-compose -f docker-compose.dev.yml up -d nexus-db nexus-redis
```

### 3. 启动后端

```bash
cd nexusarchive-java
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

等待看到：`Started NexusArchiveApplication in X.XXX seconds`

### 4. 启动前端

```bash
npm install  # 首次或依赖更新时
npm run dev
```

## License 配置（首次启动）

系统启用 License 校验，首次启动需生成：

```bash
python3 generate_license.py
```

将生成的公钥配置到 `application.yml` 的 `license.public-key` 字段。

## 常见问题

### Q: 前端报 500 错误？

**原因**：后端未启动或未就绪  
**解决**：检查后端健康接口
```bash
curl -s http://localhost:19090/api/health
```

### Q: Docker 容器启动失败？

**解决**：查看详细日志
```bash
docker-compose -f docker-compose.dev.yml logs nexus-backend
```

### Q: 端口被占用？

```bash
lsof -ti :19090 | xargs kill -9
lsof -ti :15175 | xargs kill -9
```

### Q: 数据库连接失败？

确认 Docker 服务正在运行：
```bash
docker-compose -f docker-compose.dev.yml ps
```
