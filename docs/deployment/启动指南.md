# NexusArchive 启动指南

本文档指导开发人员在本地完整启动 NexusArchive 开发环境。

**更新日期**: 2025-01-05

---

## 🚀 一键启动（推荐）

> **无需安装 JDK/Maven/PostgreSQL，只需 Node.js 和 Docker Desktop**

```bash
npm run dev
```

自动完成：
- ✅ 启动 PostgreSQL + Redis（Docker）
- ✅ 导入 seed data（首次或 volume 为空时）
- ✅ 启动后端（本地）
- ✅ 启动前端（本地）

### 访问地址

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:15175 |
| 后端 API | http://localhost:19090/api |
| API 文档 | http://localhost:19090/api/swagger-ui.html |
| 数据库 | localhost:54321 |
| Redis | localhost:16379 |

### 默认账号

- **管理员**: `admin` / `admin123`

### 停止服务

```bash
npm run dev:stop
```

## 🔄 数据同步（多台 Mac）

### 工作流程

```bash
# 离开公司前 - 导出数据
npm run db:dump
git add db/seed-data.sql
git commit -m "db: update seed data"
git push

# 回到家后 - 拉取并导入
git pull
npm run db:load
```

### 重置数据库

```bash
npm run db:reset
```

> **警告**: 这会删除所有数据并重新初始化！

## 🔧 手动启动（高级）

> 适用于需要本地调试、修改代码热重载的场景

### 1. 环境准备

确保本地已安装：
- **Node.js**: 18.0+
- **Docker Desktop**: 最新版
- **JDK**: OpenJDK 17+（仅手动启动后端时需要）
- **Maven**: 3.8+（仅手动启动后端时需要）

### 2. 启动基础设施

```bash
# 仅启动 DB + Redis
docker-compose -f docker-compose.infra.yml up -d
```

### 3. 启动后端（可选）

```bash
cd nexusarchive-java
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

等待看到：`Started NexusArchiveApplication in X.XXX seconds`

### 4. 启动前端（可选）

```bash
npm run dev:vite
```

## 📋 常用命令

```bash
# 开发环境
npm run dev        # 启动开发环境

# 数据管理
npm run db:dump    # 导出数据

# 前端单独
npm run dev:vite   # 仅启动前端
npm run build      # 生产构建

# 后端单独
cd nexusarchive-java
mvn spring-boot:run  # 启动后端
```

## 🛠️ License 配置（首次启动）

系统启用 License 校验，首次启动需生成：

```bash
python3 tools/generate_license.py
```

将生成的公钥配置到 `application.yml` 的 `license.public-key` 字段。

## ❓ 常见问题

### Q: 前端报 500 错误？

**原因**: 后端未启动或未就绪
**解决**:
```bash
curl http://localhost:19090/api/health
```

### Q: Docker 容器启动失败？

**解决**:
```bash
# 查看容器状态
docker ps | grep nexus

# 查看日志
docker logs nexus-db
```

### Q: 端口被占用？

```bash
# 查看端口占用
lsof -ti :19090
lsof -ti :15175

# 杀掉进程
lsof -ti :PORT | xargs kill -9
```

### Q: 数据库连接失败？

确认 Docker 容器状态：
```bash
docker-compose -f docker-compose.infra.yml ps

# 重启数据库
docker restart nexus-db
```
