# NexusArchive 增量更新手册

本手册指导客户如何应用系统更新，包括热补丁、增量更新和安全补丁。

---

## 1. 更新包类型

| 类型 | 命名格式 | 适用场景 | 停机时间 |
|:----|:---------|:---------|:---------|
| **热补丁** | `nexusarchive-patch-X.X.X-pN.tar.gz` | 紧急 Bug 修复 | < 1分钟 |
| **安全补丁** | `nexusarchive-security-X.X.X-secN.tar.gz` | CVE 漏洞修复 | 1-2分钟 |
| **增量更新** | `nexusarchive-update-X.X.X-to-Y.Y.Y.tar.gz` | 常规版本迭代 | 1-5分钟 |
| **完整安装** | `nexusarchive-offline-installer-X.X.X.tar.gz` | 首次安装/大版本升级 | 5-15分钟 |

---

## 2. 通用步骤

### 2.1 上传更新包

```bash
# 使用 scp 上传到服务器
scp nexusarchive-*.tar.gz root@服务器IP:/tmp/
```

### 2.2 解压

```bash
cd /tmp
tar -xzf nexusarchive-*.tar.gz
```

### 2.3 应用更新

```bash
# 热补丁
sudo ./apply.sh

# 增量更新/安全补丁
sudo ./upgrade.sh
```

---

## 3. 热补丁应用

热补丁用于快速修复紧急问题，通常只包含变更的文件。

```bash
# 1. 解压补丁包
tar -xzf nexusarchive-patch-2.0.0-p1.tar.gz

# 2. 应用补丁 (默认安装目录 /opt/nexusarchive)
sudo ./apply.sh

# 或指定安装目录
sudo ./apply.sh /opt/nexusarchive
```

**注意**: 补丁会自动备份原文件到 `/opt/nexusarchive_patch_backup_时间戳/`

---

## 4. 增量更新应用

增量更新包含完整的后端和前端，但不含 JDK 等大型依赖。

```bash
# 1. 解压更新包
tar -xzf nexusarchive-update-2.0.0-to-2.0.1.tar.gz

# 2. 运行升级脚本
sudo ./upgrade.sh

# 3. 验证服务
systemctl status nexusarchive
curl http://localhost:8080/api/health
```

**升级流程**:
1. 预检查（Java、磁盘空间）
2. 备份当前版本
3. 停止服务
4. 更新文件
5. 启动服务

---

## 5. 安全补丁应用

安全补丁用于修复 CVE 漏洞，建议优先处理。

```bash
# 1. 解压安全补丁
tar -xzf nexusarchive-security-2.0.0-sec1.tar.gz

# 2. 查看安全公告
cat docs/SECURITY_ADVISORY.md

# 3. 应用补丁
sudo ./apply.sh
```

**重要**: 请务必阅读安全公告 (`SECURITY_ADVISORY.md`)，了解修复的漏洞详情。

## 6. 回滚操作

如果更新后出现问题，可以使用备份进行回滚。

### 6.1 热补丁回滚

```bash
# 找到备份目录
ls -la /opt/nexusarchive_patch_backup_*

# 回滚
sudo systemctl stop nexusarchive
sudo cp /opt/nexusarchive_patch_backup_最新/app.jar /opt/nexusarchive/
sudo cp -r /opt/nexusarchive_patch_backup_最新/frontend /opt/nexusarchive/
sudo systemctl start nexusarchive
```

### 6.2 增量更新回滚

```bash
# 找到备份目录
ls -la /opt/nexusarchive_backup_*

# 回滚
sudo systemctl stop nexusarchive
sudo cp /opt/nexusarchive_backup_最新/app.jar /opt/nexusarchive/
sudo cp -r /opt/nexusarchive_backup_最新/frontend /opt/nexusarchive/
sudo cp /opt/nexusarchive_backup_最新/.env /opt/nexusarchive/
sudo systemctl start nexusarchive
```

### 6.3 安全补丁回滚

```bash
# 安全补丁备份文件名格式: app.jar.security_backup_日期
sudo systemctl stop nexusarchive
sudo cp /opt/nexusarchive/app.jar.security_backup_最新 /opt/nexusarchive/app.jar
sudo systemctl start nexusarchive
```

## 7. 常见问题

### Q: 服务启动失败怎么办？

```bash
# 1. 查看日志
journalctl -u nexusarchive -f

# 2. 检查 Java 版本
java -version

# 3. 检查配置文件
cat /opt/nexusarchive/.env
```

### Q: 更新后前端显示异常？

```bash
# 清理浏览器缓存，或强制刷新
# Chrome: Ctrl+Shift+R (Windows) / Cmd+Shift+R (Mac)

# 重载 Nginx
sudo systemctl reload nginx
```

### Q: 数据库迁移失败？

```bash
# 查看迁移日志
cat /opt/nexusarchive/migration_*.log

# 手动执行迁移脚本
cd /opt/nexusarchive/db/migration
source /opt/nexusarchive/.env
export PGPASSWORD=$DB_PASSWORD
psql -h localhost -U $DB_USER -d $DB_NAME -f V20__xxx.sql
```

### Q: 如何检查当前版本？

```bash
# 通过 API
curl http://localhost:8080/api/health

# 或查看 JAR
unzip -p /opt/nexusarchive/app.jar META-INF/MANIFEST.MF | grep Implementation-Version
```
