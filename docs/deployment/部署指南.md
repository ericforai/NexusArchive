# 数凭电子会计档案系统 (EAA) - 部署与运维指南

本文档详细说明了如何部署和运维 NexusArchive 系统。

**更新日期**: 2025-01-05

---

## 1. 环境架构概览

| 环境 | 用途 | 部署方式 | 配置文件 |
| :--- | :--- | :--- | :--- |
| **开发环境** | 开发、功能测试 | 混合模式（Docker 基础设施 + 本地应用） | `docker-compose.infra.yml` + `.env.local` |

## 2. 开发环境（混合模式）

### 2.1 前置准备

- **Node.js**: 18.0+
- **Docker Desktop**: 已安装并启动

### 2.2 启动开发环境

```bash
npm run dev
```

自动完成：
- ✅ 启动 PostgreSQL + Redis（Docker）
- ✅ 启动后端（本地）
- ✅ 启动前端（本地）

### 2.3 访问地址

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:15175 |
| 后端 API | http://localhost:19090/api |
| 数据库 | localhost:54321 |
| Redis | localhost:16379 |

### 2.4 停止服务

```bash
npm run dev:stop
```

## 3. 生产环境部署（Docker）

### 3.1 准备配置

在服务器创建 `.env.server`：

```env
DB_HOST=nexus-db
DB_PORT=5432
DB_NAME=nexusarchive
DB_USER=postgres
DB_PASSWORD=<强密码>
REDIS_HOST=nexus-redis
REDIS_PORT=6379
SPRING_PROFILES_ACTIVE=prod
```

### 3.2 上传并启动

```bash
# 方式一：使用部署脚本（推荐）
npm run deploy

# 方式二：手动部署
scp .env.server server:~/nexusarchive/
ssh server "cd ~/nexusarchive && \
  docker-compose -f docker-compose.infra.yml \
                 -f docker-compose.app.yml \
                 --env-file .env.server up -d"
```

### 3.3 验证

```bash
curl -s http://localhost/api/health
```

## 3. 火山云生产环境部署 (Production)

### 3.1 部署架构
- **服务器**: 火山云 ECS (Ubuntu 20.04/22.04 或 CentOS 7/8)
- **Web 服务器**: Nginx (反向代理 + 静态资源托管)
- **应用服务**: Java (Spring Boot Jar) 由 Systemd 托管
- **数据库**: 云数据库 (RDS) 或自建数据库

### 3.2 服务器初始化 (仅需一次)

**步骤 1: 安装基础运行环境**
生产环境**不需要** Maven 或 Node.js。
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install -y openjdk-17-jre nginx postgresql-client clamav-daemon

# CentOS/RHEL
# sudo yum install -y java-17-openjdk nginx postgresql clamav clamav-update
```

**步骤 2: 创建应用用户与目录**
为了安全，不建议使用 root 运行应用。
```bash
# 创建 nexus 用户
sudo useradd -r -s /bin/false nexus

# 创建应用目录
sudo mkdir -p /opt/nexusarchive
sudo chown -R nexus:nexus /opt/nexusarchive
```

**步骤 3: 配置生产环境变量**
在服务器上创建 `.env` 文件：
```bash
sudo nano /opt/nexusarchive/.env
```
内容示例（**务必修改为真实强密码**）：
```ini
SPRING_PROFILES_ACTIVE=prod-dameng  # 或 prod-kingbase / dev
SERVER_PORT=8080

# 生产数据库连接
DB_HOST=10.x.x.x       # 内网 IP
DB_PORT=5236           # 达梦默认端口
DB_NAME=NEXUSARCHIVE
DB_USER=SYSDBA
DB_PASSWORD=ComplexPassword123!

# 安全密钥 (使用 openssl rand -hex 32 生成)
JWT_SECRET=a1b2c3d4...

# 病毒扫描 (推荐生产环境开启 ClamAV)
VIRUS_SCAN_TYPE=clamav
VIRUS_SCAN_HOST=localhost
VIRUS_SCAN_PORT=3310
```
设置权限：
```bash
sudo chmod 600 /opt/nexusarchive/.env
sudo chown nexus:nexus /opt/nexusarchive/.env
```

**步骤 4: 配置 Systemd 服务**
将项目中的 `deploy/nexusarchive.service` 上传或复制到 `/etc/systemd/system/`。
```bash
sudo cp deploy/nexusarchive.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable nexusarchive
```

**步骤 5: 配置 Nginx**
将项目中的 `deploy/nginx.conf` 内容复制到 Nginx 配置中。
```bash
sudo nano /etc/nginx/sites-available/nexusarchive
# 粘贴配置内容，并修改 server_name 为您的域名或 IP
sudo ln -s /etc/nginx/sites-available/nexusarchive /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default  # 移除默认配置
sudo systemctl reload nginx
```

### 3.3 日常部署流程

我们提供了自动化脚本，实现 **"本地构建 -> 自动上传 -> 服务器重启"**。

**1. 修改部署配置**
编辑本地 `deploy/deploy.sh`，设置服务器 IP：
```bash
SERVER_HOST="123.45.67.89"  # 您的火山云公网 IP
SERVER_USER="root"          # 建议配置 SSH Key 免密登录
```

**2. 执行一键部署**
在项目根目录下运行：
```bash
./deploy/deploy.sh
```
脚本会自动执行：
1. `mvn package` (构建后端 Jar)
2. `npm run build` (构建前端静态资源)
3. 打包成 `tar.gz`
4. 上传到服务器 `/opt/nexusarchive`
5. 备份旧版本
6. 解压新版本 & 替换 Jar
7. 重启 `nexusarchive` 服务
8. 重载 `nginx`

## 4. 运维与监控

### 4.1 查看日志
```bash
# 查看后端实时日志
journalctl -u nexusarchive -f

# 查看 Nginx 访问日志
tail -f /var/log/nginx/access.log

# 查看 Nginx 错误日志
tail -f /var/log/nginx/error.log
```

### 4.2 服务管理
```bash
# 重启后端
sudo systemctl restart nexusarchive

# 停止后端
sudo systemctl stop nexusarchive

# 查看状态
sudo systemctl status nexusarchive
```

### 4.3 数据备份
建议设置 Crontab 定时备份数据库。
```bash
# PostgreSQL 示例
pg_dump -h localhost -U postgres nexusarchive > /backup/db_$(date +%F).sql
```

### 4.4 紧急回滚
部署脚本会自动备份旧版本文件。若新版本无法启动：
1. SSH 登录服务器。
2. 进入 `/opt/nexusarchive`。
3. 将备份的 `app.jar_backup_xxx` 重命名回 `app.jar`。
4. 将备份的 `frontend_backup_xxx` 重命名回 `frontend`。
5. 重启服务：`systemctl restart nexusarchive`。

## 5. 常见问题 (FAQ)

**Q: 部署后前端页面 404？**
A: 检查 Nginx 配置中的 `root` 路径是否正确指向 `/opt/nexusarchive/frontend`。确保 `try_files $uri $uri/ /index.html;` 配置存在以支持 React 路由。

**Q: 后端启动失败，报错 "Connection refused"？**
A: 检查 `.env` 中的数据库 IP 和端口是否正确。确认服务器的安全组（防火墙）是否允许 8080 端口（如果直接访问）或 Nginx 的 80 端口。

**Q: 上传文件提示 "413 Request Entity Too Large"？**
A: Nginx 默认限制上传大小为 1M。我们的 `nginx.conf` 已配置 `client_max_body_size 100M;`，请确认配置已生效。
