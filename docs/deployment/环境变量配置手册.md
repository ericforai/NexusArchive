# NexusArchive 环境变量配置手册

**更新日期**: 2026-01-07

---

## 1. 概述

NexusArchive 使用环境变量来管理不同环境（开发、测试、生产）的配置差异。所有敏感信息都通过环境变量注入，不直接写入代码仓库。

### 配置文件说明

| 文件 | 用途 | Git 状态 |
|------|------|----------|
| `.env.template` | 环境变量模板（包含所有可用变量及说明） | ✅ 追踪 |
| `.env` | 本地开发环境配置（从模板复制） | ❌ 忽略 |
| `.env.local` | 本地开发环境配置 | ❌ 忽略 |
| `.env.dev` | Docker 开发环境配置 | ❌ 忽略 |
| `.env.prod` | 生产环境配置 | ❌ 忽略 |

## 2. 环境变量完整清单（44个）

### 2.1 基础配置（2个）

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `SPRING_PROFILES_ACTIVE` | `dev` | Spring Profile：`dev`（开发）、`prod`（生产）、`demo`（演示） |
| `SERVER_PORT` | `19090` | 后端服务端口 |

| 变量名 | 开发环境默认 | 生产环境默认 | 说明 |
|--------|-------------|-------------|------|
| `DB_HOST` | `localhost` | `nexus-db` | 数据库主机（Docker内部用容器名） |
| `DB_PORT` | `54321` | `5432` | 数据库端口（开发用映射端口避免冲突） |
| `DB_NAME` | `nexusarchive` | `nexusarchive` | 数据库名称 |
| `DB_USER` | `postgres` | `nexus` | 数据库用户 |
| `DB_PASSWORD` | `postgres` | (必须设置) | 数据库密码 |

### 2.3 Redis 配置（4个）

| 变量名 | 开发环境默认 | 生产环境默认 | 说明 |

| `REDIS_PORT` | `16379` | `6379` | Redis端口（开发用映射端口避免冲突） |

## 3. 换电脑配置指南

### 3.1 从模板创建配置文件

\`\`\`bash
# 复制模板
cp .env.template .env
\`\`\`

### 3.2 检查端口冲突

如果新电脑已有服务占用以下端口，需要修改 `.env` 中的对应配置：

| 端口 | 用途 | 替代端口建议 |
|------|------|-------------|
| 54321 | PostgreSQL | 54322, 54323 |
| 16379 | Redis | 16380, 16381 |
| 19090 | 后端API | 8080, 9000 |
| 15175 | 前端 | 3000, 5173 |

### 3.3 生成新密钥（可选）

\`\`\`bash
# 生成 SM4 密钥
openssl rand -hex 16

# 生成审计日志 HMAC 密钥
openssl rand -base64 32

# JWT 密钥对已包含在代码库的 nexusarchive-java/keystore/ 目录
# 如需重新生成，使用 Java keytool：
keytool -genkeypair -alias jwt -alg RSA -keysize 2048 -keystore jwt.jks
\`\`\`

## 4. 生产环境配置指南

### 4.1 创建生产环境配置

\`\`\`bash
# 复制模板
cp .env.template .env.prod
\`\`\`

### 4.2 必须修改的配置

| 变量名 | 生产环境值 | 说明 |
|--------|-----------|------|
| `SPRING_PROFILES_ACTIVE` | `prod` | 必须设置为prod |
| `DB_PASSWORD` | (强密码) | 必须修改 |
| `SM4_KEY` | (新生成) | 必须修改 |
| `AUDIT_LOG_HMAC_KEY` | (新生成) | 必须修改 |

### 4.3 Docker 内部网络配置

当使用 Docker Compose 部署时，以下变量会被 `docker-compose.app.yml` 自动覆盖：

| 变量名 | Docker 环境值 |
|--------|---------------|
| `DB_HOST` | `nexus-db` (容器名) |
| `DB_PORT` | `5432` (Docker内部端口) |
| `REDIS_HOST` | `nexus-redis` (容器名) |
| `REDIS_PORT` | `6379` (Docker内部端口) |
| `JWT_PUBLIC_KEY_LOCATION` | `file:/app/keystore/jwt_public.pem` |
| `JWT_PRIVATE_KEY_LOCATION` | `file:/app/keystore/jwt_private.pem` |

### 4.4 注意事项

⚠️ **Docker 环境中不能使用 localhost**

| 变量 | 错误值 | 正确值 |
|------|--------|--------|
| `ES_HOSTS` | `localhost:9200` | 留空或 `elasticsearch:9200` |
| `VIRUS_SCAN_HOST` | `localhost` | 留空或 `clamav:3310` |

## 5. 配置验证

### 5.1 检查服务健康状态

\`\`\`bash
curl http://localhost:19090/api/health/self-check
\`\`\`

### 5.2 检查配置加载

启动后端时，查看日志中的配置信息：
\`\`\`bash
cd nexusarchive-java
mvn spring-boot:run
# 查看启动日志确认配置正确加载
\`\`\`

### 5.3 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 数据库连接失败 | `DB_HOST`/`DB_PORT` 不正确 | 检查Docker容器是否运行 |
| Redis连接失败 | `REDIS_HOST`/`REDIS_PORT` 不正确 | 检查Docker容器是否运行 |
| JWT验证失败 | 密钥文件路径不正确 | 检查 keystore 目录是否存在 |
| ES连接超时 | `ES_HOSTS` 使用了 localhost | Docker环境改为容器名或留空 |

## 6. 变量命名规则

### Spring Boot 环境变量映射规则

| Java/Yml 属性名 | 环境变量名 | 示例 |
|-----------------|-----------|------|
| `server.port` | `SERVER_PORT` | ✅ |
| `spring.profiles.active` | `SPRING_PROFILES_ACTIVE` | ✅ |
| `redis.host` | `REDIS_HOST` | ✅ |
| `jwt.public-key-location` | `JWT_PUBLIC_KEY_LOCATION` | ✅ |
| `app.security.cors.allowed-origins` | `APP_SECURITY_CORS_ALLOWED_ORIGINS` | ✅ |

**规则**：将属性名中的分隔符（`.` 或 `-`）替换为 `_`，然后转为大写。

### 特殊情况

1. **直接在 yml 中引用**：如 `\${TIMESTAMP_FALLBACK}`，必须使用完全相同的环境变量名
2. **System.getenv()**：如 `System.getenv("SM4_KEY")`，直接读取环境变量，不经过 Spring Boot
