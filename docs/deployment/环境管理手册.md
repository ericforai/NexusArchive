## 3. 测试/展示环境

| 项目 | 值 |
|:----|:-----|
| 服务器 | `180.184.54.214` |
| Compose 文件 | `docker-compose.prod.yml` |
| 环境变量 | `.env.prod` |
| 安装目录 | `/opt/nexusarchive` |

### 部署命令

```bash
# 上传配置
scp docker-compose.prod.yml .env.prod server:~/nexusarchive/

# 启动服务
ssh server "cd ~/nexusarchive && docker-compose -f docker-compose.prod.yml up -d"
```

## 4. 客户环境

| 项目 | 值 |
|:----|:-----|
| 网络 | 客户内网（可能离线） |
| 部署方式 | 离线安装包 或 Docker |
| 安装目录 | `/opt/nexusarchive`（默认） |

### 部署方式

**Docker 方式**：
```bash
docker load -i release-xxx.tar
docker-compose -f docker-compose.prod.yml up -d
```

**离线安装包**：
```bash
cd deploy/offline && ./install.sh
```

## 6. 更新流程

### 开发环境

```bash
git pull
./scripts/dev-start.sh
```

### 测试/生产环境

```bash
# 构建并上传
docker-compose -f docker-compose.prod.yml build
docker save nexusarchive-backend:latest nexusarchive-web:latest -o release.tar
scp release.tar docker-compose.prod.yml server:~/

# 服务器部署
ssh server
docker load -i release.tar
docker-compose -f docker-compose.prod.yml up -d
```

## 7. 常用命令

```bash
# 查看服务状态
docker-compose -f docker-compose.prod.yml ps

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f

# 重启服务
docker-compose -f docker-compose.prod.yml restart nexus-backend
```
