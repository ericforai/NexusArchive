## 4. 更新节奏

| 环境 | 更新频率 | 更新方式 |
|:----|:---------|:---------|
| 开发环境 | 实时 | 代码修改后自动热重载 |
| 测试环境 | 每周 / 按需 | `deploy.sh` 远程部署 |
| 客户环境 | 2周~1月 | 增量更新包 / 补丁包 |

---

## 5. 更新检查清单

### 测试环境更新前

- [ ] 本地测试通过
- [ ] 关键功能验收完成
- [ ] 数据库迁移脚本就绪

### 客户环境发布前

- [ ] 测试环境稳定运行 ≥ 2 天
- [ ] 更新日志编写完成
- [ ] 增量/补丁包构建并测试
- [ ] 通知客户准备更新窗口
- [ ] 备份方案确认

## 6. 常用命令

```bash
# 查看当前版本 Tag
git describe --tags

# 标记新的客户基线
git tag -a v2.0.1-customer -m "客户发布版本 2.0.1"
git push origin v2.0.1-customer

# 查看自上次客户发布以来的变更
git log v2.0.0-customer-baseline..HEAD --oneline

# 构建增量包
./deploy/build_incremental.sh -f 2.0.0 -t 2.0.1

# 部署到测试环境
./deploy/deploy.sh
```
