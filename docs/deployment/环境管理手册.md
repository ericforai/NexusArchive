## 3. 测试/展示环境

| 项目 | 值 |
|:----|:-----|
| 服务器 | `180.184.54.214` |
| Compose 文件 | `docker-compose.infra.yml` + `docker-compose.app.yml` |
| 环境变量 | `.env.server` |
| 安装目录 | `/opt/nexusarchive` |

### 部署命令

```bash
# 方式一：使用部署脚本（推荐）
npm run deploy

# 方式二：手动部署
scp .env.server server:~/nexusarchive/
ssh server "cd ~/nexusarchive && \
  docker-compose -f docker-compose.infra.yml \
                 -f docker-compose.app.yml \
                 --env-file .env.server up -d"
```

## 4. 客户环境

| 项目 | 值 |
|:----|:-----|
| 网络 | 客户内网（可能离线） |
| 部署方式 | 离线安装包 或 Docker |
| 安装目录 | `/opt/nexusarchive`（默认） |

### 部署方式

**Docker 方式**：
```bash
docker load -i release-xxx.tar
docker-compose -f docker-compose.infra.yml -f docker-compose.app.yml up -d
```

**离线安装包**：
```bash
cd deploy/offline && ./install.sh
```

## 6. 更新流程

### 开发环境

```bash
git pull
npm run dev
```

### 数据同步（多台 Mac）

```bash
# 离开公司前
npm run db:dump
git add db/seed-data.sql
git commit -m "db: update seed data"
git push

# 回到家后
git pull
npm run db:load
```

### 测试/生产环境

```bash
# 方式一：使用部署脚本（推荐）
npm run deploy

# 方式二：手动部署
docker-compose -f docker-compose.infra.yml \
               -f docker-compose.app.yml build
docker save nexusarchive-backend:latest nexusarchive-web:latest -o release.tar
scp release.tar server:~/

# 服务器部署
ssh server
docker load -i release.tar
docker-compose -f docker-compose.infra.yml \
               -f docker-compose.app.yml up -d
```

## 7. 常用命令

```bash
# 开发环境
npm run dev        # 启动开发环境
npm run dev:stop   # 停止开发环境
npm run db:dump    # 导出数据
npm run db:load    # 导入数据
npm run db:reset   # 重置数据库

# 生产环境
docker-compose -f docker-compose.infra.yml \
               -f docker-compose.app.yml ps
docker-compose -f docker-compose.infra.yml \
               -f docker-compose.app.yml logs -f
docker-compose -f docker-compose.infra.yml \
               -f docker-compose.app.yml restart nexus-backend
```
