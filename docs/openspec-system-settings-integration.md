## 1. 概述

### 1.1 目标
将"后台管理"一级菜单下的三个子功能整合到"系统设置"中，释放一级菜单空间，提升功能聚合度和用户体验。

### 1.2 背景
- 一级菜单空间宝贵，需要精简菜单结构
- "后台管理"下的功能与"系统设置"高度相关
- 用户生命周期管理应该与用户管理整合在一起
- 权限复核应该与角色权限管理整合在一起

---

## 2. 整合方案

### 2.1 整合映射

| 原菜单项 | 目标位置 | 整合方式 | 理由 |
|---------|---------|---------|------|
| **用户生命周期** | 系统设置 → 用户管理 Tab | 作为子 Tab 嵌入 | 用户生命周期是用户管理的核心功能，应统一管理 |
| **历史数据导入** | 系统设置 → 数据导入 Tab | 新增独立 Tab | 数据导入是系统级配置，适合放在系统设置 |
| **权限复核** | 系统设置 → 角色权限 Tab | 作为子 Tab 嵌入 | 权限复核是权限管理的重要组成部分 |

### 2.2 新的系统设置 Tab 结构

```
系统设置
├── 基础设置 (existing)
├── 用户管理 (enhanced)
│   ├── 用户列表 (existing)
│   └── 用户生命周期 (new sub-tab)
├── 角色权限 (enhanced)
│   ├── 角色列表 (existing)
│   └── 权限复核 (new sub-tab)
├── 组织架构 (existing)
├── 安全合规 (existing)
├── 集成中心 (existing)
├── 审计日志 (existing)
└── 数据导入 (new)
    └── 历史数据导入 (content)
```

## 3. 详细设计

### 3.1 用户管理 Tab 增强

#### 3.1.1 Tab 结构
- **主 Tab**: 用户管理
- **子 Tab**: 
  - 用户列表（默认，现有功能）
  - 用户生命周期（新增）

#### 3.1.2 用户生命周期功能
- 入职触发：选择员工、自动创建账号、分配角色
- 离职触发：选择员工、停用账号、回收权限
- 调岗触发：选择员工、调整角色和权限

#### 3.1.3 UI 设计
- 在用户管理页面顶部添加子 Tab 导航（类似于审计日志页面的 Tab）
- 默认显示"用户列表"Tab
- 点击"用户生命周期"Tab 显示生命周期管理界面

### 3.2 角色权限 Tab 增强

#### 3.2.1 Tab 结构
- **主 Tab**: 角色权限
- **子 Tab**:
  - 角色列表（默认，现有功能）
  - 权限复核（新增）

#### 3.2.2 权限复核功能
- 复核任务列表
- 复核执行：查看用户权限、确认/撤销权限
- 复核历史记录

#### 3.2.3 UI 设计
- 在角色权限页面顶部添加子 Tab 导航
- 默认显示"角色列表"Tab
- 点击"权限复核"Tab 显示权限复核界面

### 3.3 数据导入 Tab（新增）

#### 3.3.1 Tab 信息
- **标签**: 数据导入
- **图标**: Upload 或 FileSpreadsheet
- **路径**: `/system/settings/data-import`

#### 3.3.2 功能内容
直接嵌入"历史数据导入"页面的完整功能：
- 文件上传（支持拖拽）
- 导入预览（显示解析结果和验证错误）
- 执行导入（显示进度）
- 导入历史查询
- 错误报告下载

### 5.1 Tab 导航样式

#### 主 Tab（系统设置级别）
- 位置：页面顶部，在标题下方
- 样式：与现有系统设置 Tab 保持一致
- 高度：48px
- 激活状态：蓝色下划线 + 蓝色文字

#### 子 Tab（功能级别，如用户管理内部）
- 位置：在功能区域顶部
- 样式：类似主 Tab，但字体稍小（14px）
- 高度：40px
- 激活状态：蓝色下划线 + 蓝色文字
- 示例参考：`src/pages/settings/AuditLogView.tsx` 的实现

## 6. 兼容性考虑

- **不兼容**：独立的菜单项和路由将被移除
- **建议**：提前通知用户新功能位置，或在移除前提供引导

### 6.2 URL 重定向（可选）
如果需要保留旧 URL 的访问，可以添加重定向：
```
/system/admin/user-lifecycle → /system/settings/users (自动切换到生命周期 Tab)
/system/admin/access-review → /system/settings/roles (自动切换到权限复核 Tab)
/system/admin/legacy-import → /system/settings/data-import
```

**注意**：根据需求决定是否需要重定向。如果不需要，直接移除即可。

## 8. 风险评估

### 8.1 技术风险
- **风险**：Tab 嵌套可能影响页面性能
- **缓解**：使用懒加载，只在切换 Tab 时加载组件

### 8.2 用户体验风险
- **风险**：用户可能找不到原来的功能位置
- **缓解**：
  - 在移除前提供引导提示
  - 在帮助文档中更新功能位置说明
  - 考虑添加 URL 重定向（见 6.2）
