# 模块化系统自我审查清单（Self-Review SOP）

> 目标：用最少规则判断边界是否被破坏、数据是否多裁判、变体是否失控。
> 本清单作为 CI 审查基线，命中红线即阻断合并/发布。

---

## 1) 边界自审（Boundary）

1. **模块清单必须显式存在**
   - 每个模块都有唯一 ID、名称、职责一句话
   - 任何代码/功能必须归属某个模块，否则视为违规

2. **依赖必须单向且可解释**
   - 每个模块声明：允许依赖哪些模块
   - 出现未声明依赖 = 违规
   - 循环依赖 = 立即阻断

3. **禁止跨模块“偷看内部”**
   - 只能通过：公开 API / 公开事件 / 公共类型（按白名单）
   - 直接引用对方内部目录、内部类、内部表结构 = 违规

4. **模块可关闭性检查（关键）**
   - 关闭模块 X：
     - 系统必须能启动
     - 其他模块只能得到“能力不可用”的标准响应
   - 关闭即崩 = 高耦合，判定失败

---

## 2) 数据主权自审（SSOT & Ownership）

5. **每个核心数据对象必须有唯一所有者模块**
   - 对象必须标注 owner
   - 两个模块都能写同一对象的权威字段 = 违规

6. **写入路径必须唯一**
   - 对象创建/变更只能走 owner 模块入口
   - 旁路写（绕过 owner 直接写库/写服务）= 违规

7. **允许快照，不允许多裁判**
   - 合规追溯用快照
   - 快照不得反向修改主数据
   - 快照回写主数据 = 违规

---

## 3) 交互契约自审（Contract）

8. **模块对外只暴露三类**
   - Command/API（请求它做事）
   - Query（只读查询）
   - Event（声明发生了什么）
   - 共享表、共享缓存 key、共享内部函数 = 违规

9. **事件必须过去式 + 不可逆**
   - 事件表达事实：X 已发生
   - 事件不得表达命令：请你做 X

10. **契约变更必须可版本化**
   - 对外字段/事件结构变化必须有版本策略
   - 无版本变更 = 默认破坏性变更 = 违规

---

## 4) 流程自审（Cross-Module Process）

11. **跨模块流程必须显式归属编排 or 协同**
   - 编排：中枢按步骤调用
   - 协同：事件驱动各自响应
   - 互相 if-else 交织 = 违规

12. **流程必须有业务链路 ID**
   - 无链路 ID = 运维不可控，判定失败

---

## 5) 变体自审（Variability）

13. **参数化必须归类**
   - 字段扩展（schema extension）
   - 规则扩展（rule/DSL）
   - 流程扩展（workflow template）
   - 不能归类的配置 = 违规

14. **参数上限阈值**
   - 单模块配置项超过阈值（例如 30/50）必须拆分
   - 超阈值不拆 = 变体失控风险

15. **特例识别**
   - 单一客户独有路径必须标记
   - 必须选择：产品化（行业包）或拒绝

---

## 6) 自指输出（System Health Signals）

16. **每次发布必须输出模块健康报告（自动生成）**
   - 模块依赖图是否新增边
   - 是否出现循环依赖
   - 是否出现跨模块直连（目录/库/表）
   - 每个对象 owner 是否唯一
   - 参数数量是否超阈值
   - 是否出现客户特例标记

17. **红线触发即阻断合并/发布**
   - 循环依赖
   - 多裁判写入
   - 跨模块直连内部
   - 关闭模块导致系统无法启动

---

## 7) 最小落地产出（自指材料）

必须产出以下 4 份结构化材料：

1. **Module Manifest**：模块清单 + 允许依赖关系
2. **Data Ownership Map**：核心对象 → owner 模块
3. **Contract Catalog**：每模块公开 API/事件/查询清单
4. **Variability Registry**：配置项清单 + 归类 + 阈值统计 + 特例标记

对应文件：
- `docs/architecture/module-manifest.md`
- `docs/architecture/data-ownership-map.md`
- `docs/architecture/contract-catalog.md`
- `docs/architecture/variability-registry.md`

---

## 8) CI 约束

- **模块可关闭性检查** 与 **模块健康报告** 作为 CI 必跑项。
- CI 中若触发红线，默认阻断合并/发布。
