### 方式2: Playwright E2E自动化测试

#### 2.1 运行所有测试

```bash
# 运行所有E2E测试
npx playwright test

# 运行并查看UI（headed模式）
npx playwright test --headed

# 运行特定测试文件
npx playwright test tests/playwright/ui/legacy_import.spec.ts
```

#### 2.2 运行冒烟测试

```bash
# 运行核心路径冒烟测试
npm run test:smoke

# 或直接运行
npx playwright test tests/playwright/ui/smoke_core_paths.spec.ts
```

#### 2.3 运行P0功能测试

```bash
# P0功能测试
npx playwright test tests/playwright/ui/legacy_import.spec.ts

npx playwright test tests/playwright/ui/audit_evidence_package.spec.ts
npx playwright test tests/playwright/ui/auth_ticket.spec.ts
npx playwright test tests/playwright/ui/destruction_workflow.spec.ts
```

#### 2.4 运行P1功能测试

```bash
# P1功能测试
npx playwright test tests/playwright/ui/mfa.spec.ts
npx playwright test tests/playwright/ui/user_lifecycle.spec.ts
```

## 🐛 常见问题排查

### 问题1: 无法访问前端

**检查步骤**:
```bash
# 1. 检查容器状态
docker ps | grep frontend

# 2. 检查端口占用
lsof -i :15175

# 3. 查看前端日志
docker logs nexus-frontend-dev

# 4. 检查网络连接
curl -I http://localhost:15175
```

### 问题2: 登录失败

**检查步骤**:
- 确认后端服务正常运行
- 检查后端API是否可访问: `curl http://localhost:19090/api/health`
- 查看浏览器控制台错误信息
- 检查网络请求（F12 → Network）

### 问题3: 页面显示异常

**检查步骤**:
- 清除浏览器缓存
- 检查浏览器控制台错误
- 查看前端日志: `docker logs nexus-frontend-dev -f`
- 检查API响应是否正确
