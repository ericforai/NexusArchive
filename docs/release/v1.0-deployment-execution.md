## 🚀 快速部署（Docker方式）

### 前置条件
- ✅ Docker和Docker Compose已安装
- ✅ 服务器资源充足（CPU、内存、磁盘）
- ✅ 网络配置正确（端口可访问）

### 步骤1: 准备配置文件

```bash
# 1. 进入项目目录
cd /path/to/nexusarchive

# 2. 切换到v1.0.0标签（如需要）
git checkout v1.0.0

# 3. 创建生产环境配置文件（如不存在）
# 参考 .env.example 或 docker-compose.prod.yml 中的环境变量
cat > .env.prod << EOF
# 数据库配置
DB_PASSWORD=your_secure_password_here
POSTGRES_DB=nexusarchive
POSTGRES_USER=postgres

# 后端配置
SPRING_DATASOURCE_URL=jdbc:postgresql://nexus-db:5432/nexusarchive
SPRING_DATASOURCE_USERNAME=postgres
SPRING_DATASOURCE_PASSWORD=your_secure_password_here

# JWT密钥（生成强随机字符串）
JWT_SECRET=$(openssl rand -base64 32)

# 文件存储路径
FILE_STORAGE_PATH=/data/nexusarchive/files
EOF

# 4. 设置文件权限（保护敏感信息）
chmod 600 .env.prod
```

### 步骤2: 启动服务

```bash
# 1. 启动所有服务（数据库、Redis、后端、前端）
docker-compose -f docker-compose.prod.yml --env-file .env.prod up -d

# 2. 查看服务状态
docker-compose -f docker-compose.prod.yml ps

# 3. 查看日志（检查启动是否正常）
docker-compose -f docker-compose.prod.yml logs -f
```

### 步骤3: 等待服务就绪

```bash
# 等待数据库就绪
until docker-compose -f docker-compose.prod.yml exec -T nexus-db pg_isready -U postgres; do
  echo "等待数据库启动..."
  sleep 2
done

# 检查后端健康状态
curl http://localhost:19090/actuator/health

# 检查前端
curl http://localhost:15175
```

### 步骤4: 验证部署

```bash
# 1. 检查所有容器状态
docker-compose -f docker-compose.prod.yml ps

# 2. 检查后端日志（无错误）
docker-compose -f docker-compose.prod.yml logs backend | tail -50

# 3. 检查数据库连接
docker-compose -f docker-compose.prod.yml exec nexus-db psql -U postgres -d nexusarchive -c "SELECT version();"

# 4. 访问前端页面
# 浏览器访问: http://your-server-ip:15175
```

## 🔧 手动部署方式

### 后端部署

```bash
# 1. 构建后端
cd nexusarchive-java
mvn clean package -DskipTests

# 2. 准备数据库
createdb nexusarchive
# 或使用psql创建

# 3. 配置应用
cp src/main/resources/application.yml.example src/main/resources/application.yml
# 编辑 application.yml 设置数据库连接等

# 4. 启动服务
java -jar target/nexusarchive-backend-2.0.0.jar
```

### 前端部署

```bash
# 1. 安装依赖
npm install

# 2. 构建生产版本
npm run build

# 3. 配置Web服务器（Nginx）
# 参考 docs/deployment/ 中的nginx配置

# 4. 启动Web服务器
sudo nginx -t
sudo nginx -s reload
```

## 🔍 故障排除

### 问题1: 数据库连接失败
```bash
# 检查数据库容器状态
docker-compose -f docker-compose.prod.yml ps nexus-db

# 查看数据库日志
docker-compose -f docker-compose.prod.yml logs nexus-db

# 检查数据库连接配置
docker-compose -f docker-compose.prod.yml exec nexus-db psql -U postgres -c "SELECT 1;"
```

### 问题2: 后端启动失败
```bash
# 查看后端日志
docker-compose -f docker-compose.prod.yml logs backend

# 检查端口占用
netstat -tlnp | grep 19090

# 检查Java版本
docker-compose -f docker-compose.prod.yml exec backend java -version
```

### 问题3: 前端无法访问
```bash
# 检查前端容器状态
docker-compose -f docker-compose.prod.yml ps frontend

# 查看前端日志
docker-compose -f docker-compose.prod.yml logs frontend

# 检查端口占用
netstat -tlnp | grep 15175
```

## 🔄 回滚操作

如果部署出现问题，执行回滚：

```bash
# 1. 停止当前服务
docker-compose -f docker-compose.prod.yml down

# 2. 切换到上一版本（如需要）
git checkout v0.9.0  # 或上一个稳定版本

# 3. 恢复数据库备份（如需要）
# pg_restore -U postgres -d nexusarchive backup.dump

# 4. 重新部署
docker-compose -f docker-compose.prod.yml --env-file .env.prod up -d
```
