## 📋 部署前准备

### 1. 环境检查

#### 系统要求
- **操作系统**: Linux (推荐 Ubuntu 20.04+) / macOS / Windows
- **Java**: JDK 17+
- **Node.js**: 18+
- **数据库**: PostgreSQL 12+
- **Docker**: 20.10+ (如使用Docker部署)

#### 检查命令
```bash
# 检查Java版本
java -version

# 检查Node.js版本
node --version

# 检查PostgreSQL版本
psql --version

# 检查Docker版本 (如使用Docker)
docker --version
docker-compose --version
```

## 🚀 部署方式

### 方式一：Docker部署（推荐）

#### 1. 准备配置文件

```bash
# 复制环境变量模板
cp .env.example .env.prod

# 编辑生产环境配置
vi .env.prod
```

#### 2. 配置环境变量

主要配置项：
```env
# 数据库配置
POSTGRES_DB=nexusarchive
POSTGRES_USER=nexusarchive
POSTGRES_PASSWORD=<your-secure-password>

# 后端配置
SPRING_DATASOURCE_URL=jdbc:postgresql://db:5432/nexusarchive
SPRING_DATASOURCE_USERNAME=nexusarchive
SPRING_DATASOURCE_PASSWORD=<your-secure-password>

# JWT密钥
JWT_SECRET=<your-jwt-secret>

# 文件存储路径
FILE_STORAGE_PATH=/data/nexusarchive/files
```

#### 3. 启动服务

```bash
# 使用Docker Compose启动
docker-compose -f docker-compose.prod.yml up -d

# 查看服务状态
docker-compose -f docker-compose.prod.yml ps

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f
```

---

### 方式二：手动部署

#### 1. 后端部署

##### 1.1 编译构建
```bash
cd nexusarchive-java
mvn clean package -DskipTests

# 构建产物位置
# target/nexusarchive-backend-2.0.0.jar
```

##### 1.2 数据库准备
```bash
# 创建数据库
createdb nexusarchive

# 或使用psql
psql -U postgres -c "CREATE DATABASE nexusarchive;"
```

##### 1.3 执行数据库迁移
```bash
# Flyway会自动执行迁移脚本（如果已配置）
# 或手动执行迁移脚本
psql -U nexusarchive -d nexusarchive -f migration-scripts.sql
```

##### 1.4 配置文件
```bash
# 复制配置文件
cp application.yml.example application.yml

# 编辑配置
vi application.yml
```

##### 1.5 启动服务
```bash
# 启动Spring Boot应用
java -jar target/nexusarchive-backend-2.0.0.jar

# 或使用systemd服务
sudo systemctl start nexusarchive-backend
```

#### 2. 前端部署

##### 2.1 安装依赖
```bash
cd /path/to/nexusarchive
npm install
```

##### 2.2 构建生产版本
```bash
npm run build

# 构建产物位置: dist/
```

##### 2.3 配置Web服务器

**Nginx配置示例**:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/nexusarchive/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

##### 2.4 启动Web服务器
```bash
# Nginx
sudo nginx -t  # 测试配置
sudo nginx -s reload  # 重新加载

# 或使用Node.js
npm run preview  # 预览构建结果
```

## 🔄 回滚预案

如果部署出现问题，按照以下步骤回滚：

### 1. 停止当前服务
```bash
# Docker方式
docker-compose -f docker-compose.prod.yml down

# 手动部署
sudo systemctl stop nexusarchive-backend
sudo nginx -s stop
```

### 2. 恢复数据库（如需要）
```bash
# 恢复数据库备份
pg_restore -U nexusarchive -d nexusarchive backup.dump
```

### 3. 恢复代码
```bash
# 切换到上一版本标签
git checkout v0.9.0  # 或上一个稳定版本

# 重新构建和部署
```

### 4. 重启服务
```bash
# 按照部署步骤重新启动服务
```

## 🆘 故障排除

### 常见问题

1. **数据库连接失败**
   - 检查数据库服务是否运行
   - 检查连接配置是否正确
   - 检查防火墙规则

2. **前端无法访问**
   - 检查Web服务器是否运行
   - 检查端口是否被占用
   - 检查Nginx配置是否正确

3. **文件上传失败**
   - 检查文件存储路径权限
   - 检查磁盘空间
   - 检查文件大小限制配置
