# 临时文件存储路径配置问题排查

## 问题背景

2025-12-09 在测试电子凭证归档流程时发现：
- 上传的凭证无法通过四性检测
- 所有检测都返回 `FAIL` 状态
- 提交归档功能无法使用

## 问题现象

```text
四性检测结果:
{
  "status": "FAIL",
  "authenticity": {
    "message": "文件不存在于存储路径: /tmp/nexusarchive/uploads/test_document.pdf"
  }
}

Pool 状态统计:
PENDING_CHECK: 26  ← 检测后状态不变
PENDING_ARCHIVE: 0  ← 没有文件进入待归档
```

## 根本原因

1. **临时目录存储问题**：`archive.temp.path` 默认配置为 `/tmp/nexusarchive`
2. **系统清理**：macOS/Linux 会定期清理 `/tmp` 目录，重启后文件丢失
3. **级联故障**：文件物理丢失 → 四性检测失败 → 无法进入待归档状态 → 无法提交归档

## 解决方案

### 1. 修改 application.yml

将临时文件路径从 `/tmp` 改为持久化目录：

```yaml
# 开发环境 (dev profile)
archive:
  root:
    path: ${ARCHIVE_ROOT_PATH:./data/archives}
  temp:
    path: ${ARCHIVE_TEMP_PATH:./data/temp}

# 生产环境 (prod profile)
archive:
  temp:
    path: ${ARCHIVE_TEMP_PATH:/opt/nexusarchive/data/temp}
  root:
    path: /opt/nexusarchive/data/archives
```

### 2. 创建持久化目录

```bash
mkdir -p ./data/temp ./data/archives
```

### 3. 重启后端服务

## 经验教训

### ❌ 不要使用 /tmp 存储重要数据

| 目录类型 | 生命周期 | 适用场景 |
|---------|---------|---------|
| `/tmp` | 临时，重启清空 | 临时计算、缓存 |
| `./data/` | 持久化 | 上传文件、业务数据 |
| `/opt/app/data/` | 持久化 | 生产环境数据 |

### ✅ 文件存储最佳实践

1. **使用相对路径或可配置的绝对路径**
2. **区分临时文件和持久化文件**
3. **确保目录存在（启动时自动创建）**
4. **生产环境挂载独立存储卷**

### ✅ 调试技巧

1. 四性检测失败时，首先检查文件是否物理存在
2. 使用 curl 直接测试 API 可以绕过前端问题
3. 检查返回的详细错误信息（errors 数组）
