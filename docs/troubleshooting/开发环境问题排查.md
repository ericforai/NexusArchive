## 1. PostgreSQL 启动失败

### 症状
- `brew services start postgresql@14` 显示成功
- 但 `nc -zv localhost 5432` 返回 `Connection refused`

### 排查步骤

1. **查看日志**:
   ```bash
   tail -50 /opt/homebrew/var/log/postgresql@14.log
   ```

2. **常见错误**: `lock file "postmaster.pid" already exists`
   - **原因**: 上次异常退出后锁文件未清理
   - **解决**: 删除锁文件并重启

### 标准修复流程

```bash
# 1. 停止服务
brew services stop postgresql@14

# 2. 强制终止残留进程并删除锁文件
kill -9 $(cat /opt/homebrew/var/postgresql@14/postmaster.pid 2>/dev/null) 2>/dev/null
rm -f /opt/homebrew/var/postgresql@14/postmaster.pid

# 3. 重新启动
brew services start postgresql@14

# 4. 验证连接
nc -zv localhost 5432
```

> **注意**: Intel Mac 的路径为 `/usr/local/var/postgresql@14/`

## 2. 401 Unauthorized 错误排查

### 排查清单

| 步骤 | 检查项 | 说明 |
|------|--------|------|
| 1 | API 路径 | 确认是否存在双重前缀 (如 `/api/api/...`) |
| 2 | 重定向目标 | `request.ts` 中的 401 重定向路径是否存在 |
| 3 | Security Config | 端点是否在 `permitAll()` 列表中 |
| 4 | 后端日志 | 请求是否到达后端 |

### 常见案例

**案例1: API 路径双重前缀**
```typescript
// ❌ 错误
const API_BASE = '/api';
export const getList = () => request.get('/api/v1/abnormal/list');
// 实际请求: /api/api/v1/abnormal/list

// ✅ 正确
export const getList = () => request.get('/v1/abnormal/list');
```

**案例2: 401 重定向到不存在的页面**
```typescript
// ❌ 错误 - /login 不存在
if (response.status === 401) {
  window.location.href = '/login';
}

// ✅ 正确
if (response.status === 401) {
  window.location.href = '/system/login';
}
```

## 3. Entity-Migration 同步规则

> ⚠️ **核心原则**: 修改 Entity 字段时，**必须同步创建 Flyway 迁移脚本**。

### 必须创建迁移脚本的场景

| 操作 | 必须执行 |
|------|----------|
| 添加 Entity 字段 | `ALTER TABLE ADD COLUMN` |
| 新增 Entity 类 | `CREATE TABLE` |
| 修改字段类型 | `ALTER TABLE ALTER COLUMN` |
| 删除字段 | `ALTER TABLE DROP COLUMN` |

### 脚本命名规范

```
V{版本号}__{描述}.sql
```

- **双下划线** (`__`) 分隔版本号和描述
- 描述使用小写字母和下划线
- 示例: `V33__create_abnormal_voucher_table.sql`

### 脚本模板

```sql
-- 添加列 (幂等)
ALTER TABLE {table_name} ADD COLUMN IF NOT EXISTS {column_name} {type};
COMMENT ON COLUMN {table_name}.{column_name} IS '描述';

-- 创建表 (幂等)
CREATE TABLE IF NOT EXISTS {table_name} (
    id VARCHAR(50) PRIMARY KEY,
    ...
);
```

## 4. 开发环境快速恢复

### 一键重启全部服务

```bash
# 重启数据库和后端
brew services restart postgresql@14 && sleep 2 && \
cd nexusarchive-java && mvn spring-boot:run > ../backend.log 2>&1 &
```

### 检查服务状态

```bash
# 检查端口占用
lsof -i :5432 -i :8080 -i :5173

# 检查数据库连接数
lsof -i :5432 | grep ESTABLISHED | wc -l
```
