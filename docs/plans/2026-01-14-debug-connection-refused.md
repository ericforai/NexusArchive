# 连接拒绝错误诊断报告 (Connection Refused Diagnosis)

## 1. 系统状态检查

| 服务 | 端口 | 状态 | PID | 验证结果 |
|------|------|------|-----|----------|
| **后端 API** | 19090 | ✅ 运行中 | 49828 | `curl http://127.0.0.1:19090/api/health` 响应 200 OK |
| **前端 UI** | 15175 | ✅ 运行中 | 49964 | `curl http://127.0.0.1:15175/` 响应 index.html |
| **数据库** | 54321 | ✅ 运行中 | Docker | 日志显示连接正常 |

## 2. 日志分析
- **backend.log**: 显示 `NexusArchive 启动成功!`，API 已就绪。
- **frontend.log**: 显示 Vite Server 正常启动，Network 地址可用。
- **源代码检查**: 全局搜索 `localhost`, `:8080`, `:19090`，未发现非测试代码中的硬编码地址。

## 3. 根因推测
由于本地服务均可通过 IPv4/IPv6 正常访问，`net::ERR_CONNECTION_REFUSED` 极有可能是以下原因之一：
1. **浏览器缓存**：浏览器保留了服务重启期间的断开状态。
2. **启动时序**：在后端完全就绪（JVM 启动约需 10-20秒）前发起了请求。
3. **特定资源失败**：某个特定资源（如 HMR WebSocket）连接失败，而非整个应用。

## 4. 建议行动
1. **强制刷新**：请在浏览器中使用 `Cmd + Shift + R` (Mac) 或 `Ctrl + F5` (Windows) 强制刷新页面。
2. **检查 Network 面板**：如果问题依旧，请打开浏览器开发者工具 (F12) -> Network，查看具体是**哪个 URL** 爆红。
