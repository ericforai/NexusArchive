# 电子凭证池看板面板设计文档

> **设计日期**: 2025-01-09
> **页面**: `/system/pre-archive/pool` - 电子凭证池
> **设计目标**: 让会计人员直观理解档案归档流程，降低上手难度

---

## 1. 设计背景

### 1.1 问题陈述

当前电子凭证池页面存在以下问题：

1. **流程不清晰**: 10种预处理状态（DRAFT, PENDING_CHECK, CHECK_FAILED等）无法直观展示
2. **操作混乱**: 只有"提交归档"和"批量删除"两个按钮，用户不知道该做什么
3. **状态不可见**: 用户不知道有多少文件在哪个状态，需要频繁筛选查看
4. **批量操作困难**: 无法快速对同状态文件执行批量操作

### 1.2 设计目标

- **可视化流程**: 通过看板展示文件在各个状态的分布
- **降低认知负担**: 会计人员无需培训即可理解和使用
- **快速批量操作**: 一键对整列文件执行操作
- **响应式布局**: 适配不同屏幕尺寸

### 2.2 四列分组

| 主列 | 包含状态 | 说明 |
|------|---------|------|
| **待处理** | DRAFT, PENDING_CHECK | 新上传的凭证，等待初次处理 |
| **需要处理** | CHECK_FAILED, PENDING_METADATA | 需要人工干预或补充信息 |
| **准备就绪** | MATCH_PENDING, MATCHED | 完整性检测通过，等待归档 |
| **处理中** | PENDING_ARCHIVE, PENDING_APPROVAL, ARCHIVING | 归档流程进行中 |

## 3. 列设计

### 3.1 列头组件

```
┌─────────────────────────────────────────┐
│  待处理                  [12]           │
│  ─────────────────────────────────────  │
│  [草稿]    [待检测]                     │
└─────────────────────────────────────────┘
```

- **主标题**: 列名称 + 文件数量徽章
- **二级标签页**: 子状态切换，显示各子状态数量
- **固定操作区**: 列级批量操作按钮（最多3个）

### 3.2 二级标签页逻辑

- **默认显示**: 第一个有文件的子状态
- **切换行为**: 点击标签切换卡片列表，保持滚动位置
- **空状态**: 如果子状态无文件，标签显示为灰色但仍可点击

### 3.3 列级操作按钮

每列固定显示3个最常用操作，更多操作收入"⋯"菜单：

| 列 | 固定操作 | 更多操作 |
|----|---------|---------|
| 待处理 | 重新检测、删除、全部标记为待检测 | 移至需要处理 |
| 需要处理 | 编辑元数据、重试检测、删除 | 批量修正、移至待处理 |
| 准备就绪 | 智能匹配、手动关联、移入待归档 | 移至需要处理 |
| 处理中 | 查看详情、取消归档、批量审批 | 强制重试 |

## 4. 卡片设计

### 4.1 卡片布局

```
┌─────────────────────────────────────────────┐
│ ┃ 发票-2025-001        [检测中]            │
│ ┃ ──────────────────────────────────────── │
│ ┃ 📄 增值税发票_12345678.pdf               │
│ ┃ 💰 ¥12,345.67  📅 2025-01-09            │
│ ┃ 👤 销售有限公司                         │
│ ┃                                      [≡] │
│ ┃ [🔍查看] [✏️编辑] [🗑️删除]               │
│ ┃                                      [查看]│
└─────────────────────────────────────────────┘
```

**混合选择模式**:
- **左侧选择区** (┃): 选中时显示蓝色竖条
- **右侧内容区**: 显示卡片信息和操作按钮

### 4.2 卡片信息字段

| 字段 | 显示 | 说明 |
|------|------|------|
| 标题 | 加粗显示 | `${凭证类型}-${凭证编号}` |
| 状态徽章 | 彩色标签 | 当前处理状态 |
| 文件名 | 文件图标+名称 | 原始文件名，超长截断 |
| 金额 | 💰 前缀 | 格式化金额，无金额显示"—" |
| 日期 | 📅 前缀 | 业务日期 |
| 单位 | 👤 前缀 | 相关单位名称 |

### 4.3 卡片操作按钮

每个卡片底部固定显示3个操作按钮：

1. **查看** (🔍): 打开文件详情/预览
2. **编辑** (✏️): 编辑元数据/状态
3. **删除** (🗑️): 删除文件

```

- **蓝色左边条**: 表示已选中
- **背景高亮**: 轻微背景色变化
- **取消按钮**: 右上角显示"取消选择"按钮

## 5. 批量操作交互

### 5.1 操作流程

```
1. 用户点击列级操作按钮
   ↓
2. 自动选中该列当前子状态的所有文件
   ↓
3. 卡片进入选中状态（显示蓝色左边条）
   ↓
4. 用户可以点击"取消"按钮移除部分卡片
   ↓
5. 底部浮动栏显示"已选 X 个文件"
   ↓
6. 用户点击"执行"或调整选择后点击"执行"
   ↓
7. 显示批量操作进度，完成后刷新看板
```

### 5.2 底部浮动操作栏

```
┌──────────────────────────────────────────────────────────────┐
│  已选 12 个文件  [取消选择]  [执行重新检测]                  │
└──────────────────────────────────────────────────────────────┘
```

- **固定底部**: 无论滚动到哪里都保持可见
- **实时更新**: 选择变化时更新计数
- **操作按钮**: 执行按钮显示具体操作名称

### 5.3 选择模式切换

- **全选**: 点击"全选"按钮选中当前列所有子状态的文件
- **反选**: 点击"反选"按钮切换选择状态
- **取消**: 点击"取消选择"清空所有选择

## 6. 响应式设计

### 6.1 列宽策略

- **动态列宽**: `flex: 1`，所有列均分可用宽度
- **最小宽度**: 每列最小 280px，低于此值时触发横向滚动
- **最大宽度**: 每列最大 400px，防止在大屏上过宽

### 6.2 空列折叠

- **自动折叠**: 某列所有子状态都无文件时，该列自动折叠
- **手动展开**: 折叠的列显示为窄条，点击可展开
- **折叠提示**: 折叠列显示"X 个文件"提示

### 6.3 移动端适配

- **列堆叠**: 移动端列改为垂直堆叠
- **滑动切换**: 支持左右滑动切换列
- **全屏卡片**: 点击卡片全屏显示详情

// src/hooks/usePoolBatchAction.ts
export function usePoolBatchAction() {
  const [selection, setSelection] = useState<Set<string>>(new Set());

  const selectAll = (cardIds: string[]) => {
    setSelection(new Set(cardIds));
  };

  const toggleSelection = (cardId: string) => {
    const newSelection = new Set(selection);
    if (newSelection.has(cardId)) {
      newSelection.delete(cardId);
    } else {
      newSelection.add(cardId);
    }
    setSelection(newSelection);
  };

  const executeAction = async (action: string, selectedIds: string[]) => {
    // 执行批量操作
  };

  return { selection, selectAll, toggleSelection, executeAction };
}
```
