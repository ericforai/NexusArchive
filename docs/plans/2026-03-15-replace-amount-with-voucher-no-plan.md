# 实现计划：动态凭证号字段替换金额字段

## 1. 目标描述
在“新建原始凭证”弹窗中，根据用户选择的凭证类型，动态显示“凭证号/发票号/合同号”字段，并移除原有的“金额”字段。这有助于提高数据录入的准确性和合规性。

## 2. 拟议变更

### 前端组件 `CreateOriginalVoucherDialog.tsx` [MODIFY]
- **说明**：经核实，后端 `OriginalVoucher` 实体和数据库表 `arc_original_voucher` 已包含 `voucherNo` (或 `voucher_no`) 字段，因此无需修改后端代码或数据库架构。
- **状态调整**：
    - 移除 `amount` 和 `setAmount`。
    - 新增 `voucherNo` 和 `setVoucherNo`。
- **标签逻辑**：
    - 根据 `voucherType` 查找到对应的 `categoryCode`。
    - 映射逻辑：
        - `INVOICE` -> "发票号"
        - `CONTRACT` -> "合同号"
        - 其他 -> "凭证号"
- **UI 更新**：
    - 将金额输入框替换为文本输入框，并使用动态标签。
    - 移除金额相关的校验。
- **提交逻辑更新**：
    - 在调用 `createOriginalVoucher` 时，传递 `voucherNo: voucherNo`。
    - 移除 `amount` 参数。

## 3. 验证计划

### 自动化测试
- 无（本项目目前以手动验证为主，后续可补充单元测试）。

### 手动验证
1. 打开“单据池”页面，点击新建原始凭证。
2. 选择“发票”类类型，确认输入框标签变为“发票号”。
3. 选择“合同”类类型，确认标签变为“合同号”。
4. 输入编号并上传文件，点击创建，验证提交是否成功，且后台记录了凭证号。
