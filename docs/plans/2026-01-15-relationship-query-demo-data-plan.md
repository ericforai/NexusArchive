## 📝 实施细节

### 档案编码规范

| 类型 | 前缀 | 示例 | 说明 |
|------|------|------|------|
| 合同 | `HT-` | `HT-2025-02-001` | 合同编号 |
| 发票 | `FP-` | `FP-2025-01-001` | 发票编号 |
| 凭证 | `JZ-` | `JZ-2025-01-001` | 记账凭证号 |
| 银行回单 | `HD-` | `HD-2025-01-001` | 回单编号 |
| 报表 | `BB-` | `BB-2025-01` | 报表编号 |
| 账簿 | `ZB-` | `ZB-2025-GL` | 账簿编号 |
| 申请单 | `SQ-` | `SQ-2025-01-001` | 申请单号（自定义） |
| 报销单 | `BX-` | `BX-2025-01-001` | 报销单号（自定义） |
| 付款单 | `FK-` | `FK-2025-01-001` | 付款单号（自定义） |

### 关系类型说明

| 关系类型 | 代码 | 说明 | 示例 |
|----------|------|------|------|
| 依据 | `BASIS` | 合同、申请单等作为凭证依据 | 合同 → 凭证 |
| 原始凭证 | `ORIGINAL_VOUCHER` | 发票、回单等作为原始凭证 | 发票 → 凭证 |
| 资金流 | `CASH_FLOW` | 付款单、银行回单等资金流转 | 付款单 → 银行回单 |
| 归档 | `ARCHIVE` | 凭证归档到报表、账簿 | 凭证 → 报表 |

### 全宗代码

使用 `BR-GROUP` 作为默认全宗代码（与现有 seed 数据保持一致）。

## 📌 注意事项

1. **ID 唯一性**：所有 demo 数据的 ID 使用 `demo-` 前缀，避免与现有数据冲突
2. **编码唯一性**：档案编码需全局唯一，使用日期+序号格式
3. **外键约束**：先插入档案记录，再插入关系记录
4. **时间戳**：使用 `NOW()` 或固定时间戳，确保 `created_time` 字段有值
5. **删除标记**：所有记录 `deleted = 0`
