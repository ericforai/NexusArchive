## 1. 范围与边界

**纳入 v1.0**
- 电子档案核心流程与合规能力（全宗隔离、审计链路、四性检测、水印/预览）。
- P0 交付：数据迁移工具、审计验真与证据包、跨全宗票据管理完善。
- P1 交付：MIME 检测强化、借阅状态机完整性、检索索引优化。

**不纳入 v1.0**
- 数据库适配层（已暂停，当前仅支持 PostgreSQL）。
- 实物档案管理（路线图 v1.0 不包含）。
- 销毁流程增强（PRD 为 P2，暂列为后续迭代项）。

## 4. 里程碑与阶段计划

### Phase 0 | 需求闸门（0.5 周）
- 统一术语口径：`archive_year`/`fiscal_year`、`ArchiveObject`/`Archive`
- 审计算法口径：SM3 统一并同步文档
- 契约边界：DTO/Entity 传递规范、配置 JSON 结构化校验
- 形成 v1.0 验收清单与关键场景

### Phase 1 | P0 交付（3 周）
- **数据迁移工具**：CSV/Excel 导入 + 校验 + 事务回滚 + 错误报告
- **前端迁移页面**：`LegacyImportPage` + 导入进度/报告下载
- **审计验真与证据包**：`AuditVerificationPage` / `AuditEvidencePackagePage`
- **跨全宗票据管理**：列表/审批/详情联动 + 状态刷新 + 审批链展示

### Phase 2 | P1 合规与性能（3 周）
- MIME 检测强化：启用 Tika 或补齐 Magic 覆盖度并建立样本库
- 借阅状态机完善：`PENDING→APPROVED→BORROWED→RETURNED` + 异常态
- JSONB 索引迁移：GIN + 复合索引 + 性能对比报告

### Phase 3 | 上线准备与回归（1.5 周）
- 稳定性回归：导入/验真/借阅/检索链路
- 文档收敛：迁移、运维、验收说明
- 上线清单与回滚预案

---

## 5. 任务拆解（WBS）

### P0 必交付
**数据迁移工具**
- 后端：解析 CSV/Excel、字段校验、幂等导入、事务回滚、错误报告
- 前端：导入页面、模板下载、进度与失败报告
- 测试：合法/非法样本集 + 大文件导入压测

**审计验真与证据包**
- 前端：单条/批量/链路验真页面、证据包导出页面
- 接口联调：验证结果展示、导出进度、下载
- 测试：验真准确性与导出可用性

**跨全宗票据管理完善**
- 前端：列表与审批闭环、状态过滤、审批链展示
- 后端：确认权限与日志链路
- 测试：申请→审批→生效→撤销全闭环

### P1 重要优化
**MIME 检测强化**
- 决策：Tika 启用或 Magic 覆盖补齐
- 测试：常见与异常文件类型覆盖

**借阅状态机完善**
- 补齐状态流转与异常处理
- 增补状态机测试用例

**检索索引优化**
- 迁移脚本（GIN/复合索引）
- 性能对比与回滚预案

### P2 后续迭代（不进入 v1.0）
- 销毁流程增强
- SM2 签名增强
- 双人双控机制

## 6. 验收标准

- P0 业务链路端到端通过（导入/验真/票据闭环）。
- 合规能力可追溯（审计链、证据包、权限票据）。
- 性能基线可复现（导入吞吐与检索延迟）。
- 文档对齐（PRD/路线图/交付说明一致）。

---

## 7. 风险与对策

- **术语漂移风险**：`archive_year`/`fiscal_year` 未统一导致数据混乱。
  - 对策：Phase 0 统一策略 + 导入与查询双侧校验。
- **隐式耦合风险**：DTO/Entity 混用导致接口不稳。
  - 对策：强制映射层与契约校验。
- **路径与存储耦合**：环境差异导致预览与导入失败。
  - 对策：路径统一注入 + 兼容性测试。
- **验真/导出性能风险**：大批量验真耗时高。
  - 对策：批量验真分片与异步任务。
