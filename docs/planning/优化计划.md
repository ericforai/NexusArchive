# NexusArchive 电子会计档案系统 - 详细优化计划

> 基于三重角色审视（电子会计档案专家 + 系统架构师 + 创业者）
> 生成日期：2025-12-06
> **架构路线：高性能模块化单体 (Modular Monolith)**
> 适用场景：本地化 / 私有化部署

---

## 🎯 核心战略决策

> [!IMPORTANT]
> **架构原则**：坚决摒弃"伪微服务"（Eureka/Seata/Config Server），采用**模块化单体架构**。
> **理由**：
> 1. **合规性**：电子会计档案是法律证据库，必须使用单一数据库 ACID 事务保证"凭证 + 审计日志"的强一致性，分布式事务的"最终一致性"不可接受。
> 2. **交付效率**：私有化部署场景，单 JAR 包 + Docker 一键交付，极大降低运维成本。
> 3. **性能优先**：单体 + Redis 的性能，远超未调优的微服务集群。

**Store 设计示例：**
```typescript
// src/store/useAuthStore.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

interface AuthState {
  token: string | null
  user: User | null
  permissions: string[]
  login: (token: string, user: User) => void
  logout: () => void
  hasPermission: (perm: string) => boolean
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set, get) => ({
      token: null,
      user: null,
      permissions: [],
      login: (token, user) => set({ token, user, permissions: user.permissions }),
      logout: () => set({ token: null, user: null, permissions: [] }),
      hasPermission: (perm) => get().permissions.includes(perm),
    }),
    { name: 'auth-storage' }
  )
)
```
