# Borrowing Workflow TODOs (后续优化待办)

本待办清单记录借阅流程下一步需要补齐的功能，便于规划和拆解。当前实现为“申请 → 审批/拒绝 → 归还/取消”的简化流程，未区分审批人/复核人/归档人，也未做任务指派与多步复核。以下为待开发项：

1) **角色与权限粒度**
   - 背景：目前审批、归还、取消仅依赖状态校验，没有限定角色或用户。
   - 需求：在 `sys_role`/`sys_permission` 增加借阅相关权限（如 `borrowing_approve`, `borrowing_review`, `borrowing_archive`），接口侧校验权限或角色方可执行对应动作。
   - 交付物：权限常量、角色示例、接口层鉴权逻辑、前端按权限展示按钮。

2) **审批/复核/归档指派**
   - 背景：没有指定“谁审批/谁复核/谁归档”，按钮对所有人开放（只受权限控制）。
   - 需求：在借阅记录或独立任务表中增加 `approverId` / `reviewerId` / `archiverId`（或 `assigneeId`），创建时按规则自动指派（部门、档案密级、轮询等），支持接口更新指派人。
   - 交付物：数据字段、指派规则实现、列表支持按 `assignee` / `stage` 过滤，“我的待办”仅显示本人待处理。

3) **多步状态机**
   - 背景：当前状态机：`PENDING -> APPROVED/REJECTED -> RETURNED`（CANCELLED为申请人终止）。
   - 需求：扩展为多节点，例如：`PENDING_APPROVAL -> APPROVED -> PENDING_REVIEW -> REVIEWED -> PENDING_ARCHIVE -> ARCHIVED -> RETURNED`，每步限定可操作人/角色。
   - 交付物：状态枚举、迁移校验、接口分拆(`/approve`, `/review`, `/archive`)、前端按钮/标签同步。

4) **任务/工作流持久化**
   - 背景：WorkflowService 目前是内存 mock，未落数据库。
   - 需求：设计任务表（`taskId`, `businessType`, `businessId`, `assigneeId`, `step`, `status`, `comment`, `createdAt`, `completedAt`），提供任务列表、完成任务接口。可选接入 Camunda/Flowable。
   - 交付物：DDL、Mapper/Service/Controller、前端“我的待办”视图。

5) **审批规则与路由**
   - 背景：审批人选择缺省。
   - 需求：根据档案属性（密级、所属部门、保管期限、借阅天数）或组织树自动选择审批/复核人，支持静态路由配置表或简单脚本化规则。
   - 交付物：规则配置存储、匹配算法、创建时展示选中的审批/复核人。

6) **审计与追踪**
   - 背景：仅有全局审计，没有细化到借阅动作。
   - 需求：对创建、审批、拒绝、取消、归还、指派等动作记录审计日志（操作者、原状态→新状态、IP、备注），前端可查看操作历史。
   - 交付物：审计切面或显式写日志，借阅详情页的时间线组件。

7) **通知与提醒**
   - 背景：无消息触达。
   - 需求：待办/审批结果/逾期提醒，通过站内通知 + 可选邮件/短信；借阅即将到期、已逾期自动提醒。
   - 交付物：通知触发点、模板、渠道配置，前端提醒展示。

8) **逾期与SLA**
   - 背景：未处理逾期。
   - 需求：支持设置最长借阅天数、逾期标记与统计，必要时阻止重复借阅；可配置审批SLA超时提醒。
   - 交付物：逾期计算任务、字段（如 `overdue`、`dueDays`）、列表过滤/排序。

9) **归档复核与实物流转**
   - 背景：归还仅是状态变化。
   - 需求：可选“归档人签收/实物核验”步骤（物理档案），需要归档人确认后才算完结；支持上传回执/拍照。
   - 交付物：归档确认接口、附件上传、列表显示签收人及时间。

10) **前端体验补充**
    - 背景：当前仅基本按钮与筛选。
    - 需求：详情侧边栏/抽屉（显示指派人、审批/复核历史、附件）、批量操作（批量审批/拒绝）、高级筛选（部门、档案类别、日期范围），待办视图与已办视图分离。
    - 交付物：UI 组件、接口参数支持、导出包含审批链路与指派人。

11) **接口契约与错误码**
    - 背景：仅简单业务异常。
    - 需求：梳理借阅相关的错误码/文案（如 EAA_BORROWING_40x），在前端映射友好提示；补充 Swagger/接口文档。

12) **数据迁移与兼容**
    - 背景：新增字段/状态需要兼容存量数据。
    - 需求：提供迁移脚本（默认为单审批节点，指派人可为空），确保老数据不阻塞新流程；必要时提供回填任务脚本。

> 如需立即推进，请确认指派规则（按部门/密级/轮询等）、需要的流程节点（是否包含复核/归档）、以及通知渠道（站内/邮件/短信），再按优先级拆分迭代。
