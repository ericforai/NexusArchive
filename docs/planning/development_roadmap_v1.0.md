# 电子会计档案系统 (v1.0) 开发战略规划

> **建议人**: [Antigravity] (首席架构师)
> **原则**: 风险前置 (Front-load Risks) | 核心优先 (Kernel First) | 信创实战 (Xinchuang Reality)

面对如此复杂的信规级系统，切忌“瀑布式”一次性开发。我建议采用 **“内核 -> 合规 -> 业务”** 的同心圆迭代策略。

---

## 📅 阶段一：不可变内核 (The Immutable Kernel) - week 1-2
**目标**：建立坚不可摧的“隔离与存储”地基。**不要做 UI，只写 API 和单元测试。**

1.  **数据库适配层 (DB Adapter)**
    *   **开发**: 封装 MyBatis Plus，通过 SPI 实现 `PostgresDialect` 和 `DamengDialect`。
    *   **验证**: 必须在 Docker 容器中同时启动 PostgreSQL 和 达梦(开发版)，确保同一套 Entity 代码能跑通两套库的 DDL。
2.  **全宗隔离拦截器 (Fonds Barrier)**
    *   **开发**: 实现 MyBatis 拦截器，强制注入 `WHERE fonds_no = ?`。
    *   **验证**: 写一个“恶意测试用例”，尝试绕过拦截器读取数据，必须失败。
3.  **核心元数据模型**
    *   **开发**: `ArchiveObject` 结构化字段定义。
    *   **输出**: 能跑通“增删改查”的 MVP，且物理数据必须带有 `fonds_no`。

> **老兵忠告**: 这一步如果不稳，后面盖多高都会塌。

---

## 🔒 阶段二：合规这一关 (Compliance Gate) - week 3-4
**目标**：解决技术上最难啃的骨头——四性检测与电子证据。

1.  **四性检测引擎 (Four-Nature Engine)**
    *   **开发**: 集成 BouncyCastle (国密), Apache Tika (MIME检测), OFD Parser。
    *   **验证**: 准备“坏文件样本库”（改后缀的 exe、签名失效的 xml），确保引擎能 100% 拦截。
2.  **审计哈希链 (Audit Chain)**
    *   **开发**: 实现 `AuditLogService`，计算 `curr_hash = SHA256(prev_hash + data)`。
    *   **验证**: 模拟修改 DB 数据，运行“验真工具”报警。
3.  **防伪水印 (Server-side Watermark)**
    *   **开发**: 集成 PDFBox/iText，实现服务端流式加水印。

---

## 📦 阶段三：实物与业务 (Physical & Business) - week 5-7
**目标**：补全业务流程，让系统“好用”。

1.  **实物档案管理**
    *   **开发**: 库房模型、装盒逻辑、标签打印。
    *   **难点**: 标签打印的 PDF 坐标计算，需反复调试。
2.  **借阅与流程**
    *   **开发**: 状态机 (`BORROWED` -> `RETURNED`)、跨全宗授权票据。
3.  **检索增强**
    *   **开发**: 结构化字段索引优化，JSONB 查询性能调优（针对 PG 和 达梦分别优化）。

---

## 🖥️ 阶段四：UI 与交付 (UI & Delivery) - week 8+
**目标**：面向用户的最终组装。

1.  **前端组装**: 对接 API，实现“流式预览组件”和“动态表单”。
2.  **数据迁移工具**: 开发 `LegacyImportTool`，通过 CSV/Excel 导入历史数据并自动生成初始全宗结构。
3.  **信创环境压测**: 在鲲鹏/海光服务器上进行全链路压测，定位国产 CPU 的性能瓶颈（通常是加密解密和 JSON 解析）。

---

## 💡 关键决策建议

1.  **数据库选型**: 开发环境用 Docker 版 PG，但 **CI/CD 流水线必须包含达梦/金仓的集成测试**。不要等到最后一天才发现 SQL 语法不兼容。
2.  **OFD 预览**: 前端不要自己造轮子，直接采购成熟的 Web OFD 阅读器组件（如数科、福昕的 Web SDK），或者后端转 PDF 预览（降级方案）。
3.  **小步快跑**: 每个阶段结束都应该是一个“可部署”的版本。

> *“系统越复杂，越要尽早看见它运行的样子。”*
