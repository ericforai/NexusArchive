# 智能匹配功能说明 (Intelligent Matching Functional Specification)

## 1. 功能概述
智能匹配（Smart Matching）是电子会计档案系统的核心自动化功能，旨在自动将财务凭证（Voucher）与对应的原始凭据（Evidence，如发票、银行回单、合同等）建立关联。

该功能能够显著减少人工挂接附件的工作量，确保"账证相符"。

## 2. 核心工作流
1. **用户触发**：在档案列表选择凭证，点击"智能匹配"。
2. **场景识别**：系统自动分析凭证的借贷方科目、摘要关键词，识别业务场景（如：付款、收款、报销）。
3. **规则加载**：根据识别到的场景，加载对应的预设匹配规则模板。
4. **候选召回**：系统在原始凭证池中，根据规则（日期范围、金额匹配、关键词）搜索候选文件。
5. **智能评分**：对候选文件进行评分。
    - 90分以上：高置信度，自动推荐。
    - 60-90分：中置信度，需人工确认。
    - 多个高分候选项：若分差不足10分，提示人工介入。
6. **结果确认**：用户在"匹配预览"弹窗中查看并确认关联建议。

## 3. 算法与逻辑细节

### 3.1 业务场景识别 (Business Scene Identification)
系统通过 `BusinessSceneIdentifier` 组件分析凭证特征：

| 场景 | 识别逻辑示例 | 置信度 |
| --- | --- | --- |
| **付款业务** (PAYMENT) | 贷方：银行/现金 + 借方：应付账款 | 中高 |
| **费用报销** (EXPENSE) | 贷方：银行/现金 + 借方：费用类 + 摘要含"报销" | 高 |
| **收款业务** (RECEIPT) | 借方：银行/现金 + 贷方：应收账款 | 中高 |
| **薪资发放** (SALARY) | 贷方：银行/现金 + 借方：应付职工薪酬 | 高 |

### 3.2 匹配规则 (Matching Rules)
规则存储于数据库 `match_rule_template` 表中，支持动态配置。

**默认规则示例 (T01_PAYMENT - 付款业务):**
*   **必须关联 (Must Link)**:
    *   **税务凭据 (发票)**: 要求金额精确匹配 (`AMOUNT_EXACT`) + 日期邻近 (`DATE_PROXIMITY`, 默认 ±7天)。
    *   **结算凭据 (银行回单)**: 要求金额精确匹配 + 日期邻近。
*   **建议关联 (Should Link)**:
    *   **合同依据**: 采用模糊名称匹配 (`FUZZY_NAME`) 或 单据号关联 (`REF_NO`)。

### 3.3 搜索策略 (Candidate Finding)
`CandidateFinder` 采用两阶段搜索：
1.  **索引粗筛**：基于数据库索引，快速过滤出日期范围（默认 ±7天）和金额误差范围（默认 ±5%）内的文档。
2.  **内存精筛**：对粗筛结果进行更精细的模糊匹配（如对方单位名称相似度计算）。

## 4. 技术架构
*   **前端**: `useSmartMatching.ts` (Hook)
*   **后端**: `VoucherMatchingController` -> `VoucherMatchingEngine`
*   **核心组件**:
    *   `BusinessSceneIdentifier`: 场景识别算子
    *   `RuleTemplateManager`: 规则模板加载器
    *   `CandidateFinder`: 检索引擎
    *   `MatchingScorer`: 评分引擎

## 5. 依赖数据
*   **科目体系**: 依赖标准的会计科目属性（Account Role）。
*   **元数据**: 依赖原始单据（PDF/OFD）提取的元数据（金额、日期、发票号等）。
