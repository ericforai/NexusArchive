## 🎯 业务目标

**用户故事**: 作为系统架构师，我需要在信创环境（鲲鹏/海光 CPU）上进行全链路压测，定位性能瓶颈，确保系统在国产 CPU 环境下的性能满足要求。

**业务价值**:
- 验证系统在信创环境下的性能表现
- 定位国产 CPU 的性能瓶颈
- 提供性能优化建议
- 满足信创合规要求

**问题背景**:
- 系统已在 x86 架构上完成开发和测试
- 信创环境（ARM64 架构）可能存在性能差异
- 需要验证关键性能指标（加密解密、JSON 解析、数据库查询等）

## 📋 功能范围

### 1. 测试环境准备

#### 1.1 硬件环境

**测试服务器要求**:
- **CPU**: 鲲鹏 920 或海光处理器（ARM64 架构）
- **内存**: 32GB 以上
- **存储**: SSD 500GB 以上
- **网络**: 千兆以太网

**测试环境配置**:
- 操作系统: 统信 UOS / 麒麟 / CentOS（ARM64）
- Java 运行时: OpenJDK 17+ (ARM64)
- 数据库: PostgreSQL / 达梦 / 金仓（ARM64）
- Redis: Redis 7+ (ARM64)

#### 1.2 测试环境部署

**部署步骤**:
1. 安装操作系统和基础软件
2. 部署数据库和 Redis
3. 编译并部署应用（ARM64 版本）
4. 配置测试数据
5. 验证环境可用性

### 2. 压测场景设计

#### 2.1 基础性能场景

**场景 1: 归档接口压力测试**
- **脚本**: `archive_soak.k6.js`
- **测试内容**: 
  - 文件上传和归档
  - 四性检测性能
  - 哈希计算性能（SM3）
  - 数据库写入性能
- **性能指标**: TPS、响应时间、成功率

**场景 2: 检索接口峰值测试**
- **脚本**: `search_peak.k6.js`
- **测试内容**:
  - 高级检索查询
  - JSONB 字段查询
  - 全文检索
  - 数据库查询性能
- **性能指标**: QPS、响应时间、并发处理能力

**场景 3: 大文件上传测试**
- **脚本**: `upload_1gb.k6.js`
- **测试内容**:
  - 大文件上传（1GB+）
  - 流式处理性能
  - 内存使用情况
- **性能指标**: 上传速度、内存占用、CPU 使用率

#### 2.2 关键性能点测试

**加密解密性能**:
- SM3 哈希计算性能
- SM2 签名/验签性能
- 文件加密/解密性能

**JSON 解析性能**:
- JSONB 字段解析性能
- JSON 序列化/反序列化性能
- JSON 查询性能

**数据库性能**:
- SQL 查询性能
- JSONB 索引查询性能
- 事务处理性能
- 连接池性能

### 3. 性能基准对比

#### 3.1 x86 vs ARM64 性能对比

**对比指标**:
- CPU 性能（单核、多核）
- 内存带宽
- 加密解密性能（SM3/SM2）
- JSON 解析性能
- 数据库查询性能
- 整体 TPS/QPS

**基准数据**:
- x86 架构性能数据（已有）
- ARM64 架构性能数据（待测试）
- 性能差异百分比

#### 3.2 性能瓶颈分析

**分析方法**:
- CPU Profiling（性能分析）
- 内存 Profiling（内存分析）
- 数据库慢查询分析
- 网络 I/O 分析

**可能瓶颈**:
- CPU 性能差异（ARM64 单核性能可能较低）
- 内存访问性能
- 加密解密性能（国密算法在 ARM64 上的性能）
- JVM 性能（OpenJDK ARM64 版本）

### 4. 压测执行

#### 4.1 压测脚本

**使用现有 K6 脚本**:
- `perf/archive_soak.k6.js` - 归档接口压力测试
- `perf/search_peak.k6.js` - 检索接口峰值测试
- `upload_1gb.k6.js` - 大文件上传测试

**压测参数**:
- 并发用户数: 50, 100, 200, 500
- 持续时间: 5 分钟, 15 分钟, 30 分钟
- 压测目标: 逐步增加负载，找到性能瓶颈

#### 4.2 监控指标

**系统监控**:
- CPU 使用率
- 内存使用率
- 磁盘 I/O
- 网络 I/O

**应用监控**:
- JVM GC 情况
- 线程池使用情况
- 数据库连接池使用情况
- 接口响应时间分布

**数据库监控**:
- 慢查询日志
- 连接数
- 锁等待情况
- 缓存命中率

### 5. 性能优化

#### 5.1 优化方向

**JVM 优化**:
- 堆内存大小调优
- GC 算法选择（G1GC vs ZGC）
- JIT 编译优化

**数据库优化**:
- 索引优化
- 查询优化
- 连接池配置优化

**应用优化**:
- 线程池配置
- 缓存策略
- 批量处理优化

#### 5.2 优化验证

**验证方法**:
- 重新执行压测
- 对比优化前后性能数据
- 确认性能提升幅度
