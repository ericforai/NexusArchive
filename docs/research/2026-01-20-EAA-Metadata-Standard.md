# 电子会计档案元数据标准指南 (基于 DA/T 94-2022)

## 1. 什么是元数据？

**元数据（Metadata）** 是“关于数据的数据”。在电子会计档案领域，元数据是描述会计档案的**背景、内容、结构及其管理过程**的数据。

简单理解：如果档案内容是“一张发票”，那么元数据就是“谁在什么时候开的票、票据金额是多少、它的档号是什么、它的哈希值是多少”。

**元数据的作用**：
- **证明真实性**：通过摘要值（Hash）证明档案未被篡改。
- **保证完整性**：描述档案包含多少附件，确保资料不丢失。
- **提升可用性**：通过档号、题名、关键字快速检索。
- **实现安全性**：记录保管期限、密级和审计日志。

## 2. DA/T 94-2022 具体要求

《DA/T 94-2022 电子会计档案管理规范》将元数据分为四大实体类别，以下是核心必备项：

### 2.1 文件实体元数据 (File Entity)
这是描述电子文件本身特征的数据：
- **题名 (Title)**：档案的名称。
- **形成日期 (Creation Date)**：文件产生的具体时间。
- **档号 (Archival Code)**：唯一标识，格式通常为：全宗号-分类号-年度-保管期限-机构-案卷/件号。
- **保管期限 (Retention Period)**：如 10年、30年、永久。
- **密级 (Security Level)**：如 公开、内部、秘密。
- **文件描述元数据**：
    - **文件格式**：PDF, OFD, XML 等。
    - **摘要值 (Fixity/Hash)**：文件的唯一指纹（如 SM3 或 SHA-256 算法生成）。
    - **算法标识**：生成摘要所用的算法。

### 2.2 业务实体元数据 (Business Entity)
描述与会计业务相关的数据：
- **业务流水号/单号**：对应 ERP 中的凭证号、报销单号。
- **金额 (Amount)**：涉及的财务金额。
- **币种 (Currency)**：人民币 (CNY) 等。
- **会计科目**：涉及的借贷明细（对于记账凭证）。

### 2.3 机构人员实体元数据 (Agent Entity)
描述谁参与了档案的生命周期：
- **形成人/制单人**：谁创建了这个文件。
- **审核人/记账人**：谁进行了业务确认。
- **管理人员**：谁进行了归档操作。

### 2.4 实体关系元数据 (Relationship)
描述不同实体间的关联：
- **关联标识**：记账凭证与原始凭证（发票、附件）的关联关系。

## 3. 为什么在方案审核中提到它？

在《统一预归档库设计方案》中，由于引入了**手工批量上传**，系统无法像 API 同步那样自动获取元数据。

按照 DA/T 94 的要求，**任何入库的电子文件必须具备上述核心元数据**。如果不补齐（如题名、形成人、形成时间、保管期限），这批归档文件在法律合规性上将是不合格的，无法通过电子档案的“四性检测”。
