# 用户生命周期管理功能缺失组织关联问题分析

> **问题日期**: 2025-01  
> **问题描述**: 用户生命周期管理功能缺少组织架构关联，无法在集团型架构下使用  
> **优先级**: P0（功能无法使用）

---

## 🔍 问题分析

### 当前实现状态

**用户生命周期管理功能** (`UserLifecyclePage.tsx`)：
- ✅ 已有用户入职、离职、调岗、权限复核的UI界面
- ✅ 已有对应的API接口调用
- ❌ **缺少组织架构选择/关联功能**
- ❌ 无法在集团型架构下选择目标组织/部门

### 问题根源

1. **UI层面**：
   - 用户入职/调岗时，没有组织/部门选择器
   - 无法指定用户所属的组织或部门
   - 无法在组织层级中选择目标位置

2. **数据层面**：
   - API接口可能未包含组织/部门参数
   - 后端服务可能未实现组织关联逻辑

3. **架构设计**：
   - 系统支持集团型架构（多全宗、多法人实体）
   - 但用户生命周期管理未考虑组织层级关系

---

## 📋 功能需求

### 1. 用户入职（Onboarding）

**需要支持**：
- 选择目标组织/部门（组织架构树选择器）
- 关联到指定的法人实体（Entity）
- 关联到指定的全宗（Fonds）
- 设置组织内的角色和权限

**当前缺失**：
- 组织/部门选择器
- 组织架构数据获取
- 组织与用户的关联逻辑

### 2. 用户调岗（Transfer）

**需要支持**：
- 从当前组织/部门选择
- 选择目标组织/部门
- 保留或调整权限
- 记录调岗历史

**当前缺失**：
- 组织选择器
- 组织变更逻辑
- 权限迁移/调整机制

### 3. 用户离职（Offboarding）

**需要支持**：
- 识别用户所属的所有组织/部门
- 批量处理组织关联解除
- 权限回收
- 数据交接

**当前缺失**：
- 组织关联查询
- 批量解除关联
- 权限回收机制

### 4. 权限复核（Access Review）

**需要支持**：
- 按组织/部门查看用户权限
- 复核组织内的权限分配
- 识别跨组织权限

**当前缺失**：
- 组织维度的权限查询
- 权限复核的组织上下文

---

## 🔧 解决方案设计

### 方案A：集成现有组织架构管理（推荐）

**前提条件**：
- 系统已有组织架构管理功能（`OrgSettings`）
- 组织架构数据已存在

**实现步骤**：

1. **UI集成**：
   - 在用户生命周期页面中添加组织选择器组件
   - 调用组织架构API获取组织树
   - 实现组织树选择器（可复用组织管理页面的组件）

2. **API增强**：
   - 用户入职API：增加 `organizationId`、`departmentId` 参数
   - 用户调岗API：增加 `fromOrgId`、`toOrgId` 参数
   - 用户离职API：自动查询并处理所有组织关联

3. **数据关联**：
   - 用户表与组织表的关联
   - 用户与组织的多对多关系（一个用户可属于多个组织）

### 方案B：简化实现（临时方案）

**如果组织架构管理功能未完善**：

1. **使用全宗/实体作为组织层级**：
   - 用户入职时选择目标全宗（Fonds）
   - 用户调岗时选择目标全宗
   - 通过全宗关联来间接表示组织关系

2. **限制功能范围**：
   - 先实现基本功能（基于全宗）
   - 后续再扩展为完整的组织架构
