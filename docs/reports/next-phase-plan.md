## 🔴 P0 优先级前端任务

### 1. 审计证据链验真界面 ❌

**任务描述**: 创建审计证据链验真页面，支持单条/批量/链路验真

**后端API**:
- `POST /api/audit-log/verify/single` - 单条日志验真
- `POST /api/audit-log/verify/batch` - 批量验真
- `POST /api/audit-log/verify/chain` - 链路验真
- `GET /api/audit-log/sampling/random` - 随机抽检

**需要实现的页面**:
- `src/pages/audit/AuditVerificationPage.tsx`

**功能需求**:
- [ ] 单条日志验真（输入日志ID）
- [ ] 批量日志验真（上传日志ID列表）
- [ ] 按条件验真（全宗、时间范围、操作类型）
- [ ] 随机抽检验真
- [ ] 验真结果展示（通过/失败、失败原因）
- [ ] 验真历史记录

**预计工作量**: 2-3天

---

### 2. 证据包导出页面 ❌

**任务描述**: 创建证据包导出页面，支持导出审计证据包

**后端API**:
- `POST /api/audit-log/evidence-package/export` - 导出证据包

**需要实现的页面**:
- `src/pages/audit/AuditEvidencePackagePage.tsx`

**功能需求**:
- [ ] 选择导出条件（全宗、时间范围、操作类型）
- [ ] 导出格式选择（JSON/XML/ZIP）
- [ ] 导出进度显示
- [ ] 下载证据包
- [ ] 导出历史记录

**预计工作量**: 2-3天

---

### 3. 授权票据管理完善 ⚠️

**任务描述**: 完善授权票据管理页面，补充缺失功能

**后端API**:
- `GET /api/auth-ticket/list` - 获取票据列表（需要集成）
- `POST /api/auth-ticket/{id}/approve` - 审批票据（需要集成）
- `POST /api/auth-ticket/{id}/revoke` - 撤销票据（需要集成）

**现有页面**:
- `src/pages/security/AuthTicketApplyPage.tsx` ✅ 已实现
- `src/pages/security/AuthTicketListPage.tsx` ⚠️ 需要完善

**需要完善的功能**:

- [ ] 审批链/详情渲染（审批记录展示、审计日志关联）
- [ ] 状态更新与刷新（审批/撤销后刷新列表）

**预计工作量**: 1-2天
