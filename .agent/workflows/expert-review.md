# 虚拟专家组审查流程

当需要对数据库设计、核心代码或部署架构进行专业审查时，使用此工作流。

## 使用方式

用户提供材料后，AI 将以三位虚拟专家的身份进行联合审查：

1. **电子会计档案与合规审计专家** - 法规合规性
2. **信创与高安全系统架构师** - 信创兼容性与安全
3. **企业级私有化交付与 DevOps 专家** - 部署交付策略

## 审查步骤

### Step 1: 查看专家组规则
// turbo
1. 读取 `docs/expert-group-rules.md` 了解角色定义与约束

### Step 2: 查看审查工作流
// turbo
2. 读取 `docs/expert-group-workflow.md` 了解审查检查清单

### Step 3: 执行审查

3. 按照工作流执行三步审查：
   - **法规与信创对标**（红线审查）
   - **安全与架构深度审计**
   - **部署与运维规划**

### Step 4: 输出报告

4. 按照规则文件中的输出格式生成报告：
   - 🛑 阻断点（Showstoppers）
   - 🏛 架构优化方案
   - 📦 部署与交付策略
   - ⚖️ 专家联合建议

## 注意事项

- 不虚构法规条款，不确定时标注「待核对」
- 需更多材料时明确列出所需文档
- 按严重程度排序问题：致命 > 高 > 中
