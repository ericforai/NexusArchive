# 后台管理页面 (Admin Pages)

本目录包含系统后台管理相关的页面组件。

## 目录结构

- `AdminLayout.tsx`: 后台管理布局组件（包含侧边栏和内容区域）。
- `EntityManagementPage.tsx`: 法人管理页面（法人 CRUD 操作、法人与全宗关联）。
- `EntityConfigPage.tsx`: 法人配置管理页面（为每个法人设置独立的配置）。
- `EnterpriseArchitecturePage.tsx`: 集团架构树视图页面（展示"法人 -> 全宗 -> 档案"层级关系）。
- `FondsManagement.tsx`: 全宗管理页面（全宗 CRUD 操作、显示所属法人）。
- `FondsHistoryPage.tsx`: 全宗沿革管理页面（迁移、合并、分立、重命名操作）。
- `FondsHistoryListPage.tsx`: 全宗沿革历史查看页面（查询和展示沿革历史记录）。
- `PositionManagement.tsx`: 岗位管理页面。

## 功能说明

### 法人管理
- 法人的创建、编辑、删除
- 法人基本信息管理（名称、税号、地址、联系人等）
- 查看法人下的全宗列表
- 法人状态管理（活跃/停用）

### 法人配置管理
- 为每个法人设置独立的配置
- 配置类型：
  - **ERP 集成配置**: ERP 接口地址、密钥等
  - **业务规则配置**: 业务规则参数
  - **合规策略配置**: 合规策略参数
- 配置以 JSON 格式存储，支持灵活扩展

### 集团架构树视图
- 直观展示"法人 -> 全宗 -> 档案"的层级关系
- 显示每个法人的统计信息（全宗数量、档案数量、总容量）
- 显示每个全宗的统计信息（档案数量、容量、归档年度数量）
- 支持展开/折叠查看

### 全宗管理
- 全宗的创建、编辑、删除
- 全宗号修改权限检查
- 全宗列表展示
- 显示所属法人信息
- 创建全宗时可以选择法人

### 全宗沿革管理
- **全宗迁移**: 将档案从一个全宗迁移到另一个全宗
- **全宗合并**: 将多个全宗合并为一个全宗
- **全宗分立**: 将一个全宗分立为多个新全宗
- **全宗重命名**: 修改全宗号（需满足条件）
- **历史查询**: 查看全宗的沿革历史记录

## 规范

1. 所有页面组件必须使用 TypeScript
2. 遵循 React Hooks 最佳实践
3. 使用 `lucide-react` 图标库
4. 统一的错误处理和加载状态展示
