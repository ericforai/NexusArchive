# 后台管理页面 (Admin Pages)

本目录包含系统后台管理相关的页面组件。

## 目录结构

- `AdminLayout.tsx`: 后台管理布局组件（包含侧边栏和内容区域）。
- `EntityManagementPage.tsx`: 法人管理页面（树形结构视图、法人 CRUD 操作、层级关系管理）。
- `EntityConfigPage.tsx`: 法人配置管理页面（为每个法人设置独立的配置）。
- `EnterpriseArchitecturePage.tsx`: 集团架构树视图页面（展示"法人 -> 全宗 -> 档案"层级关系和统计）。
- `FondsManagement.tsx`: 全宗管理页面（全宗 CRUD 操作、显示所属法人）。
- `FondsHistoryPage.tsx`: 全宗沿革管理页面（迁移、合并、分立、重命名操作）。
- `FondsHistoryListPage.tsx`: 全宗沿革历史查看页面（查询和展示沿革历史记录）。
- `PositionManagement.tsx`: 岗位管理页面。
- `LegacyImportPage/`: 历史数据导入页面（模块化组件目录）。

## 功能说明

### 法人管理
- **树形结构视图**: 展示法人的层级关系（母公司-子公司）
- **展开/折叠**: 支持节点的展开和折叠
- **CRUD 操作**: 法人的创建、编辑、删除
- **基本信息管理**: 名称、税号、地址、联系人等
- **关联全宗**: 查看法人下的全宗数量
- **状态管理**: 法人状态（活跃/停用）
- **ERP 同步**: 从 YonSuite 同步法人实体（支持增量同步，自动过滤部门数据）

### 法人配置管理
- 为每个法人设置独立的配置
- 配置类型：ERP 集成配置、业务规则配置、合规策略配置

### 集团架构树视图
- 直观展示"法人 -> 全宗 -> 档案"的层级关系
- 显示统计信息（全宗数量、档案数量、容量）
- 只读统计视图，区别于法人管理的维护功能
- **业务说明**: 电子会计档案架构为"法人 → 全宗 → 档案"三层结构，法人层级关系仅用于管理维度（母公司-子公司），数据隔离基于全宗号

### 全宗管理
- 全宗的创建、编辑、删除
- 显示所属法人信息

### 全宗沿革管理
- 全宗迁移、合并、分立、重命名操作
- 历史查询

## 规范

1. 所有页面组件必须使用 TypeScript
2. 遵循 React Hooks 最佳实践
3. 使用 `lucide-react` 图标库
4. 统一的错误处理和加载状态展示
