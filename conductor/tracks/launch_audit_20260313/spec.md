# 任务规约 (Spec) - 系统上线前按钮与 API 映射审计

## 目标
1. **风险控制**: 确认所有高风险操作（删除、销毁、重置、归档）在前端均有二次确认逻辑。
2. **权限闭环**: 验证前端业务按钮对应的后端 API 均有严格的 `@PreAuthorize` 权限校验。
3. **交互优化**: 确保所有图标按钮（无文字）均配有提示信息（Tooltip/title）。

## 成功标准
1. ✅ 扫描 `buttons_inventory.csv` 中的敏感关键词，确认对应代码包含 `Popconfirm` 或 `Modal.confirm`。
2. ✅ 抽查 `ArchiveListView` 等核心页面的批量操作，确认后端 Controller 方法包含权限注解。
3. ✅ 清理所有标记为 `[Icon/Unknown]` 的按钮，确认其具备描述性属性。

## 实施路径
1. **数据分析**: 对已生成的按钮清单进行关键词过滤。
2. **代码走查**: 深入前端源码验证交互逻辑。
3. **前后端对齐**: 追踪 API 调用链，验证后端权限。
