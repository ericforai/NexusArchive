# Vercel 部署指南

## 方法一：通过 Vercel CLI 部署（推荐）

### 1. 安装 Vercel CLI

```bash
npm i -g vercel
```

### 2. 登录 Vercel

```bash
vercel login
```

### 3. 部署项目

在项目根目录执行：

```bash
vercel
```

首次部署会提示：
- Set up and deploy? **Yes**
- Which scope? 选择你的账户
- Link to existing project? **No**（首次部署）
- Project name? 输入项目名称（如：nexusarchive）
- Directory? **./**（直接回车）
- Override settings? **No**（直接回车）

### 4. 生产环境部署

```bash
vercel --prod
```

## 方法二：通过 GitHub 自动部署（推荐）

### 1. 将代码推送到 GitHub

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 在 GitHub 创建新仓库，然后推送
git remote add origin https://github.com/yourusername/nexusarchive.git
git branch -M main
git push -u origin main
```

### 2. 在 Vercel 中导入项目

1. 访问 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 **Add New Project**
3. 选择你的 GitHub 仓库
4. 配置项目：
   - **Framework Preset**: Vite
   - **Root Directory**: `./`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`

### 3. 配置环境变量（可选）

如果需要使用 Gemini API：

1. 在项目设置中找到 **Environment Variables**
2. 添加变量：
   - **Name**: `GEMINI_API_KEY`
   - **Value**: 你的 API Key
   - **Environment**: Production, Preview, Development（全选）

### 4. 部署

点击 **Deploy** 按钮，Vercel 会自动：
- 安装依赖
- 构建项目
- 部署到生产环境

## 方法三：通过 Vercel Dashboard 直接部署

1. 访问 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 **Add New Project**
3. 选择 **Import Git Repository** 或 **Upload**
4. 如果选择 Upload，需要先打包项目：

```bash
npm run build
# 然后上传 dist 目录
```

## 部署后配置

### 自定义域名

1. 在 Vercel 项目设置中找到 **Domains**
2. 添加你的自定义域名
3. 按照提示配置 DNS 记录

### 环境变量管理

- **Production**: 生产环境变量
- **Preview**: 预览环境变量（PR 部署）
- **Development**: 本地开发变量

## 注意事项

### 1. 构建优化

项目已配置好 Vite 构建，会自动：
- 代码分割
- 资源压缩
- Tree-shaking

### 2. 路由配置

`vercel.json` 已配置 SPA 路由重写，所有路由都会指向 `index.html`。

### 3. 静态资源

- 图片、字体等静态资源会自动优化
- CDN 资源（Tailwind CSS、图标库）会从 CDN 加载

### 4. 数据存储

⚠️ **重要提示**：项目使用 localStorage 存储数据，这意味着：
- 数据存储在用户浏览器中
- 不同设备/浏览器数据不共享
- 清除浏览器缓存会丢失数据

如需持久化存储，建议：
- 集成后端 API
- 使用数据库
- 使用 Vercel KV 或 Vercel Postgres

## 常见问题

### 构建失败

1. 检查 Node.js 版本（Vercel 默认使用 Node 18）
2. 检查 `package.json` 中的依赖
3. 查看构建日志中的错误信息

### 路由 404

确保 `vercel.json` 中的 rewrites 配置正确。

### 环境变量未生效

1. 检查环境变量名称是否正确
2. 确保在正确的环境（Production/Preview）中设置
3. 重新部署项目

## 更新部署

### 通过 Git 自动部署

每次推送到主分支，Vercel 会自动：
1. 检测代码变更
2. 运行构建命令
3. 部署新版本

### 手动重新部署

在 Vercel Dashboard 中：
1. 进入项目
2. 点击 **Deployments**
3. 找到要重新部署的版本
4. 点击 **⋯** → **Redeploy**

## 性能优化建议

1. **启用 Vercel Analytics**：监控性能指标
2. **使用 Vercel Speed Insights**：分析页面加载速度
3. **配置缓存策略**：已在 `vercel.json` 中配置静态资源缓存
